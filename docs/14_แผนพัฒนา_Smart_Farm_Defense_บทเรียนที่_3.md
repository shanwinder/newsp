# แผนการพัฒนาเกมบทเรียนที่ 3 แบบใหม่
# Smart Farm Defense: ศึกป้องกันสวนอัจฉริยะ

**บทเรียน:** บทที่ 3 เครื่องรดน้ำอัจฉริยะ (Condition)  
**แนวเกม:** Lane Defense / Tower Defense แบบง่าย พร้อมภาพเคลื่อนไหว  
**แรงบันดาลใจด้านรูปแบบ:** เกมป้องกันเลนแนว Plants vs. Zombies แต่ต้องออกแบบตัวละคร ฉาก ศัตรู และกติกาเป็นของตนเอง ไม่ลอกชื่อ ภาพ ตัวละคร หรือทรัพย์สินทางปัญญาใด ๆ  
**กลุ่มเป้าหมาย:** นักเรียนชั้นประถมศึกษาปีที่ 4  
**แนวคิดการเรียนรู้:** If / If-Else / Else If / Else และการจัดลำดับเงื่อนไข  
**เป้าหมายหลัก:** ทำให้บทเรียนที่ 3 กลายเป็น “เกมจริง” ที่มีภาพเคลื่อนไหว มีศัตรู มี Wave มีแรงกดดัน มีความลุ้น และใช้เงื่อนไขเป็นเครื่องมือเอาชนะเกม

---

## 1. สรุปแนวคิดหลัก

เกมบทเรียนที่ 3 ควรถูกปรับจากกิจกรรมลากคำตอบหรือจับคู่สถานการณ์ ให้กลายเป็นเกมแนว **Smart Farm Defense** หรือ “เกมป้องกันสวนอัจฉริยะ” โดยใช้ระบบเลนแบบง่าย ผู้เล่นต้องปกป้องแปลงผักจากภัยคุกคาม เช่น แดดแรง ฝนตก ถังน้ำรั่ว และบอสเมฆดำ

ผู้เรียนจะไม่ได้ควบคุมการรดน้ำโดยตรง แต่จะต้อง “ตั้งกฎ” ให้ระบบรดน้ำอัจฉริยะทำงานแทน เช่น

```text
ถ้าดินแห้ง → รดน้ำ
มิฉะนั้น → หยุดรดน้ำ
```

หรือในด่านที่ยากขึ้น

```text
ถ้าถังน้ำหมด → แจ้งเติมน้ำ
มิฉะนั้นถ้าฝนตก → หยุดรดน้ำ
มิฉะนั้นถ้าดินแห้ง → รดน้ำ
มิฉะนั้น → สังเกตต่อ
```

เมื่อกดเริ่ม Wave ศัตรูจะเคลื่อนที่เข้าหาแปลงผัก ระบบจะตรวจเงื่อนไขตามกฎที่ผู้เรียนวางไว้ แล้วเครื่องรดน้ำจะทำงานอัตโนมัติ หากกฎถูกต้อง พืชจะฟื้น ศัตรูถูกผลักกลับ ผู้เรียนได้คะแนนและคอมโบ หากกฎผิด พืชเหี่ยว น้ำท่วม ถังน้ำหมด หรือสวนเสีย HP

หัวใจของเกมคือ

> ศัตรูบุก → นักเรียนตั้งเงื่อนไข → ระบบทำงาน → ภาพเคลื่อนไหวแสดงผล → นักเรียนปรับกฎ → สวนรอด

## 2. เหตุผลที่ต้องออกแบบใหม่

จากการทดสอบรูปแบบเดิม พบว่าปัญหาหลักไม่ได้อยู่ที่เนื้อหาผิด แต่เกิดจาก “ประสบการณ์การเล่น” ยังไม่เหมือนเกมจริง เพราะมีองค์ประกอบสำคัญไม่ครบ ได้แก่

1. ไม่มีโลกของเกมที่เคลื่อนไหวตลอดเวลา
2. ไม่มีศัตรูหรือภัยคุกคามที่สร้างความลุ้น
3. ไม่มีแรงกดดันจากเวลา Wave หรือ HP
4. ไม่มีผลลัพธ์เชิงภาพที่ทำให้เข้าใจทันทีว่ากฎผิดเพราะอะไร
5. ไม่มีความรู้สึกว่าผู้เล่นกำลัง “เอาชนะ” อะไรบางอย่าง
6. ไม่มี animation ที่ทำให้ระบบดูมีชีวิต
7. กิจกรรมยังใกล้เคียงแบบฝึกมากกว่าเกม

ดังนั้น การพัฒนาใหม่ควรเริ่มจากการสร้าง **สนามเกมเคลื่อนไหว** และ **กลไกป้องกันสวน** ก่อน แล้วค่อยเชื่อมกับเนื้อหา If-Then-Else

## 3. เป้าหมายการพัฒนา

### 3.1 เป้าหมายด้านความสนุก

เกมใหม่ต้องทำให้นักเรียนรู้สึกว่า

- กำลังป้องกันสวนจริง ๆ
- มีศัตรูกำลังบุกเข้ามา
- ต้องรีบคิดก่อนสวนเสียหาย
- ถ้าตั้งกฎถูกจะเห็นผลทันที
- ถ้าตั้งกฎผิดจะเห็นปัญหาเป็นภาพ
- อยากเล่นซ้ำเพื่อให้ได้ 3 ดาว
- ด่านสุดท้ายรู้สึกเหมือนสู้บอส

### 3.2 เป้าหมายด้านการเรียนรู้

หลังเล่นจบ นักเรียนควรสามารถ

1. อธิบายความหมายของเงื่อนไขแบบ `ถ้า...แล้ว...`
2. เข้าใจการเลือกคำสั่งตามสถานการณ์
3. เข้าใจว่า Else คือกรณีที่ไม่เข้าเงื่อนไขก่อนหน้า
4. เข้าใจว่าการตรวจเงื่อนไขจากบนลงล่างมีผลต่อผลลัพธ์
5. เข้าใจว่าบางเงื่อนไขต้องมาก่อน เช่น ฝนตกควรมาก่อนดินแห้ง
6. เข้าใจว่าถ้าถังน้ำหมด ต้องตรวจถังน้ำก่อนสั่งรดน้ำ
7. แก้ไขกฎเมื่อผลลัพธ์ผิดพลาดได้
8. อธิบายเหตุผลของกฎที่ตนเองวางได้

## 4. รูปแบบเกมโดยรวม

### 4.1 ประเภทเกม

เกมนี้เป็นแนว **Lane Defense / Tower Defense แบบง่าย** โดยมี 3–4 เลน แต่ละเลนคือแปลงผักหนึ่งแปลง ศัตรูหรือภัยคุกคามเคลื่อนที่จากด้านขวาเข้าหาแปลงผักทางด้านซ้าย

### 4.2 สิ่งที่ผู้เล่นทำ

ผู้เล่นไม่ได้ยิงศัตรูเอง แต่ทำหน้าที่เป็น “วิศวกรระบบน้ำ” โดยตั้งกฎควบคุมระบบรดน้ำ เช่น

```text
ถ้า [ฝนตก] → [หยุดรดน้ำ]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [สังเกตต่อ]
```

เมื่อเริ่ม Wave ระบบจะทำงานตามกฎที่ผู้เล่นสร้าง

### 4.3 สิ่งที่ระบบเกมทำ

ระบบเกมจะ

1. สร้างสถานะของแต่ละเลน เช่น ดินแห้ง ฝนตก ถังน้ำหมด
2. ปล่อยศัตรูหรือภัยคุกคามเข้ามา
3. ประเมินกฎของผู้เรียนจากบนลงล่าง
4. สั่งให้หัวรดน้ำหรืออุปกรณ์ในสวนทำงาน
5. แสดง animation ตามผลลัพธ์
6. คิดคะแนน HP สวน น้ำในถัง และดาว
7. ให้ feedback เชิงเหตุผลหลังแต่ละ Wave

### 5.2 หลักการออกแบบหน้าจอ

หน้าจอต้องทำให้ผู้เรียนเห็น 3 สิ่งพร้อมกัน

1. **เกิดอะไรขึ้นในสวน** เช่น แดดบุก เมฆฝนลอย ถังน้ำรั่ว พืชเหี่ยว
2. **ระบบกำลังใช้กฎอะไร** เห็นแผง If / Else If / Else ชัดเจน
3. **ผลจากกฎเป็นอย่างไร** เช่น น้ำพุ่ง พืชฟื้น น้ำท่วม ศัตรูถอย หรือ HP ลด

## 6. ตัวละครและองค์ประกอบภาพ

### 6.1 ตัวละครผู้ช่วย

**ชื่อ:** น้องหยดน้ำ  
**บทบาท:** ตัวละครประจำบทเรียน คอยแสดงท่าทางดีใจ ตกใจ ชี้แนะ และฉลองเมื่อผ่านด่าน

Animation ที่ควรมี:

- Idle: เด้งขึ้นลงเบา ๆ
- Happy: กระโดดดีใจ
- Alert: ตกใจเมื่อศัตรูเข้าใกล้
- Sad: หน้าเศร้าเมื่อพืชเสีย HP
- Victory: หมุนตัวหรือปล่อยประกาย

### 6.2 พืชในแปลง

พืชควรมีอย่างน้อย 4 สถานะ

1. Healthy: สดชื่น สีเขียว
2. Dry: เหี่ยว ใบตก
3. Recovering: กำลังฟื้นหลังได้รับน้ำ
4. Flooded: จมน้ำหรือมีน้ำขัง

### 6.3 หัวรดน้ำ

หัวรดน้ำควรมี animation

1. Idle: หยุดนิ่ง
2. Activate: หันหัวหรือสั่นก่อนเปิดน้ำ
3. Spray: พ่นน้ำเป็นหยด
4. Error: สั่นและมีเครื่องหมายเตือนเมื่อถังน้ำหมด

### 6.4 ถังน้ำ

ถังน้ำควรแสดงระดับน้ำ

1. Full
2. Medium
3. Low
4. Empty
5. Leaking

### 6.5 ศัตรูและภัยคุกคาม

ศัตรูไม่ควรเป็นซอมบี้ แต่ควรเป็นภัยธรรมชาติหรือปัญหาของสวน เพื่อให้เหมาะกับบริบทการเรียน

| ศัตรู | ภาพ | ผลกระทบ |
|---|---|---|
| แดดแผดเผา | ดวงอาทิตย์หน้าโกรธ | ทำให้ดินแห้งเร็ว |
| เมฆฝนจอมป่วน | เมฆดำ | ทำให้ฝนตก ถ้ารดน้ำซ้ำจะน้ำท่วม |
| ถังน้ำรั่ว | ถังน้ำเดินได้หรือรั่ว | ทำให้น้ำลด |
| หนอนดินแห้ง | หนอนตัวเล็ก | ลดความชื้นของดิน |
| บอสเมฆดำ | เมฆใหญ่มี HP | เปลี่ยนสถานการณ์หลายแบบ |

## 7. กลไกเกมหลัก

### 7.1 ระบบเลน

แต่ละเลนมีข้อมูลสถานะ เช่น

```javascript
{
  laneId: 1,
  soil: 'dry',
  rain: false,
  tank: 'ready',
  plantHp: 80,
  moisture: 30,
  flooded: false,
  enemyType: 'sun'
}
```

สถานะสำคัญ ได้แก่

- `soil`: dry / wet
- `rain`: true / false
- `tank`: ready / empty
- `plantHp`: 0–100
- `moisture`: 0–100
- `flooded`: true / false
- `enemyType`: sun / rainCloud / leakingTank / boss

### 7.2 ระบบ Wave

Wave คือรอบการบุกของศัตรู ในแต่ละ Wave ระบบจะกำหนดว่าเลนใดมีศัตรูอะไร และสถานะของแปลงเป็นอย่างไร

ตัวอย่าง Wave:

```javascript
{
  name: 'Wave 1: แดดบุก',
  duration: 20,
  lanes: [
    { laneId: 1, soil: 'dry', rain: false, tank: 'ready', enemyType: 'sun' },
    { laneId: 2, soil: 'wet', rain: false, tank: 'ready', enemyType: null }
  ]
}
```

### 7.3 ระบบกฎ

ผู้เรียนสร้างกฎในแผงควบคุม เช่น

```javascript
[
  { type: 'if', condition: 'rain', action: 'stop' },
  { type: 'else_if', condition: 'soil_dry', action: 'water' },
  { type: 'else', condition: 'else', action: 'observe' }
]
```

### 7.4 ระบบประเมินกฎ

เกมต้องประเมินกฎจากบนลงล่าง

```javascript
function evaluateRules(lane, rules) {
  for (const rule of rules) {
    if (rule.type === 'else') return rule.action;
    if (checkCondition(lane, rule.condition)) return rule.action;
  }
  return 'no_action';
}
```

### 7.5 ระบบตรวจเงื่อนไข

```javascript
function checkCondition(lane, condition) {
  if (condition === 'soil_dry') return lane.soil === 'dry';
  if (condition === 'soil_wet') return lane.soil === 'wet';
  if (condition === 'rain') return lane.rain === true;
  if (condition === 'tank_empty') return lane.tank === 'empty';
  if (condition === 'tank_ready') return lane.tank === 'ready';
  return false;
}
```

### 7.6 ระบบใช้คำสั่ง

```javascript
function applyAction(lane, action) {
  if (action === 'water') {
    if (lane.tank === 'empty') return { result: 'no_water', hpChange: -15, waterChange: 0 };
    if (lane.rain === true || lane.soil === 'wet') return { result: 'flood', hpChange: -20, waterChange: -10 };
    return { result: 'recover', hpChange: +20, waterChange: -10 };
  }

  if (action === 'stop') {
    if (lane.soil === 'dry' && !lane.rain) return { result: 'too_dry', hpChange: -15, waterChange: 0 };
    return { result: 'safe_stop', hpChange: +5, waterChange: 0 };
  }

  if (action === 'refill') {
    if (lane.tank === 'empty') return { result: 'refill_called', hpChange: +5, waterChange: +30 };
    return { result: 'unneeded_refill', hpChange: -5, waterChange: 0 };
  }

  if (action === 'observe') return { result: 'observe', hpChange: 0, waterChange: 0 };

  return { result: 'no_action', hpChange: -10, waterChange: 0 };
}
```

## 8. ภาพเคลื่อนไหวที่ต้องมี

### 8.1 Animation ขั้นต่ำที่จำเป็นที่สุด

ชุดนี้ต้องมีเพื่อให้เกมรู้สึกมีชีวิต

1. **Enemy Move** — ศัตรูเคลื่อนที่จากขวาไปซ้าย
2. **Plant Idle / Dry / Recover** — พืชมีสถานะและเปลี่ยนท่าทางได้
3. **Water Spray** — หัวรดน้ำพ่นน้ำออกมาเป็นหยดหรือเส้นน้ำ
4. **Rain Fall** — เมฆปล่อยฝนลงเลน
5. **Flood Rise** — น้ำค่อย ๆ สูงขึ้นเมื่อรดน้ำผิดเวลา
6. **Tank Water Level** — น้ำในถังลดหรือหมด
7. **Hit / Damage Effect** — พืชสั่นหรือมีเครื่องหมายเตือนเมื่อเสีย HP
8. **Success Effect** — ประกายหรือแสงเมื่อแก้เงื่อนไขถูก

### 8.2 Animation ระดับเพิ่มความสนุก

1. Combo text เด้งขึ้น
2. กล้องสั่นเบา ๆ ตอนพลาด
3. ศัตรูถอยหลังเมื่อโดนน้ำถูกต้อง
4. บอสโดนโจมตีและ HP ลด
5. ดาวหมุนตอนผ่านด่าน
6. น้องหยดน้ำกระโดดดีใจ

## 9. ทางเลือกด้านภาพและ Asset

ข้อดี:

- ดูเป็นเกมที่สุด
- animation ลื่น
- ควบคุมท่าทางได้
- เหมาะกับ Phaser

- ต้องมีไฟล์ภาพหลายชุด
- ต้องออกแบบหรือจัดหา asset
- ใช้เวลาพัฒนามากกว่า

### 9.2 ทางเลือก B: ใช้ภาพ PNG หลายสถานะ

เช่น พืชดี พืชเหี่ยว พืชจมน้ำ ศัตรูแดด เมฆฝน ถังน้ำเต็ม ถังน้ำหมด

ข้อดี:

# Stage 7: แดดบุกสวน

## 10.1 เป้าหมายการเรียนรู้

มี 2 เลน ศัตรูคือ “แดดแผดเผา” เคลื่อนที่เข้าหาพืช ดินในบางเลนแห้งลงเรื่อย ๆ หากผู้เรียนไม่ตั้งกฎให้ระบบรดน้ำ พืชจะเหี่ยวและ HP สวนลด

## 10.3 องค์ประกอบในด่าน

- เลน 1: ดินแห้ง มีศัตรูแดด
- เลน 2: ดินชื้น ไม่มีศัตรูหรือมีแดดอ่อน
- หัวรดน้ำประจำเลน
- ถังน้ำพร้อมใช้งาน
- แผงกฎ If / Else
- การ์ดเงื่อนไข: ดินแห้ง
- การ์ดคำสั่ง: รดน้ำ, หยุดรดน้ำ

## 10.4 Gameplay

1. เกมแสดงพืช 2 เลน
2. แดดเริ่มเคลื่อนเข้ามา
3. ผู้เรียนวางกฎในแผง
4. กดเริ่ม Wave
5. ถ้ากฎถูก หัวรดน้ำพ่นน้ำใส่เลนดินแห้ง
6. พืชฟื้น ศัตรูแดดถอย
7. ถ้ารดน้ำเลนดินชื้น จะเกิดน้ำขังและเสียคะแนน
8. ผ่านด่านเมื่อพืชทั้งสองเลนรอด

## 10.5 Animation เฉพาะด่าน

- ดวงอาทิตย์หน้าโกรธลอยจากขวาไปซ้าย
- พื้นดินเปลี่ยนจากชื้นเป็นแห้ง
- พืชก้มลงเมื่อแห้ง
- หัวรดน้ำหมุนและพ่นน้ำ
- พืชเด้งขึ้นเมื่อฟื้น
- ข้อความ “Water Save!” เด้งขึ้น

## 10.6 Feedback ตัวอย่าง


กรณีผิด:

```text
ดินแห้งแต่ระบบไม่ได้รดน้ำ พืชจึงเหี่ยว ลองใช้กฎ ถ้าดินแห้ง → รดน้ำ
```

## 10.13 เป้าหมายการเรียนรู้

#### 1 ดาว

## 14. ข้อจำกัดที่ต้องพิจารณาตามตรง

### 14.1 Asset คือปัจจัยสำคัญที่สุด

ถ้าไม่มีภาพหรือ animation ที่ดีพอ เกมจะยังดูเหมือน prototype แม้กลไกจะดีขึ้นแล้ว ดังนั้นควรเตรียม asset ขั้นต่ำ ได้แก่

- เมฆฝน
- ถังน้ำ
- หัวรดน้ำ
- น้ำท่วม
- ฉากสวน
- เอฟเฟกต์น้ำ

### 14.2 อย่าทำ PvZ เต็มระบบ

แนวนี้ควรเป็น **PvZ-lite** ไม่ใช่เกมเต็มแบบ Plants vs. Zombies เพราะหากทำระบบปลูกพืช ร้านค้า การสะสมพลังงาน ศัตรูหลายชนิด และด่านยาวมาก จะซับซ้อนเกินไปสำหรับ ป.4 และใช้เวลาพัฒนาสูง

- ด่านแรกใช้ 1 เงื่อนไข
- ด่านสองใช้ 2 เงื่อนไข
- ด่านสามใช้ 3 เงื่อนไข + Else

ไม่ควรใส่เงื่อนไขมากเกินไปในครั้งเดียว

### 14.4 มือถืออาจลากยาก

ต้องมีระบบ tap-to-place สำรอง ไม่ควรพึ่ง drag-and-drop อย่างเดียว

### 14.5 Phaser ทำได้ แต่ต้องระวัง performance

หากใส่ particle ฝน น้ำ เอฟเฟกต์ และศัตรูจำนวนมาก อุปกรณ์เก่าอาจกระตุก ควรจำกัดจำนวนวัตถุบนหน้าจอ และใช้ animation แบบประหยัดทรัพยากร

### 14.6 ไม่ควรเปลี่ยนฐานข้อมูลถ้าไม่จำเป็น

ระบบเดิมมีการบันทึก stage, score, duration และ attempts อยู่แล้ว ควรใช้ระบบเดิมให้มากที่สุด เพื่อลดความเสี่ยง

## 21. สรุป

แนวทางใหม่ของบทเรียนที่ 3 ควรขยับจากเกมลากคำตอบไปสู่เกม **Smart Farm Defense** ที่มีสนามเกมเคลื่อนไหว มีศัตรู มี Wave มี HP มีภาพพืชเหี่ยว/ฟื้น มีน้ำท่วม มีหัวรดน้ำพ่นน้ำ และมีบอสในด่านสุดท้าย

สิ่งที่ทำให้เกมนี้ต่างจากแบบฝึกคือ

> นักเรียนไม่ได้ตอบคำถามเรื่องเงื่อนไข แต่กำลังใช้เงื่อนไขเพื่อป้องกันสวนให้รอด

แนวทางนี้ยังคงสอนเนื้อหา If-Then-Else ได้ครบถ้วน แต่เพิ่มความสนุก ความตื่นเต้น และภาพจำที่ชัดเจนกว่าเดิมมาก โดยเฉพาะเมื่อมี animation ที่ทำให้ผู้เรียนเห็นผลของการคิดผิดและคิดถูกในทันที
