# แผนการพัฒนาเกมบทเรียนที่ 3
# Smart Farm Manager v3: สายพานตรรกะฟาร์มอัจฉริยะ

**บทเรียน:** บทที่ 3 ผู้จัดการฟาร์มอัจฉริยะ  
**ชื่อเกมรวม:** Smart Farm Manager: สายพานตรรกะฟาร์มอัจฉริยะ  
**รูปแบบเกม:** Conveyor Belt Logic Puzzle + Visual Block Programming + Drag & Drop  
**กลุ่มเป้าหมาย:** นักเรียนชั้นประถมศึกษาปีที่ 4  
**แกนการเรียนรู้:** If / If-Else / If-Else If-Else  
**ธีมหลัก:** ผลผลิตจากพืชผัก ผลไม้ และผลผลิตจากสัตว์  
**เป้าหมายของแผนนี้:** ปรับปรุงบทที่ 3 ให้ธีมชัดขึ้น เล่นสนุกขึ้น ท้าทายขึ้น และแก้ปัญหา UI/UX ที่พบจากการทดสอบ โดยเฉพาะ item tray ด้านล่าง การคลิกตรวจสอบวัตถุ ตัวหลอก และการจัดพื้นที่ไม่ให้วัตถุถูกบดบัง

---

## 1. สรุปแนวคิดการปรับปรุง

จากภาพรวมล่าสุด บทที่ 3 มีทิศทางที่ดีแล้ว โดยใช้แนวเกมสายพานตรรกะที่ให้ผู้เรียนลากบล็อกคำสั่งไปสร้างโปรแกรมควบคุมฟาร์มอัตโนมัติ แต่ยังควรปรับให้ธีมของแต่ละเกมสอดคล้องกันมากขึ้น และเพิ่มความลึกในการเล่นด้วยตัวหลอกและระบบตรวจสอบวัตถุ

การปรับปรุงรอบนี้กำหนดโครงสร้างใหม่ดังนี้

| เกม | ธีมใหม่ | ตรรกะหลัก | แนวคิด |
|---|---|---|---|
| เกมที่ 1 / Stage 7 | ผลผลิตจากพืชผัก | If | พบกรณีพิเศษแล้วส่งไปจัดการ |
| เกมที่ 2 / Stage 8 | ผลผลิตจากผลไม้ | If / Else | แยกผลไม้เป็น 2 เส้นทาง |
| เกมที่ 3 / Stage 9 | ผลผลิตจากสัตว์ | If / Else If / Else | คัดเกรดผลผลิตหลายระดับ |

หลักการสำคัญของเวอร์ชันนี้คือ

> ผู้เรียนไม่ได้แค่ลากบล็อกให้ครบ แต่ต้องสังเกตวัตถุ วิเคราะห์ตัวหลอก และสร้างเงื่อนไขให้ระบบฟาร์มอัจฉริยะคัดแยกผลผลิตได้ถูกต้อง

## 3. Core Game Loop

เกมทุกด่านใช้วงจรการเล่นเดียวกัน เพื่อให้เด็กเข้าใจระบบเร็วและโฟกัสกับตรรกะมากขึ้น

```text
ดูภารกิจ
→ ตรวจรายการวัตถุในรอบนี้
→ คลิกดูรายละเอียดวัตถุหรือตัวหลอกใน item tray
→ ลากบล็อกเงื่อนไขและคำสั่งไปวางในแผงสร้างโปรแกรม
→ กดเริ่มสายพาน
→ วัตถุเคลื่อนผ่านเครื่องสแกน
→ ระบบตรวจเงื่อนไขจากบนลงล่าง
→ ประตูหรือเครื่องจักรทำงานตามกฎ
→ แสดงผลถูก/ผิดด้วย Animation
→ สรุปผลเป็นดาวและ feedback
```

### 3.1 สิ่งที่ต้องคงไว้

- ใช้ Drag & Drop เป็นแกนหลัก
- กฎที่ผู้เรียนสร้างต้องส่งผลต่อเกมจริง
- มี Animation ของวัตถุ เครื่องสแกน ประตู และผลลัพธ์
- ใช้ดาวเป็นคะแนนหลัก
- ไม่มีคะแนนตัวเลข คอมโบ หรือเหรียญในเวอร์ชันนี้
- รองรับ asset abstraction เพื่อเปลี่ยนจากไอคอนเป็นภาพกราฟิกในอนาคต

## 4. โครงสร้างเกมที่ 1: โรงคัดผักสวนครัว

**Stage:** 7  
**ธีม:** ผลผลิตจากพืชผัก  
**ตรรกะหลัก:** If  
**แนวคิด:** ถ้าพบเงื่อนไขพิเศษ ให้ส่งวัตถุไปจัดการ ถ้าไม่เข้าเงื่อนไข ให้ปล่อยผ่านลงตะกร้าปกติ

### 4.1 รูปแบบกฎของเกมที่ 1

```text
ถ้า [พบเงื่อนไขพิเศษ] → [ส่งไปจัดการ]
ถ้าไม่เข้าเงื่อนไข → ปล่อยผ่านอัตโนมัติ
```

ใน UI ควรแสดงเฉพาะแถว `ถ้า ... → ...` ไม่ต้องแสดง Else เพื่อให้ตรงกับเป้าหมายว่าเกมที่ 1 สอน If เท่านั้น

---

### 4.2 ด่าน 7-1: แครอทเปื้อนโคลน

**โจทย์:** คัดแครอทเปื้อนโคลนไปล้างน้ำก่อนลงตะกร้า  
**ตรรกะ:**

```text
ถ้า [แครอทเปื้อนโคลน] → [ส่งเข้าเครื่องล้าง]
```

#### วัตถุหลัก

- แครอทเปื้อนโคลน
- แครอทสะอาด

#### ตัวหลอก

- แครอทสีเข้ม แต่ไม่มีคราบโคลน
- แครอทมีดินติดปลายเล็กน้อย แต่ไม่ถือว่าเปื้อนโคลน
- แครอทมีเงาสีน้ำตาลจากแสง ไม่ใช่คราบโคลน

#### จุดประสงค์ของตัวหลอก

ให้ผู้เรียนแยกให้ออกระหว่าง “สีเข้ม” กับ “คราบโคลนจริง” เพื่อฝึกการอ่านเงื่อนไขให้ชัด

#### Animation

- แครอทวิ่งบนสายพาน
- เครื่องสแกนส่องไฟตรวจคราบโคลน
- ประตูเปิดส่งแครอทเข้าเครื่องล้าง
- เครื่องล้างพ่นน้ำและแครอทเด้งออกมาสะอาด
- ถ้าพลาด แครอทเปื้อนโคลนตกลงตะกร้าและตะกร้ามีคราบเลอะ

#### Feedback ตัวอย่าง

```text
แครอทชิ้นนี้สีเข้ม แต่ไม่ได้มีคราบโคลนจริง จึงไม่ต้องส่งเข้าเครื่องล้าง
```

---

### 4.3 ด่าน 7-2: ผักกาดมีหนอน

**โจทย์:** ส่งผักกาดที่มีหนอนไปโต๊ะตรวจศัตรูพืช  
**ตรรกะ:**

```text
ถ้า [ผักกาดมีหนอน] → [ส่งไปโต๊ะตรวจศัตรูพืช]
```

#### วัตถุหลัก

- ผักกาดมีหนอน
- ผักกาดปกติ

#### ตัวหลอก

- ผักกาดมีจุดดิน
- ผักกาดมีรูเล็ก ๆ แต่ไม่พบหนอน
- ผักกาดใบงอ แต่ไม่มีศัตรูพืช

#### จุดประสงค์ของตัวหลอก

ให้เด็กสังเกตว่าเงื่อนไขของด่านคือ “มีหนอน” ไม่ใช่เพียงผักดูไม่สมบูรณ์

#### Animation

- หนอนขยับบนใบผักอย่างน่ารัก
- ถ้าส่งถูก โต๊ะตรวจมีแว่นขยายและหนอนเด้งออก
- ถ้าส่งผิด หนอนกัดใบและผักทำหน้าบูด
- ถ้าผักปกติถูกส่งไปตรวจ เครื่องสแกนสั่นและขึ้นข้อความ “ตรวจเกินจำเป็น”

---

### 4.4 ด่าน 7-3: มันฝรั่งงอก

**โจทย์:** แยกมันฝรั่งที่มีหน่องอกออกจากมันฝรั่งปกติ  
**ตรรกะ:**

```text
ถ้า [มันฝรั่งมีหน่องอก] → [ส่งไปคัดแยกพิเศษ]
```

#### วัตถุหลัก

- มันฝรั่งมีหน่องอก
- มันฝรั่งปกติ

#### ตัวหลอก

- มันฝรั่งเปื้อนดิน
- มันฝรั่งรูปร่างบิดเบี้ยว
- มันฝรั่งมีจุดสีเข้ม แต่ไม่มีหน่องอก

#### จุดประสงค์ของตัวหลอก

เพิ่มความยากของเกม If โดยใช้วัตถุที่ดูคล้ายกันมากขึ้น ผู้เรียนต้องสังเกตเฉพาะคุณลักษณะที่ตรงเงื่อนไข

#### Animation

- มันฝรั่งงอกมีหน่อเล็ก ๆ โผล่ขึ้นมา
- เครื่องสแกนมีกรอบขยายช่วยมองหน่อ
- ถ้าคัดถูก ประตูพิเศษเปิดและมีป้าย “คัดแยก”
- ถ้าพลาด มันฝรั่งงอกลงตะกร้าปกติและมีสัญลักษณ์เตือน

---

## 5. โครงสร้างเกมที่ 2: โรงคัดผลไม้แสนอร่อย

**Stage:** 8  
**ธีม:** ผลผลิตจากผลไม้  
**ตรรกะหลัก:** If / Else  
**แนวคิด:** แยกผลไม้เป็น 2 เส้นทาง ถ้าเข้าเงื่อนไขไปทางหนึ่ง ถ้าไม่เข้าเงื่อนไขไปอีกทางหนึ่ง

### 5.1 รูปแบบกฎของเกมที่ 2

```text
ถ้า [เข้าเงื่อนไข] → [ปลายทาง A]
นอกเหนือจากนี้ → [ปลายทาง B]
```

---

### 5.2 ด่าน 8-1: ส้มใหญ่ / ส้มเล็ก

**โจทย์:** แยกส้มลูกใหญ่ไปกล่องพรีเมียม และส้มที่เหลือไปกล่องแปรรูป  
**ตรรกะ:**

```text
ถ้า [ส้มลูกใหญ่] → [กล่องพรีเมียม]
นอกเหนือจากนี้ → [กล่องแปรรูปน้ำส้ม]
```

#### วัตถุหลัก

- ส้มลูกใหญ่
- ส้มลูกเล็ก

#### ตัวหลอก

- ส้มขนาดกลางที่เกือบใหญ่แต่ยังไม่ถึงเกณฑ์
- ส้มลูกเล็กแต่สีสวย
- ส้มลูกใหญ่แต่สีอ่อนกว่า

#### จุดประสงค์ของตัวหลอก

ให้เด็กเข้าใจว่าเงื่อนไขคือ “ขนาด” ไม่ใช่สีหรือความสวย

#### Animation

- ส้มกลิ้งบนสายพาน
- เครื่องวัดขนาดมีวงแหวนสแกน
- ส้มลูกใหญ่เด้งเข้ากล่องพรีเมียม
- ส้มอื่น ๆ ไหลไปกล่องแปรรูป
- ถ้าผิด กล่องพรีเมียมสั่นและขึ้นสัญลักษณ์ตกใจ

---

### 5.3 ด่าน 8-2: กล้วยสุก / กล้วยดิบ

**โจทย์:** คัดกล้วยสุกไปขายหน้าร้าน และกล้วยที่ยังไม่สุกไปห้องบ่มต่อ  
**ตรรกะ:**

```text
ถ้า [กล้วยสุกสีเหลือง] → [ส่งไปขายหน้าร้าน]
นอกเหนือจากนี้ → [ส่งไปห้องบ่มต่อ]
```

#### วัตถุหลัก

- กล้วยสุกสีเหลือง
- กล้วยดิบสีเขียว

#### ตัวหลอก

- กล้วยสีเหลืองอมเขียว
- กล้วยมีจุดดำเล็กน้อยแต่ยังขายได้
- กล้วยสีเหลืองแต่ปลายยังเขียว

#### จุดประสงค์ของตัวหลอก

ฝึกให้ผู้เรียนมองเกณฑ์ “สุกพร้อมขาย” ไม่ใช่แค่เห็นสีเหลืองเล็กน้อยแล้วตัดสินทันที

#### Animation

- เครื่องสแกนสีผลไม้
- กล้วยสุกเข้าตะกร้าหน้าร้านพร้อมป้ายราคา
- กล้วยดิบเข้าห้องบ่ม มีไอน้ำอุ่น
- ถ้ากล้วยดิบถูกส่งไปขาย ลูกค้าในภาพทำหน้างง

---

### 5.4 ด่าน 8-3: แตงโมเกรดดี / แปรรูป

**โจทย์:** คัดแตงโมเกรดดีไปติดสติ๊กเกอร์เกรด A ส่วนแตงโมที่ไม่ผ่านเกณฑ์ส่งไปแปรรูป  
**ตรรกะ:**

```text
ถ้า [แตงโมเกรดดี] → [ติดสติ๊กเกอร์เกรด A]
นอกเหนือจากนี้ → [ส่งไปแปรรูป]
```

#### เงื่อนไขแตงโมเกรดดี

```text
แตงโมเกรดดี = น้ำหนักพอดี และเสียงกังวาน
```

#### วัตถุหลัก

- แตงโมเกรดดี
- แตงโมไม่ผ่านเกณฑ์

#### ตัวหลอก

- แตงโมน้ำหนักพอดีแต่เสียงไม่กังวาน
- แตงโมเสียงกังวานแต่น้ำหนักน้อย
- แตงโมสีสวยแต่ไม่ผ่านเกณฑ์น้ำหนัก
- แตงโมลายสวยแต่เสียงทึบ

#### จุดประสงค์ของตัวหลอก

เพิ่มความยากของ If / Else ด้วยเงื่อนไขที่ประกอบด้วยข้อมูล 2 อย่าง แต่ยังคงเป็นการแยก 2 เส้นทาง ไม่ใช่ Else If

#### Animation

- แตงโมขึ้นแท่นชั่ง
- เครื่องเคาะแตงโมมีเสียง “ป๊อก!”
- ถ้าเกรดดี ติดสติ๊กเกอร์ดาว
- ถ้าไม่ผ่าน ส่งไปเครื่องแปรรูปน้ำแตงโม
- ถ้าผิด แตงโมเด้งผิดกล่องและมีเสียง “ตุ้บ!”

---

## 6. โครงสร้างเกมที่ 3: โรงคัดผลผลิตจากฟาร์มสัตว์

**Stage:** 9  
**ธีม:** ผลผลิตจากสัตว์  
**ตรรกะหลัก:** If / Else If / Else  
**แนวคิด:** คัดเกรดผลผลิตหลายระดับ โดยตรวจเงื่อนไขจากบนลงล่าง

### 6.1 รูปแบบกฎของเกมที่ 3

```text
ถ้า [เงื่อนไข A] → [ปลายทาง A]
นอกเหนือจากนี้ถ้า [เงื่อนไข B] → [ปลายทาง B]
นอกเหนือจากนี้ → [ปลายทาง C]
```

---

### 6.2 ด่าน 9-1: คัดเกรดไข่ไก่

**โจทย์:** คัดเกรดไข่ไก่ตามขนาดและสภาพเปลือก  
**ตรรกะ:**

```text
ถ้า [ไข่ใบใหญ่และเปลือกสมบูรณ์] → [ถาดไข่พรีเมียม]
นอกเหนือจากนี้ถ้า [ไข่ใบเล็กแต่ไม่แตก] → [ถาดไข่มาตรฐาน]
นอกเหนือจากนี้ → [ถาดคัดทิ้ง]
```

#### วัตถุหลัก

- ไข่ใบใหญ่ เปลือกสมบูรณ์
- ไข่ใบเล็ก เปลือกสมบูรณ์
- ไข่ร้าว

#### ตัวหลอก

- ไข่ใหญ่แต่มีรอยร้าวเล็กน้อย
- ไข่เล็กแต่เปลือกสวยมาก
- ไข่สีเข้มแต่ไม่ได้ร้าว
- ไข่มีเงาสะท้อนที่ดูคล้ายรอยร้าว

#### จุดประสงค์ของตัวหลอก

ให้ผู้เรียนเข้าใจว่าถาดพรีเมียมต้องดูทั้ง “ขนาด” และ “เปลือกสมบูรณ์” ไม่ใช่ดูเพียงคุณสมบัติเดียว

#### Animation

- ไข่กลิ้งบนสายพานแบบนุ่มนวล
- เครื่องสแกนรอยร้าวมีแสงผ่านเปลือกไข่
- ไข่เข้าถาดพรีเมียม/มาตรฐาน/คัดทิ้ง
- ถ้าผิด ไข่แตก “แปะ!” แบบตลกแต่ไม่รุนแรง

### 6.3 ด่าน 9-2: คัดคุณภาพขนแกะ

**โจทย์:** คัดคุณภาพขนแกะตามความสะอาดและสภาพขน  
**ตรรกะ:**

```text
ถ้า [ขนแกะสะอาดและนุ่ม] → [ม้วนไหมพรมพรีเมียม]
นอกเหนือจากนี้ถ้า [ขนแกะมีเศษหญ้า] → [ส่งไปเครื่องทำความสะอาด]
นอกเหนือจากนี้ → [ส่งเข้าถังรีไซเคิล]
```

#### วัตถุหลัก

- ขนแกะสะอาดและนุ่ม
- ขนแกะมีเศษหญ้า
- ขนแกะเปียกหรือสกปรกมาก

#### ตัวหลอก

- ขนแกะสะอาดแต่ไม่ฟู
- ขนแกะมีเศษหญ้าน้อยมาก
- ขนแกะสีเข้มตามธรรมชาติแต่ไม่ได้สกปรก
- ขนแกะฟูแต่มีคราบน้ำ

#### จุดประสงค์ของตัวหลอก

เพิ่มความละเอียดในการสังเกต และฝึกให้เด็กอ่านเงื่อนไขตามลำดับ ไม่ด่วนสรุปจากภาพรวม

#### Animation

- ก้อนขนแกะฟู ๆ ลอยหรือกลิ้งบนสายพาน
- เครื่องเป่าลมและเครื่องหวีขนทำงาน
- ขนดีถูกม้วนเป็นไหมพรม
- ขนมีเศษหญ้าเข้าเครื่องทำความสะอาด
- ขนเสียตกถังรีไซเคิลพร้อมควันเล็ก ๆ

### 6.4 ด่าน 9-3: ตรวจคุณภาพน้ำนม

**โจทย์:** ตรวจคุณภาพน้ำนมด้วยอุณหภูมิ และคัดไปยังปลายทางที่เหมาะสม  
**ตรรกะ:**

```text
ถ้า [อุณหภูมินมต่ำกว่า 8°C] → [บรรจุขวดพร้อมขาย]
นอกเหนือจากนี้ถ้า [อุณหภูมิ 8–15°C] → [ส่งเข้าห้องทำความเย็น]
นอกเหนือจากนี้ → [คัดทิ้ง]
```

#### วัตถุหลัก

- นมเย็นคุณภาพดี ต่ำกว่า 8°C
- นมอุณหภูมิ 8–15°C ต้องทำความเย็น
- นมอุณหภูมิสูงกว่า 15°C ไม่ผ่านเกณฑ์

#### ตัวหลอก

- นมอุณหภูมิ 7.9°C ใกล้ขอบเขต
- นมอุณหภูมิ 8°C พอดี
- นมอุณหภูมิ 15°C พอดี
- ขวดนมสีสวยแต่ตัวเลขอุณหภูมิไม่ผ่าน
- นมที่มีฉลากสวยแต่คุณภาพไม่ได้ขึ้นกับฉลาก

#### จุดประสงค์ของตัวหลอก

เป็นด่านยากสุดของบทที่ 3 เพราะต้องใช้การเปรียบเทียบตัวเลขและขอบเขตของเงื่อนไข

#### Animation

- ขวดนมเคลื่อนผ่านเครื่องวัดอุณหภูมิ
- ตัวเลขอุณหภูมิเด้งขึ้นบนจอ
- นมเย็นดีเข้ากล่องบรรจุขวด
- นมที่ต้องทำความเย็นเข้าเครื่องเย็น มีไอเย็น
- นมที่ไม่ผ่านถูกคัดออกพร้อมป้ายเตือน
- ถ้าเลือกผิด เครื่องแจ้งเตือนด้วยไฟสีแดง

## 7. ระบบตัวหลอก

### 7.1 หลักการออกแบบตัวหลอก

ตัวหลอกต้องช่วยให้ผู้เรียนฝึกสังเกต ไม่ใช่หลอกแบบไม่ยุติธรรม ตัวหลอกที่ดีควรมีลักษณะดังนี้

1. ดูคล้ายวัตถุที่เข้าเงื่อนไข แต่ไม่ตรงเงื่อนไขจริง
2. ทำให้ผู้เรียนต้องอ่านเงื่อนไขอย่างละเอียด
3. มีคำอธิบายเมื่อคลิกดูรายละเอียด
4. มี feedback เฉพาะเมื่อผู้เรียนเลือกผิด
5. ไม่ควรยากเกินระดับ ป.4

### 7.2 อัตราตัวหลอกที่เหมาะสม

| ระดับด่าน | อัตราตัวหลอกที่เสนอ |
|---|---|
| ด่านแรกของแต่ละเกม | 10–20% |
| ด่านที่สอง | 20–30% |
| ด่านที่สาม | 30–40% |

### 7.3 Feedback เมื่อติดกับตัวหลอก

```text
แครอทชิ้นนี้สีเข้ม แต่ไม่ได้มีคราบโคลนจริง จึงไม่ต้องส่งเข้าเครื่องล้าง
```

```text
แตงโมลูกนี้น้ำหนักพอดี แต่เสียงไม่กังวาน จึงยังไม่ใช่แตงโมเกรดดี
```

```text
ไข่ใบนี้ใหญ่ แต่มีรอยร้าวเล็กน้อย จึงไม่ควรเข้าถาดพรีเมียม
```

## 8. การปรับแถบรายการวัตถุด้านล่าง

### 8.1 ปัญหาปัจจุบัน

1. วัตถุบางอย่างที่อยู่บริเวณมุมด้านล่างของกรอบเกมถูกแถบรายการวัตถุบดบังเล็กน้อย
2. แถบรายการวัตถุยังเป็นเพียงตัวแสดงลำดับ ไม่สามารถคลิกเพื่อตรวจสอบข้อมูลได้

### 8.2 แนวทางแก้ปัญหาการบดบัง

ให้ item tray เป็นพื้นที่แยกจาก conveyor stage ไม่ใช่ overlay ที่ทับบนพื้นที่เล่นจริง

โครงสร้างที่เสนอ

```text
┌──────────────────────────────┐
│ Conveyor Stage               │
│ สายพาน / จุดสแกน / ประตู     │
│ item เคลื่อนใน safe area      │
├──────────────────────────────┤
│ Item Tray: รายการวัตถุรอบนี้ │
└──────────────────────────────┘
```

ข้อกำหนดสำคัญ

- item tray ไม่ควรใช้ position absolute ทับพื้นที่สายพาน
- ถ้าจำเป็นต้อง overlay ต้องกำหนด safe area ด้านล่าง
- item ที่เคลื่อนบนสายพานต้องไม่เคลื่อนลงไปในพื้นที่ของ item tray
- ต้องทดสอบ item ขนาดใหญ่ เช่น แตงโม ขวดนม ก้อนขนแกะ

### 8.3 CSS ที่เสนอ

```css
}

  z-index: 1;
}
```

- มีคุณสมบัติอะไร
- เข้าเงื่อนไขหรือไม่
- เป็นตัวหลอกหรือไม่
- ควรสังเกตอะไรเป็นพิเศษ

### 9.2 พฤติกรรมที่ต้องการ

7. คลิก item อื่นเพื่อเปลี่ยนข้อมูลได้
8. บนมือถือแสดงเป็น bottom sheet หรือ panel ที่เลื่อนปิดได้

## 11. ระบบตรวจเงื่อนไขกับตัวหลอก

ตัวหลอกไม่ควรใช้ logic พิเศษ แต่ควรใช้ props ปกติเหมือน item อื่น เพื่อให้ engine ตรวจตามกฎเดียวกัน

ตัวอย่าง condition checker

```javascript
function checkCondition(item, conditionId) {
  switch (conditionId) {
    case 'lettuce_has_worm':
      return item.props.type === 'lettuce' && item.props.hasWorm === true;

    case 'watermelon_grade_a':
      return item.props.type === 'watermelon'
        && item.props.weightOk === true
        && item.props.sound === 'clear';

    case 'premium_egg':
      return item.props.type === 'egg'
        && item.props.size === 'large'
        && item.props.shell === 'perfect';

    case 'milk_below_8':
      return item.props.type === 'milk'
        && item.props.temperature < 8;

    case 'milk_between_8_15':
      return item.props.type === 'milk'
        && item.props.temperature >= 8
        && item.props.temperature <= 15;

    default:
      return false;
  }
}
```

เมื่อผู้เรียนพลาดตัวหลอก ให้ดึง `decoyReason` มาใช้ใน feedback

```javascript
function getDecoyFeedback(item) {
  if (item.isDecoy) {
    return item.decoyReason || 'วัตถุชิ้นนี้เป็นตัวหลอก ต้องสังเกตเงื่อนไขให้ละเอียดขึ้น';
  }
  return null;
}
```

## 14. ระบบดาว

ยังคงใช้ดาวเป็นคะแนนหลักเท่านั้น ไม่เพิ่มคะแนนตัวเลข คอมโบ หรือเหรียญ

### 14.1 เกณฑ์ดาวที่เสนอ

```text
3 ดาว = ถูก 90% ขึ้นไป และเสียหายไม่เกิน 1 ชิ้น
2 ดาว = ถูก 75% ขึ้นไป
1 ดาว = ผ่านเกณฑ์ขั้นต่ำ 60%
0 ดาว = ต่ำกว่า 60% หรือกฎไม่ครบ
```

### 14.2 สรุปผลหลังจบด่าน

ควรแสดงเฉพาะข้อมูลจำเป็น

```text
ถูกต้อง: 18/20
ผิดพลาด: 2
สินค้าเสียหาย: 1
ดาวที่ได้รับ: 3 ดาว
```

ไม่ต้องแสดงคะแนน คอมโบ เหรียญ หรือร้านค้าในรอบนี้

## 19. สรุป

Smart Farm Manager v3 ปรับบทที่ 3 ให้ธีมแน่นและสอดคล้องขึ้น โดยแบ่งเป็น 3 เกมหลัก ได้แก่
