# แผนพัฒนาอย่างละเอียด: ระบบสร้างชิ้นงานบทที่ 2 และหน้าแสดงผลงาน

> เอกสารนี้จัดทำขึ้นสำหรับพัฒนาต่อจากโค้ดล่าสุดของโปรเจกต์  
> โฟกัสเฉพาะ **บทที่ 2: เส้นทางเดินรถไถ**  
> ครอบคลุม 2 ส่วนหลัก  
> 1. การปรับปรุงหน้า `pages/create_project_algo.php` หรือ “สตูดิโอออกแบบเส้นทางรถไถ”  
> 2. การปรับปรุงหน้า `pages/showcase.php` หรือ “ลานโชว์ผลงาน” ให้สื่อสารชัดเจนขึ้นว่าเป็นผลงานของบทเรียนใด

---

## 1. สรุปสถานะปัจจุบัน

ระบบสร้างชิ้นงานบทที่ 2 มีฐานที่ดีแล้ว โดยเฉพาะหน้า `pages/create_project_algo.php` ซึ่งมีองค์ประกอบสำคัญครบในระดับใช้งานได้ ได้แก่

- เลือกประเภทภารกิจได้ 3 แบบ: ไปถึงจุดหมาย, หลบสิ่งกีดขวาง, เก็บเกี่ยวผลผลิต
- มี toolbox สำหรับวางวัตถุบนแผนที่
- มีแผนที่ grid 6 x 5
- มีปุ่มคำสั่งลูกศรสำหรับสร้างคำสั่งเฉลย
- มีปุ่มทดสอบเส้นทาง
- มี validation ก่อนส่ง
- มีการบันทึก `work_data` แบบ `tractor_route`
- มีการบันทึกคำอธิบายวิธีคิดของนักเรียน
- มีการส่งไปหน้า `showcase.php` หลังบันทึกสำเร็จ

อย่างไรก็ตาม ยังมีจุดที่ควรปรับปรุงเพื่อให้ระบบใช้งานง่ายขึ้น เหมาะกับนักเรียน ป.4 มากขึ้น และทำให้ครูหรือผู้ชมเข้าใจผลงานได้ชัดเจนขึ้น

## 2. ปัญหาและข้อเสนอที่ต้องแก้

### 2.1 รถไถยังหันหน้าไม่ถูกทาง

รถไถใช้ emoji `🚜` เป็น prototype asset และแสดงเป็น icon คงที่ จึงไม่สามารถสื่อได้ชัดว่ารถไถกำลังมุ่งหน้าไปทิศใดระหว่างทดสอบเส้นทาง

### 2.2 เครื่องมือยางลบยังกลมกลืนกับวัตถุอื่น

ยางลบอยู่รวมกับวัตถุอื่นใน toolbox และใช้รูปแบบปุ่มเหมือนกัน ทำให้นักเรียนอาจเข้าใจว่าเป็นวัตถุชนิดหนึ่งแทนที่จะเป็น “โหมดลบวัตถุ”

### 2.3 หน้าโชว์ผลงานยังไม่สื่อชัดว่าผลงานเป็นของบทเรียนใด

หน้า `showcase.php` รับ `game_id` แล้ว แต่ header และ card ยังแสดงภาพรวมว่า “ลานโชว์ผลงาน” โดยยังไม่เด่นพอว่าเป็นผลงานของบทที่ 2 หรือบทเรียนใด

### 2.4 ระบบสร้างชิ้นงานควรช่วยนำทางนักเรียนมากขึ้น

แม้มี validation ก่อนส่งแล้ว แต่เด็กจะรู้ว่าขาดอะไรเมื่อกดส่งหรือกดทดสอบเท่านั้น ควรมี checklist ความสมบูรณ์ของชิ้นงานให้เห็นตลอดเวลา

### 2.5 หน้าแสดงผลงานควรสรุปข้อมูลชิ้นงานให้เข้าใจเร็ว

Card ควรมีข้อมูลสรุป เช่น ประเภทภารกิจ จำนวนคำสั่ง จำนวนอุปสรรค จำนวนผลผลิต และสถานะทดสอบผ่าน

## 3. เป้าหมายการพัฒนา

### 3.1 เป้าหมายด้านหน้า Create Project

4. แสดงจำนวนคำสั่ง อุปสรรค ผลผลิต และสถานะทดสอบผ่าน
5. Modal นำเสนอผลงานต้องแสดงข้อมูลบทเรียนและประเภทภารกิจ
6. เตรียมรองรับผลงานบทที่ 3 และ 4 ในอนาคต

## 5. ปรับรถไถให้สื่อทิศทางระหว่างทดสอบเส้นทาง

### 5.1 ปัญหา

รถไถแสดงด้วย emoji `🚜` และไม่มีทิศทางประกอบ ทำให้ระหว่างทดสอบเส้นทาง นักเรียนอาจไม่เห็นว่ารถไถกำลังมุ่งหน้าไปทางใด

เนื่องจากตอนนี้ยังใช้ emoji เป็น prototype asset การหมุน emoji โดยตรงอาจให้ผลไม่สวยหรือไม่เหมือนกันในแต่ละเครื่อง จึงไม่ควรแก้ด้วยการหมุน emoji เป็นหลัก

### 5.2 แนวทางที่แนะนำ

ใช้วิธีเพิ่ม “ลูกศรบอกทิศทาง” เป็น badge เล็ก ๆ กำกับรถไถ

แนวคิดคือ

- รถไถยังเป็น emoji เดิม
- เมื่อทดสอบเส้นทาง ให้แสดงลูกศรเล็กบนรถไถหรือมุมขวาบนของช่อง
- ลูกศรเปลี่ยนตามคำสั่งล่าสุดที่รถไถกำลังเดิน

### 5.7 เพิ่ม helper สำหรับรถไถ

```javascript
function addTractorObject(scene, x, y, size, facingDir = null) {
    const group = scene.add.container(x, y);

    const tractor = scene.add.text(0, 0, ASSET_MAP.start.emoji, {
        fontSize: `${Math.max(30, Math.floor(size * .72))}px`,
        fontFamily: '"Apple Color Emoji", "Segoe UI Emoji", "Noto Color Emoji", sans-serif'
    }).setOrigin(.5);

    group.add(tractor);

    if (facingDir && DIRS[facingDir]) {
        const badgeBg = scene.add.circle(size * .28, -size * .28, 13, 0x2563eb, 1)
            .setStrokeStyle(2, 0xffffff, 1);

        const arrow = scene.add.text(size * .28, -size * .28, DIRS[facingDir].icon, {
            fontSize: '16px',
            fontFamily: '"Apple Color Emoji", "Segoe UI Emoji", "Noto Color Emoji", sans-serif'
        }).setOrigin(.5);

        group.add([badgeBg, arrow]);
    }

    return group;
}
```

### 5.8 ปรับ `addGameObject()`

```javascript
function addGameObject(scene, key, x, y, size, facingDir = null) {
    if (key === 'start') {
        return addTractorObject(scene, x, y, size, facingDir);
    }

    const asset = ASSET_MAP[key] || ASSET_MAP.target;

    if (USE_EMOJI_ASSETS) {
        return scene.add.text(x, y, asset.emoji, {
            fontSize: `${Math.max(30, Math.floor(size * .72))}px`,
            fontFamily: '"Apple Color Emoji", "Segoe UI Emoji", "Noto Color Emoji", sans-serif'
        }).setOrigin(.5);
    }

    return scene.add.image(x, y, key).setDisplaySize(size, size).setOrigin(.5);
}
```

### 5.9 ข้อควรระวัง

- อย่าหมุน emoji โดยตรงเป็นหลัก
- ถ้าอนาคตเปลี่ยนเป็นภาพ HD ค่อยใช้ sprite rotation หรือ sprite sheet
- เวลาทดสอบจบแล้ว ถ้าต้อง redrawBoard แบบปกติ สามารถส่ง `facingDir` ล่าสุดหรือ `null` ก็ได้
- ถ้าไม่มีคำสั่งแรก รถไถไม่จำเป็นต้องแสดงลูกศร

### 7.4 เพิ่ม function `updateChecklist()`

```javascript
function updateChecklist() {
    const list = document.getElementById('project-checklist');
    if (!list) return;

    const thinking = document.getElementById('thinking-input').value.trim();

    const hasDestination = missionType === 'harvest' ? !!objects.barn : !!objects.target;
    const hasMissionRequirement =
        missionType === 'target'
        || (missionType === 'obstacle' && objects.obstacles.length > 0)
        || (missionType === 'harvest' && objects.crops.length > 0);

    const checks = [
        { label: 'วางรถไถจุดเริ่มต้นแล้ว', ok: !!objects.start },
        { label: missionType === 'harvest' ? 'วางโรงนาแล้ว' : 'วางเป้าหมายแล้ว', ok: hasDestination },
        { label: 'ทำตามเงื่อนไขของประเภทภารกิจแล้ว', ok: hasMissionRequirement },
        { label: 'มีคำสั่งเฉลยแล้ว', ok: commands.length > 0 },
        { label: 'ทดสอบเส้นทางผ่านแล้ว', ok: testPassed },
        { label: 'เขียนคำอธิบายวิธีคิดแล้ว', ok: thinking.length > 0 }
    ];

    list.innerHTML = checks.map(item => `
        <li class="check-item ${item.ok ? 'done' : 'missing'}">
            <span>${item.ok ? '✅' : '⬜'}</span>
            <span>${item.label}</span>
        </li>
    `).join('');
}
```

## 13. ปรับ preview ของ tractor route ให้แสดงเส้นทาง

### 13.1 เหตุผล

`renderTractorRoutePreview()` แสดงแผนที่และคำสั่งเฉลยแล้ว แต่ถ้าเพิ่มลูกศรหรือเลขลำดับบน grid จะทำให้เข้าใจกระบวนการคิดมากขึ้น

### 13.2 เพิ่ม helper คำนวณเส้นทาง

```javascript
function buildRoutePath(data) {
    const dirs = {
        UP: { dc: 0, dr: -1, icon: '⬆️' },
        DOWN: { dc: 0, dr: 1, icon: '⬇️' },
        LEFT: { dc: -1, dr: 0, icon: '⬅️' },
        RIGHT: { dc: 1, dr: 0, icon: '➡️' }
    };

    const start = data.start;
    if (!start || !Array.isArray(data.commands)) return [];

    let pos = {
        col: start.col ?? start.c,
        row: start.row ?? start.r
    };

    const path = [{ ...pos, step: 0 }];

    data.commands.forEach((cmd, index) => {
        const dir = dirs[cmd];
        if (!dir) return;

        pos = {
            col: pos.col + dir.dc,
            row: pos.row + dir.dr
        };

        path.push({
            ...pos,
            step: index + 1,
            dir: cmd,
            icon: dir.icon
        });
    });

    return path;
}
```

### 13.3 ใช้ใน `renderTractorRoutePreview()`

ก่อน loop grid

```javascript
const path = buildRoutePath(data);
```

ในแต่ละ cell

```javascript
const pathStep = path.find(item => item.col === col && item.row === row && item.step > 0);
```

ถ้ามี path step และ cell ไม่มี object หลัก ให้แสดงเลขหรือลูกศรเล็ก

```javascript
if (!cell.textContent && pathStep) {
    cell.innerHTML = `
        <div style="display:flex; flex-direction:column; align-items:center; line-height:1;">
            <span style="font-size:16px;">${pathStep.icon || ''}</span>
            <span style="font-size:12px; font-weight:800; color:#1d4ed8;">${pathStep.step}</span>
        </div>
    `;
}
```

### 13.4 ข้อควรระวัง

- ถ้า path ออกนอก grid ไม่ควร render step นั้น
- ถ้า path ไปทับ object หลัก ให้ object หลักสำคัญกว่า
- ไม่ควรทำให้ preview รกเกินไป

```text
1. รถไถต้องสื่อทิศทาง
2. ยางลบต้องแยกจากวัตถุอื่น
3. นักเรียนต้องเห็น checklist ก่อนส่ง
4. หน้า showcase ต้องบอกว่าผลงานเป็นของบทเรียนใด
5. card และ modal ต้องสรุปข้อมูลชิ้นงานบทที่ 2 ได้ทันที
```

การปรับเหล่านี้จะทำให้ระบบสร้างชิ้นงานไม่ใช่แค่ “วางวัตถุแล้วส่ง” แต่กลายเป็นกระบวนการออกแบบโจทย์ลำดับคำสั่งที่เข้าใจง่าย ตรวจสอบได้ และนำเสนอผลงานได้ชัดเจนมากขึ้น
