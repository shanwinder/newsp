# แผนการพัฒนาเกมบทที่ 4  
# Smart Farm Debugger: ภารกิจจับบั๊กกู้ฟาร์ม

**บทเรียน:** บทที่ 4 การตรวจหาและแก้ไขข้อผิดพลาด (Debugging)  
**ชื่อบทเรียนที่เสนอ:** ช่างซ่อมระบบฟาร์มอัจฉริยะ  
**ชื่อเกมรวม:** Smart Farm Debugger: ภารกิจจับบั๊กกู้ฟาร์ม  
**แนวเกม:** Bug Hunt + System Repair + Visual Block Debugging + Farm Simulation  
**ธีม:** ฟาร์มอัจฉริยะ / ระบบเกษตรอัตโนมัติ / เครื่องจักรฟาร์มรวน / แมลงบั๊ก  
**กลุ่มเป้าหมาย:** นักเรียนชั้นประถมศึกษาปีที่ 4  
**เป้าหมายหลัก:** ให้นักเรียนเรียนรู้การสังเกตข้อผิดพลาด วิเคราะห์สาเหตุ ค้นหาบั๊ก แก้ไขบล็อกคำสั่ง และทดสอบระบบใหม่จนทำงานถูกต้อง

---

## 1. แนวคิดหลักของบทที่ 4

บทที่ 4 ควรต่อยอดจากบทที่ 3 โดยไม่ซ้ำรูปแบบการเล่นเดิม บทที่ 3 เด็กได้เรียนรู้การสร้างกฎ If / Else / Else If เพื่อควบคุมระบบฟาร์มอัจฉริยะแล้ว ส่วนบทที่ 4 ควรเปลี่ยนบทบาทของผู้เรียนจาก “ผู้สร้างระบบ” เป็น “ช่างซ่อมระบบ” ที่ต้องแก้ระบบที่ทำงานผิดพลาด

แกนหลักของเกมคือ

> ระบบฟาร์มทำงานผิดพลาด → เด็กสังเกตอาการ → ระบุบล็อกที่ผิด → แก้บล็อก → ทดสอบใหม่ → ระบบกลับมาทำงานถูกต้อง

เด็กจะไม่ได้เริ่มจากหน้าว่าง แต่จะได้รับ “ระบบที่มีบั๊ก” อยู่แล้ว เช่น สายพานส่งผลผลิตผิดทาง เครื่องให้อาหารสัตว์จ่ายอาหารผิดชนิด หรือโรงเรือนเปิดอุปกรณ์ผิดสภาพอากาศ เด็กต้องสังเกตผลลัพธ์ก่อน แล้วจึงหาว่าเกิดจากเงื่อนไขผิด คำสั่งผิด ลำดับผิด หรือขาดกรณี Else

หัวใจสำคัญคือ

> นักเรียนต้องเข้าใจว่า Debugging คือการหาสาเหตุของความผิดพลาด ไม่ใช่แค่เดาคำตอบใหม่

---

## 2. ความเชื่อมโยงกับบทที่ 3

บทที่ 3 คือ **Smart Farm Manager: สายพานตรรกะฟาร์มอัจฉริยะ** ซึ่งฝึกการสร้างกฎควบคุมระบบฟาร์ม

บทที่ 4 ควรใช้โลกเดียวกันแต่เปลี่ยนภารกิจเป็นการแก้ปัญหา เช่น

- บทที่ 3: เด็กสร้างกฎให้สายพานคัดแยกผลผลิต
- บทที่ 4: สายพานคัดแยกผิด เด็กต้องหาว่ากฎผิดตรงไหน

- บทที่ 3: เด็กกำหนด If / Else / Else If
- บทที่ 4: เด็กได้รับ If / Else / Else If ที่ผิด แล้วต้องแก้

- บทที่ 3: เด็กเห็นผลจากกฎของตนเอง
- บทที่ 4: เด็กเห็นอาการผิด แล้วย้อนกลับไปวิเคราะห์กฎ

ดังนั้นบทที่ 4 จะทำให้การเรียนรู้สมบูรณ์ขึ้น เพราะเด็กไม่ได้แค่เขียนโปรแกรม แต่ได้ฝึกตรวจสอบและปรับปรุงโปรแกรมด้วย

---

## 3. เป้าหมายการเรียนรู้

เมื่อเรียนจบบทที่ 4 นักเรียนควรสามารถ

1. อธิบายได้ว่า Debugging คือการตรวจหาและแก้ไขข้อผิดพลาด
2. สังเกตอาการผิดพลาดของระบบจากผลลัพธ์ที่เกิดขึ้น
3. แยกประเภทข้อผิดพลาดพื้นฐานได้ เช่น เงื่อนไขผิด คำสั่งผิด ลำดับผิด หรือขาด Else
4. ระบุบล็อกที่น่าจะเป็นสาเหตุของปัญหาได้
5. แก้ไขบล็อกคำสั่งให้ถูกต้องได้
6. ทดสอบระบบซ้ำหลังแก้ไข
7. อธิบายเหตุผลได้ว่าทำไมจุดนั้นจึงเป็นบั๊ก
8. เข้าใจว่าการแก้บั๊กต้องดูผลลัพธ์และเหตุผล ไม่ใช่เดาสุ่ม
9. เชื่อมโยงการ Debug กับสถานการณ์ในชีวิตจริง เช่น เครื่องจักรทำงานผิด ต้องหาสาเหตุอย่างเป็นขั้นตอน

---

## 4. โครงสร้างบทที่ 4

บทที่ 4 แบ่งเป็น 3 เกมหลัก หรือ 3 Stage ตามระดับความยาก

| Stage | ชื่อเกม | จุดเน้น |
|---|---|---|
| Stage 10 | บั๊กจิ๋วในโรงคัดแยก | เงื่อนไขผิด / คำสั่งผิด |
| Stage 11 | โรงเรือนรวน ลำดับเพี้ยน | ลำดับเงื่อนไขผิด / ขาด Else / เครื่องหมายผิด |
| Stage 12 | ศูนย์ควบคุมฟาร์มฉุกเฉิน | Debug หลายระบบ / วิเคราะห์หลายอาการ |

โครงสร้างนี้ทำให้ความยากค่อย ๆ เพิ่มขึ้นจากการแก้บล็อกผิดจุดเดียว ไปสู่การแก้ระบบที่มีหลายเงื่อนไขและหลายจุดเสีย

# 5. Core Game Loop

## 5.1 ขั้นที่ 1: ดูระบบทำงานผิดพลาด

ทุกด่านควรเริ่มด้วยการให้ระบบทำงานผิดก่อน 1 รอบ เช่น สายพานเริ่มเดิน วัตถุผ่านเครื่องสแกน แล้วเกิดผลลัพธ์ผิด

ตัวอย่าง:

```text
ไข่ร้าวถูกส่งเข้าถาดพรีเมียม
เครื่องสแกนร้องเตือน
ถาดพรีเมียมสั่นและขึ้นเครื่องหมายผิด
```

ช่วงนี้เรียกว่า **Run Before Fix** หรือ “ดูก่อนว่าเสียอย่างไร”

เหตุผลคือเด็กจะได้ฝึกสังเกตอาการ ไม่ใช่เห็นบล็อกแล้วเดา

---

## 5.2 ขั้นที่ 2: วิเคราะห์อาการ

หลังระบบทำงานผิด เกมถามเด็ก เช่น

```text
เกิดอะไรขึ้นกับระบบ?
```

เด็กเลือกคำตอบเชิงสังเกต เช่น

- ส่งผิดปลายทาง
- เครื่องจักรทำงานผิดเวลา
- ระบบไม่ทำอะไรเลย
- วัตถุที่เป็นตัวหลอกหลุดไปผิดที่
- อุณหภูมิผิดแต่ระบบไม่แก้

ขั้นนี้ช่วยให้เด็กเชื่อม “ผลลัพธ์” กับ “สาเหตุ” ก่อนแก้บล็อก

---

## 5.3 ขั้นที่ 3: ใช้แว่นขยายหาบั๊ก

เด็กใช้เครื่องมือ เช่น “แว่นขยายจับบั๊ก” คลิกหรือแตะบล็อกที่คิดว่าผิด

ระบบแสดงบล็อกคำสั่ง เช่น

```text
ถ้า [ไข่ใบใหญ่] → [ถาดพรีเมียม]
นอกเหนือจากนี้ถ้า [ไข่ใบเล็ก] → [ถาดมาตรฐาน]
นอกเหนือจากนี้ → [ถังคัดทิ้ง]
```

เด็กต้องทำเครื่องหมายว่าบล็อกไหนเป็นบั๊ก

ถ้าเลือกถูก บั๊กการ์ตูนจะกระโดดออกจากบล็อก  
ถ้าเลือกผิด เกมให้ Feedback ว่า “บล็อกนี้อาจไม่ใช่สาเหตุ ลองดูผลลัพธ์อีกครั้ง”

---

## 5.4 ขั้นที่ 4: แก้บล็อก

เมื่อเจอบั๊กแล้ว เกมเปิดพื้นที่แก้ไข เช่น

```text
บล็อกเดิม:
ถ้า [ไข่ใบใหญ่] → [ถาดพรีเมียม]

เลือกบล็อกใหม่:
[ไข่ใบใหญ่และไม่ร้าว]
[ไข่ใบเล็กและไม่ร้าว]
[ไข่ร้าว]
```

เด็กต้องลากบล็อกที่ถูกต้องไปแทน หรือเลือกคำสั่งใหม่ให้ถูกต้อง

รูปแบบการแก้ควรรองรับ:

- เปลี่ยนเงื่อนไข
- เปลี่ยนคำสั่ง
- สลับลำดับเงื่อนไข
- เพิ่ม Else
- แก้เครื่องหมายมากกว่า/น้อยกว่า

---

## 5.5 ขั้นที่ 5: ทดสอบระบบอีกครั้ง

หลังแก้แล้ว เด็กกด

```text
[ทดสอบระบบอีกครั้ง]
```

ระบบรัน Animation ซ้ำ ถ้าถูกต้อง ฟาร์มกลับมาทำงานปกติ ถ้าผิด ระบบแสดงอาการผิดที่ยังเหลืออยู่ และให้เด็กแก้ต่อ

---

## 5.6 ขั้นที่ 6: สรุปผล

เมื่อผ่านด่าน ระบบสรุปว่า

```text
บั๊กที่พบ: เงื่อนไขไม่ละเอียดพอ
จุดที่แก้: เปลี่ยน “ไข่ใบใหญ่” เป็น “ไข่ใบใหญ่และไม่ร้าว”
ผลลัพธ์: ไข่ร้าวไม่หลุดเข้าถาดพรีเมียมแล้ว
ดาวที่ได้รับ: 3 ดาว
```

การสรุปควรเน้นเหตุผล ไม่ใช่แค่ผ่าน/ไม่ผ่าน

---

# 6. ประเภทของบั๊กในบทที่ 4

บทที่ 4 ควรสอนบั๊กพื้นฐาน 5 ประเภท

## 6.1 บั๊กคำสั่งผิด

เงื่อนไขถูก แต่คำสั่งผิด

ตัวอย่าง:

```text
ถ้า [แครอทเปื้อนโคลน] → [ปล่อยลงตะกร้า]
```

ควรแก้เป็น

```text
ถ้า [แครอทเปื้อนโคลน] → [ส่งเข้าเครื่องล้าง]
```

---

## 6.2 บั๊กเงื่อนไขผิด

คำสั่งถูก แต่เงื่อนไขผิด

ตัวอย่าง:

```text
ถ้า [แครอทสะอาด] → [ส่งเข้าเครื่องล้าง]
```

ควรแก้เป็น

```text
ถ้า [แครอทเปื้อนโคลน] → [ส่งเข้าเครื่องล้าง]
```

---

## 6.3 บั๊กเงื่อนไขไม่ละเอียดพอ

เงื่อนไขกว้างเกินไป ทำให้ตัวหลอกหลุดไปผิดทาง

ตัวอย่าง:

```text
ถ้า [ไข่ใบใหญ่] → [ถาดพรีเมียม]
```

ทำให้ไข่ใบใหญ่แต่ร้าวเข้าถาดพรีเมียมผิด

ควรแก้เป็น

```text
ถ้า [ไข่ใบใหญ่และไม่ร้าว] → [ถาดพรีเมียม]
```

---

## 6.4 บั๊กลำดับเงื่อนไขผิด

เงื่อนไขถูกทุกอัน แต่เรียงผิดลำดับ

ตัวอย่าง:

```text
ถ้า [ดินแห้ง] → [เปิดสปริงเกอร์]
นอกเหนือจากนี้ถ้า [ฝนตก] → [ปิดสปริงเกอร์]
```

ถ้าดินแห้งและฝนตกพร้อมกัน ระบบจะรดน้ำซ้ำจนผิด

ควรแก้เป็น

```text
ถ้า [ฝนตก] → [ปิดสปริงเกอร์]
นอกเหนือจากนี้ถ้า [ดินแห้ง] → [เปิดสปริงเกอร์]
```

---

## 6.5 บั๊กขาด Else

ระบบไม่รู้จะทำอะไรเมื่อไม่เข้าเงื่อนไขใดเลย

ตัวอย่าง:

```text
ถ้า [อุณหภูมิ > 35°C] → [เปิดพัดลม]
นอกเหนือจากนี้ถ้า [อุณหภูมิ < 15°C] → [เปิดหลอดไฟ]
```

แต่ถ้าอุณหภูมิ 25°C ระบบไม่มีคำสั่งประหยัดไฟ

ควรเพิ่ม

```text
นอกเหนือจากนี้ → [ปิดระบบทั้งหมด]
```

---

## 6.6 บั๊กเครื่องหมายตัวเลขผิด

ใช้เครื่องหมายมากกว่า/น้อยกว่าผิด หรือค่าขีดจำกัดผิด

ตัวอย่าง:

```text
ถ้า [อุณหภูมิ > 50°C] → [เปิดพัดลม]
```

แต่โจทย์จริงคือสูงกว่า 35°C ต้องเปิดพัดลม

ควรแก้เป็น

```text
ถ้า [อุณหภูมิ > 35°C] → [เปิดพัดลม]
```

# 7. รายละเอียด Stage 10: บั๊กจิ๋วในโรงคัดแยก

**จุดเน้น:** บั๊กพื้นฐาน เช่น เงื่อนไขผิด คำสั่งผิด เงื่อนไขไม่ละเอียดพอ  
**ความยาก:** ง่าย  
**ธีม:** สายพานคัดแยกผลผลิตจากบทที่ 3  
**เป้าหมาย:** ให้เด็กเข้าใจว่าบล็อกบางตัวในโปรแกรมอาจเป็นสาเหตุของผลลัพธ์ผิด

---

## 7.1 ด่าน 10-1: แครอทเลอะหลุดตะกร้า

### อาการผิดพลาด

แครอทเปื้อนโคลนหลุดลงตะกร้าปกติ ทำให้ตะกร้าเลอะ

### บล็อกที่มีบั๊ก

```text
ถ้า [แครอทเปื้อนโคลน] → [ปล่อยลงตะกร้า]
```

### ประเภทบั๊ก

คำสั่งผิด

### วิธีแก้

```text
ถ้า [แครอทเปื้อนโคลน] → [ส่งเข้าเครื่องล้าง]
```

### Animation

- แครอทเปื้อนโคลนวิ่งผ่านเครื่องสแกน
- ประตูไม่เปิดเข้าเครื่องล้าง
- แครอทตกลงตะกร้า
- ตะกร้ามีคราบโคลนกระเด็น
- เมื่อตรวจเจอ บั๊กตัวเล็กกระโดดออกจากบล็อกคำสั่ง

### Feedback

```text
เงื่อนไขถูกแล้ว แต่คำสั่งผิด แครอทเปื้อนโคลนควรถูกส่งไปล้าง ไม่ใช่ลงตะกร้าปกติ
```

---

## 7.2 ด่าน 10-2: ส้มเล็กเข้ากล่องพรีเมียม

### อาการผิดพลาด

ส้มลูกเล็กถูกส่งเข้ากล่องพรีเมียม ทั้งที่ควรส่งไปกล่องแปรรูป

### บล็อกที่มีบั๊ก

```text
ถ้า [ส้มลูกเล็ก] → [กล่องพรีเมียม]
นอกเหนือจากนี้ → [กล่องแปรรูป]
```

### ประเภทบั๊ก

เงื่อนไขผิด

### วิธีแก้

```text
ถ้า [ส้มลูกใหญ่] → [กล่องพรีเมียม]
นอกเหนือจากนี้ → [กล่องแปรรูป]
```

### Animation

- ส้มลูกเล็กเข้ากล่องพรีเมียม
- กล่องสั่นและมีสัญลักษณ์ผิด
- ลูกค้าการ์ตูนทำหน้างง
- เมื่อแก้ถูก ส้มลูกใหญ่เข้ากล่องพรีเมียม และส้มลูกเล็กไปแปรรูป

### Feedback


# 9. รายละเอียด Stage 12: ศูนย์ควบคุมฟาร์มฉุกเฉิน

**จุดเน้น:** Debug หลายระบบ วิเคราะห์หลายอาการ และเลือกแก้ให้ตรงจุด  
**ความยาก:** สูงสุดของบท  
**ธีม:** ศูนย์ควบคุมฟาร์มอัจฉริยะ  
**เป้าหมาย:** ให้เด็กนำความรู้ทั้งหมดมาวิเคราะห์และแก้ปัญหาระบบฟาร์มหลายส่วน

---

## 9.1 รูปแบบรวมของ Stage 12

หน้าจอเป็นแผนที่ฟาร์ม มีระบบหลายจุด เช่น

- โรงคัดผลผลิต
- โรงคัดผลไม้
- โรงคัดไข่
- โรงเรือนผัก
- เครื่องให้อาหารสัตว์
- ถังน้ำ
- แผงควบคุมกลาง

แต่ละจุดอาจมีอาการเสีย เด็กต้องเลือกจุดที่ผิด แล้วเข้าไปแก้บล็อก

เด็กมีจำนวนครั้งในการแก้จำกัด เช่น 3 ครั้งต่อด่าน เพื่อให้คิดก่อนแก้ ไม่เดาสุ่ม

---

## 9.2 ด่าน 12-1: สัญญาณเตือนจากโรงคัดแยก

### อาการ

มีระบบเสีย 2 จุด แต่มีบั๊กจริงเพียง 1 จุดที่เป็นสาเหตุหลัก

ตัวอย่าง:

- กล่องพรีเมียมมีของเสีย
- ถังคัดทิ้งว่างผิดปกติ

### บั๊กจริง

เงื่อนไขไม่ละเอียดพอ เช่น ไข่ร้าวเข้าถาดดี

### เป้าหมาย

เด็กต้องเลือกจุดที่เป็นสาเหตุหลัก ไม่ใช่แก้ปลายเหตุ

### Feedback

```text
อาการที่กล่องพรีเมียมมีของเสีย เกิดจากเงื่อนไขคัดไข่ไม่ละเอียดพอ ไม่ใช่ปัญหาที่กล่อง
```

---

## 9.3 ด่าน 12-2: ระบบโรงเรือนกับสายพานเสียพร้อมกัน

### อาการ

มี 2 ระบบผิดพร้อมกัน

1. โรงเรือนร้อนแต่พัดลมไม่เปิด
2. สายพานส่งผลผลิตผิดกล่อง

เด็กต้องเลือกแก้ทีละระบบ โดยมีจำนวนครั้งจำกัด

### บั๊กที่อาจพบ

- ค่าอุณหภูมิผิด
- คำสั่งสายพานผิด

### เป้าหมาย

แก้ระบบให้ทำงานถูกอย่างน้อย 2 จุด

### Feedback

```text
ระบบแรกเป็นบั๊กตัวเลข ส่วนระบบที่สองเป็นบั๊กคำสั่ง ต้องแก้คนละวิธี
```

---

## 9.4 ด่าน 12-3: วิกฤตฟาร์มก่อนส่งตลาด

### อาการ

หลายระบบทำงานผิดในรอบเดียว เช่น

- ไข่ร้าวเข้าถาดดี
- ฝนตกแต่สปริงเกอร์ยังเปิด
- อุณหภูมิปกติแต่ระบบค้าง
- ผลไม้บางชนิดเข้ากล่องผิด

เด็กต้องวิเคราะห์และแก้ 3 จุดหลักให้ทัน

### บั๊กที่ใช้รวม

- เงื่อนไขไม่ละเอียดพอ
- ลำดับเงื่อนไขผิด
- ขาด Else

### เป้าหมาย

แก้ให้ระบบหลัก 3 จุดกลับมาทำงานถูกต้อง

### Animation

- ไฟเตือนสีแดงในศูนย์ควบคุม
- จุดเสียกระพริบบนแผนที่
- เมื่อตรวจเจอบั๊ก ไฟเปลี่ยนเป็นสีเหลือง
- เมื่อแก้สำเร็จ ไฟเปลี่ยนเป็นสีเขียว
- จบด่านด้วยฟาร์มกลับมาสดใส

### Feedback

```text
คุณแก้ครบทั้ง 3 ระบบแล้ว ฟาร์มพร้อมส่งผลผลิตทันเวลา!
```

## 10.2 เครื่องมือ Debug

ควรมีเครื่องมือหลัก 4 ชิ้น

1. **แว่นขยายจับบั๊ก**

4. **ปุ่มขอดูอาการซ้ำ**  
   เล่น replay ของอาการผิดอีกครั้งโดยไม่ลดดาวมาก หรืออาจลดเล็กน้อยตามความเหมาะสม

# 11. ระบบดาว

## 11.1 ตัวแปรที่ใช้คิดดาว

- จำนวนครั้งที่เลือกบล็อกผิด
- จำนวนครั้งที่แก้ผิด
- จำนวนครั้งที่กดคำใบ้
- จำนวนครั้งที่ทดสอบซ้ำ
- แก้ระบบสำเร็จหรือไม่

## 11.2 เกณฑ์ดาว

```text
3 ดาว = เจอบั๊กถูกจุด แก้ถูกในครั้งแรก ไม่ใช้คำใบ้
2 ดาว = แก้สำเร็จ แต่เลือกผิดหรือทดสอบซ้ำเล็กน้อย
1 ดาว = แก้สำเร็จหลังลองหลายครั้งหรือใช้คำใบ้
0 ดาว = แก้ไม่สำเร็จ
```

# 14. ระบบสร้างชิ้นงานบทที่ 4

บทที่ 4 ควรมีระบบสร้างชิ้นงานที่สอดคล้องกับตัวเกม คือให้นักเรียนสร้าง “โจทย์บั๊กของฉัน”

## 14.1 ชื่อระบบ

```text
สร้างโจทย์บั๊กฟาร์มของฉัน
```

## 14.2 สิ่งที่นักเรียนต้องสร้าง

1. เลือกระบบฟาร์ม เช่น สายพาน ผลไม้ ไข่ โรงเรือน
2. สร้างกฎที่ถูกต้อง
3. เลือกจุดที่จะใส่บั๊ก
4. สร้างกฎที่ผิด
5. เขียนอาการที่จะเกิดขึ้น
6. เขียนวิธีแก้ที่ถูกต้อง
7. ทดลองให้ระบบแสดงอาการผิด
8. ส่งให้เพื่อนหรือครูลองแก้

## 14.3 ตัวอย่างชิ้นงาน


# 24. สรุป

บทที่ 4 ควรเป็นเกม **Smart Farm Debugger: ภารกิจจับบั๊กกู้ฟาร์ม** ที่ต่อยอดจากบทที่ 3 อย่างเป็นธรรมชาติ โดยเปลี่ยนผู้เรียนจากผู้สร้างกฎเป็นช่างซ่อมระบบ

จุดเด่นของแนวนี้คือ

- ไม่ซ้ำบทที่ 3
- ใช้โลกฟาร์มเดิมต่อยอดได้
- สอน Debugging ได้ตรงมาก
- มีภาพจำชัดผ่านแมลงบั๊กและระบบฟาร์มรวน
- ฝึกการสังเกต วิเคราะห์ แก้ไข และทดสอบซ้ำ
- รองรับระบบสร้างชิ้นงานที่ให้เด็กออกแบบโจทย์บั๊กของตนเอง

แกนสำคัญที่สุดคือ

> เด็กต้องไม่ได้แค่ตอบว่าอะไรถูก แต่ต้องสืบให้ได้ว่า “ผิดตรงไหน เพราะอะไร และแก้อย่างไร”
