# แผนการพัฒนาบทเรียนที่ 3  
# เกม “เครื่องรดน้ำอัจฉริยะ (Condition)” รูปแบบ Puzzle Simulation + Block Programming

**โครงการ:** แบบฝึกทักษะออนไลน์เพื่อส่งเสริมทักษะการแก้ปัญหา เรื่อง การแก้ปัญหาอย่างเป็นขั้นตอน  
**บทเรียนที่พัฒนา:** บทที่ 3 เครื่องรดน้ำอัจฉริยะ (Condition)  
**กลุ่มเป้าหมาย:** นักเรียนชั้นประถมศึกษาปีที่ 4  
**แนวทางหลัก:** Game-Based Learning + Puzzle Simulation + Block Programming  
**เป้าหมายการพัฒนา:** ปรับบทเรียนที่ 3 จากแบบฝึกจับคู่คำตอบให้เป็นเกมเชิงระบบที่ผู้เรียนได้ “สร้างเงื่อนไข ทดลองระบบ เห็นผลลัพธ์ และแก้ไขกฎด้วยตนเอง”

---

## 1. บทนำและเหตุผลในการพัฒนา

บทเรียนที่ 3 ในระบบปัจจุบันมีหัวข้อหลักคือ **เครื่องรดน้ำอัจฉริยะ (Condition)** มุ่งฝึกแนวคิดเรื่องเงื่อนไข เช่น `If`, `If-Else`, `If-Else If-Else` โดยใช้บริบทของระบบรดน้ำอัตโนมัติในฟาร์ม เช่น ดินแห้ง ดินชื้น ฝนตก ถังน้ำหมด และการเลือกคำสั่งที่เหมาะสม ได้แก่ รดน้ำ หยุดรดน้ำ แจ้งเตือนเติมน้ำ หรือสังเกตต่อ

อย่างไรก็ตาม รูปแบบเกมเดิมยังมีลักษณะใกล้เคียงกับ “แบบฝึกจับคู่คำตอบ” มากกว่า “เกม” กล่าวคือ ผู้เรียนอ่านสถานการณ์แต่ละแปลง แล้วลากคำสั่งไปวางให้ตรงกับคำตอบที่กำหนดไว้ล่วงหน้า แม้กิจกรรมดังกล่าวมีการโต้ตอบและมีการตรวจคำตอบ แต่ยังไม่ทำให้ผู้เรียนได้สัมผัสแก่นสำคัญของการใช้เงื่อนไขอย่างแท้จริง เพราะผู้เรียนยังไม่ได้ “สร้างกฎ” หรือ “ทดสอบระบบ” ที่เกิดจากกฎของตนเอง

ดังนั้น จึงควรพัฒนาบทเรียนที่ 3 ใหม่ให้มีความเป็นเกมมากขึ้น โดยเปลี่ยนแกนกิจกรรมจาก

> อ่านสถานการณ์ → ลากคำตอบ → ตรวจถูกผิด

เป็น

> อ่านข้อมูลเซ็นเซอร์ → สร้างกฎ If-Then-Else → รันระบบ → สังเกตผลลัพธ์ → แก้ไขกฎ → ผ่านภารกิจ

แนวทางใหม่นี้จะช่วยให้ผู้เรียนเข้าใจว่า “เงื่อนไข” ไม่ใช่เพียงการเลือกคำตอบที่ถูกต้อง แต่คือการออกแบบกฎให้ระบบตัดสินใจอย่างเป็นลำดับขั้น ซึ่งเป็นหัวใจของวิทยาการคำนวณและการแก้ปัญหาอย่างเป็นระบบ

## 2. วิเคราะห์ปัญหาของบทเรียนที่ 3 ปัจจุบัน

### 2.1 จุดแข็งของระบบเดิม

ระบบเดิมมีฐานที่ดีหลายด้าน ได้แก่

1. มีโครงสร้างบทเรียนชัดเจน แยกเป็นบทที่ 1 Logic, บทที่ 2 Sequence, บทที่ 3 Condition และบทที่ 4 Debugging
2. บทที่ 3 มีบริบทที่เหมาะสมกับเด็ก คือฟาร์มและเครื่องรดน้ำอัจฉริยะ
3. มีการใช้ Phaser.js และ JavaScript ทำให้สามารถพัฒนาเป็นเกมเชิงโต้ตอบได้
4. มีระบบบันทึกคะแนน ดาว เวลา และจำนวนครั้งที่พยายาม
5. มีระบบด่าน ได้แก่ stage 7, stage 8 และ stage 9 ซึ่งสามารถต่อยอดได้โดยไม่ต้องรื้อโครงสร้างหลัก
6. มีหน้าคู่มือและคำอธิบายบทเรียนที่เชื่อมโยงกับ If-Then-Else แล้ว
7. มีหน้าให้ผู้เรียนสร้างชิ้นงานหลังผ่านด่านครบ ซึ่งสามารถต่อยอดเป็นภารกิจสร้างกฎระบบรดน้ำของตนเองได้

### 2.2 ข้อจำกัดของรูปแบบเกมเดิม

ปัญหาสำคัญของบทเรียนที่ 3 คือกิจกรรมยังไม่สร้างประสบการณ์แบบเกมเต็มรูปแบบ เนื่องจาก

1. **ผู้เรียนยังไม่ได้สร้างกฎจริง**  
   ผู้เรียนเพียงเลือกคำสั่งให้ตรงกับสถานการณ์ เช่น ดินแห้งให้รดน้ำ ฝนตกให้หยุดรดน้ำ แต่ไม่ได้ประกอบโครงสร้างเงื่อนไขด้วยตนเอง

2. **ไม่มีผลลัพธ์เชิงระบบที่เกิดจากกฎของผู้เรียน**  
   ถ้าผู้เรียนตอบผิด ระบบเพียงแจ้งว่าผิด แต่ยังไม่แสดงผลกระทบเชิงสถานการณ์ เช่น พืชเหี่ยว น้ำท่วม หรือน้ำในถังหมด

3. **ยังไม่เห็นความสำคัญของลำดับเงื่อนไข**  
   การเรียนเรื่อง If-Else โดยเฉพาะในด่านฝนตกและดินแห้ง ควรทำให้ผู้เรียนเห็นว่าการตรวจเงื่อนไขก่อนหรือหลังมีผลต่อผลลัพธ์ แต่ระบบเดิมยังไม่ทำให้เรื่องนี้เด่นชัด

4. **ความท้าทายยังน้อย**  
   การจับคู่สถานการณ์กับคำตอบเป็นกิจกรรมที่จบเร็วและคาดเดาได้ง่าย ทำให้ขาดความรู้สึกของการลองผิดลองถูก การวางแผน และการแก้ปัญหา

5. **ยังไม่เกิดวงจรเกมที่สมบูรณ์**  
   เกมที่ดีควรมีวงจรหลัก ได้แก่ เป้าหมาย → การตัดสินใจ → การกระทำ → ผลลัพธ์ → feedback → ปรับกลยุทธ์ แต่กิจกรรมเดิมยังมีเพียงคำตอบถูก/ผิด

## 3. แนวคิดการออกแบบใหม่

### 3.1 รูปแบบเกมที่เสนอ

บทเรียนที่ 3 ควรพัฒนาเป็นเกมแนว

> **Puzzle Simulation + Block Programming**

โดยใช้ชื่อแนวคิดว่า

> **“วิศวกรระบบน้ำอัจฉริยะ”**

ผู้เรียนรับบทเป็นวิศวกรน้อยของฟาร์ม ที่ต้องสร้างกฎให้เครื่องรดน้ำอัตโนมัติสามารถดูแลแปลงผักได้ถูกต้องตามเงื่อนไขต่าง ๆ เช่น ดินแห้ง ฝนตก ถังน้ำหมด หรือดินชื้นพอดี

### 3.2 แกนหลักของเกม

ผู้เรียนจะไม่ได้เลือกคำตอบทีละสถานการณ์ แต่จะต้องสร้าง “ชุดกฎ” ให้ระบบนำไปใช้กับทุกสถานการณ์ แล้วกดทดสอบเพื่อดูผลลัพธ์

ตัวอย่างกฎที่ผู้เรียนต้องสร้างในด่านสูงขึ้น:

```text
ถ้าถังน้ำหมด → แจ้งเตือนเติมน้ำ
มิฉะนั้นถ้าฝนตก → หยุดรดน้ำ
มิฉะนั้นถ้าดินแห้ง → รดน้ำ
มิฉะนั้น → สังเกตต่อ
```

### 3.3 ประสบการณ์ที่ต้องการให้ผู้เรียนได้รับ

ผู้เรียนควรได้ประสบการณ์ดังนี้

1. อ่านข้อมูลจากเซ็นเซอร์
2. เลือกเงื่อนไขที่เกี่ยวข้อง
3. จัดลำดับเงื่อนไข
4. เลือกคำสั่งที่เหมาะสมกับแต่ละเงื่อนไข
5. รันระบบเพื่อดูผลลัพธ์
6. วิเคราะห์ข้อผิดพลาดจากผลลัพธ์
7. ปรับปรุงกฎจนระบบทำงานถูกต้อง
8. สรุปแนวคิด If-Then-Else จากการเล่น

## 4. เป้าหมายการพัฒนา

### 4.1 เป้าหมายด้านการเรียนรู้

เมื่อผู้เรียนจบบทเรียนที่ 3 แล้ว ควรสามารถ

1. อธิบายความหมายของเงื่อนไขแบบ `ถ้า...แล้ว...`
2. เลือกคำสั่งให้เหมาะสมกับสถานการณ์
3. เข้าใจการทำงานของ `If-Else`
4. เข้าใจการทำงานของหลายเงื่อนไขแบบ `If-Else If-Else`
5. เห็นความสำคัญของลำดับการตรวจเงื่อนไข
6. อธิบายเหตุผลของกฎที่ตนเองออกแบบได้
7. ทดสอบและแก้ไขกฎเมื่อผลลัพธ์ไม่ถูกต้อง
8. เชื่อมโยงการใช้เงื่อนไขกับการแก้ปัญหาในชีวิตจริง

### 4.2 เป้าหมายด้านความเป็นเกม

ระบบใหม่ควรมีองค์ประกอบของเกมที่ชัดเจน ได้แก่

1. มีบทบาทของผู้เล่น: วิศวกรระบบน้ำอัจฉริยะ
2. มีภารกิจ: ช่วยฟาร์มให้รอดจากดินแห้ง ฝนตก น้ำท่วม และถังน้ำหมด
3. มีความท้าทาย: ต้องสร้างกฎให้ถูกต้องและครอบคลุม
4. มีการทดลอง: กดรันระบบเพื่อดูผลลัพธ์
5. มีผลลัพธ์เชิงภาพ: พืชฟื้น พืชเหี่ยว น้ำท่วม ถังน้ำหมด
6. มี feedback เชิงเหตุผล: บอกว่าผิดเพราะอะไร ไม่ใช่แค่ผิด
7. มีคะแนนและดาว: วัดจากความถูกต้อง ความรวดเร็ว จำนวนครั้ง และการใช้คำใบ้
8. มีความก้าวหน้า: จากเงื่อนไขเดี่ยว ไปสู่หลายเงื่อนไข
9. มีภารกิจสรุป: ออกแบบระบบรดน้ำของตนเองหลังผ่านด่าน

# 6. รายละเอียดการออกแบบแต่ละด่าน

---

## 6.1 Stage 7: เช็คดินแห้งหรือเปียก

### 6.1.1 แนวคิดการเรียนรู้

ด่านนี้เป็นด่านพื้นฐานของ `If-Else` โดยให้ผู้เรียนเข้าใจว่า ระบบจะตรวจเงื่อนไขหนึ่งอย่าง แล้วเลือกทำคำสั่งที่แตกต่างกันตามผลของเงื่อนไข

แนวคิดหลัก:

```text
ถ้าดินแห้ง → รดน้ำ
มิฉะนั้น → หยุดรดน้ำ
```

### 6.1.2 ภารกิจในเกม

ผู้เรียนได้รับมอบหมายให้ตั้งค่าระบบรดน้ำสำหรับแปลงผัก 2 แปลง

- แปลง A: ดินแห้ง ใบเริ่มเหี่ยว
- แปลง B: ดินชื้นพอดี ต้นกล้าสดชื่น

ผู้เรียนต้องลากบล็อกเงื่อนไขและบล็อกคำสั่งลงในแผงกฎ จากนั้นกด “ทดสอบระบบ”

### 6.1.3 องค์ประกอบหน้าจอ

หน้าจอควรประกอบด้วย

1. **พื้นที่แสดงแปลงผัก**
   - แปลง A
   - แปลง B
   - แถบความชื้นดิน
   - ภาพพืชเหี่ยวหรือสดชื่น
   - ไอคอนดินแห้งหรือดินชื้น

2. **แผงบล็อกเงื่อนไข**
   - `ถ้าดินแห้ง`
   - `ถ้าดินชื้น`
   - `มิฉะนั้น`

3. **แผงบล็อกคำสั่ง**
   - `รดน้ำ`
   - `หยุดรดน้ำ`

4. **พื้นที่สร้างกฎ**
   - ช่อง If
   - ช่อง Then
   - ช่อง Else

5. **ปุ่มควบคุม**
   - ทดสอบระบบ
   - ล้างกฎ
   - คำใบ้

### 6.1.4 ลำดับการเล่น

1. ระบบแสดงแปลงผัก 2 แปลง
2. ผู้เรียนอ่านสถานะของแต่ละแปลง
3. ผู้เรียนลากบล็อก `ถ้าดินแห้ง` ไปวางในช่อง If
4. ผู้เรียนลากบล็อก `รดน้ำ` ไปวางในช่อง Then
5. ผู้เรียนลากบล็อก `หยุดรดน้ำ` ไปวางในช่อง Else
6. ผู้เรียนกด “ทดสอบระบบ”
7. ระบบจำลองผลลัพธ์
8. ถ้าถูกต้อง พืชในแปลง A ฟื้น และแปลง B ยังคงปกติ
9. ถ้าผิด ระบบแสดงผลกระทบ เช่น แปลง A ยังเหี่ยว หรือแปลง B น้ำมากเกินไป
10. ผู้เรียนแก้ไขกฎจนผ่านด่าน

### 6.1.5 Feedback ที่ควรมี

ตัวอย่าง feedback เมื่อผิด:

- “แปลง A ดินแห้ง แต่ระบบไม่ได้รดน้ำ พืชจึงยังเหี่ยวอยู่”
- “แปลง B ดินชื้นแล้ว แต่ระบบยังรดน้ำ ทำให้น้ำมากเกินไป”
- “ลองคิดว่า ถ้าดินแห้ง ระบบควรทำอะไร และถ้าไม่แห้งควรทำอะไร”

ตัวอย่าง feedback เมื่อถูก:

- “ยอดเยี่ยม! ระบบตรวจดินแห้งแล้วรดน้ำได้ถูกต้อง”
- “คุณเข้าใจ If-Else พื้นฐานแล้ว”

### 6.1.6 เกณฑ์ผ่านด่าน

ผ่านด่านเมื่อ

1. แปลงดินแห้งได้รับน้ำ
2. แปลงดินชื้นไม่ถูกรดน้ำเพิ่ม
3. กฎมีโครงสร้าง If-Else ครบถ้วน

## 6.2 Stage 8: รับมือฝนฟ้าคะนอง

### 6.2.1 แนวคิดการเรียนรู้

ด่านนี้เน้น “ลำดับความสำคัญของเงื่อนไข” โดยเฉพาะกรณีที่มีมากกว่า 1 เงื่อนไขเกิดพร้อมกัน เช่น ฝนตกและดินแห้ง

แนวคิดหลัก:

```text
ถ้าฝนตก → หยุดรดน้ำ
มิฉะนั้นถ้าดินแห้ง → รดน้ำ
มิฉะนั้น → หยุดรดน้ำ
```

### 6.2.2 ภารกิจในเกม

ฟาร์มกำลังมีเมฆฝน ระบบรดน้ำต้องตัดสินใจให้ถูกต้อง ไม่เปิดน้ำซ้ำขณะฝนตก และยังต้องช่วยแปลงที่ดินแห้งเมื่อไม่มีฝน

สถานการณ์ในด่าน:

- แปลง A: ฝนตกและดินแห้ง
- แปลง B: ไม่มีฝนและดินแห้ง
- แปลง C: ไม่มีฝนแต่ดินชื้น

### 6.2.3 จุดเน้นของเกม

ด่านนี้ต้องทำให้ผู้เรียนเห็นว่า

> ถ้าตรวจ “ดินแห้ง” ก่อน “ฝนตก” ระบบอาจรดน้ำในวันที่ฝนตก ทำให้เกิดน้ำท่วม

นี่คือจุดสำคัญของการสอนหลายเงื่อนไข ผู้เรียนจะเข้าใจว่าลำดับของกฎมีผลต่อผลลัพธ์

### 6.2.4 องค์ประกอบหน้าจอ

1. **พื้นที่แสดงแปลงผัก 3 แปลง**
   - แสดงฝนตก/ไม่ตก
   - แสดงดินแห้ง/ดินชื้น
   - แสดงระดับน้ำในแปลง

2. **บล็อกเงื่อนไข**
   - `ถ้าฝนตก`
   - `ถ้าดินแห้ง`
   - `ถ้าดินชื้น`
   - `มิฉะนั้น`

3. **บล็อกคำสั่ง**
   - `รดน้ำ`
   - `หยุดรดน้ำ`
   - `รอดูสถานการณ์`

4. **พื้นที่สร้างกฎหลายชั้น**
   - If
   - Else If
   - Else

5. **ปุ่มทดสอบระบบ**
   - รันระบบ 1 รอบ
   - แสดงผลลัพธ์แต่ละแปลง

### 6.2.5 ลำดับการเล่น

1. ระบบแนะนำว่า “วันนี้มีโอกาสฝนตก ต้องระวังน้ำมากเกินไป”
2. ผู้เรียนสร้างกฎโดยลากบล็อกลงในลำดับที่ต้องการ
3. ผู้เรียนกด “ทดสอบระบบ”
4. ระบบจำลองการทำงานกับทุกแปลง
5. ถ้ากฎตรวจฝนก่อน แปลงฝนตกจะหยุดรดน้ำ
6. ถ้ากฎตรวจดินแห้งก่อน แปลง A จะเกิดน้ำท่วม
7. ระบบให้ feedback ตามผลลัพธ์
8. ผู้เรียนแก้ลำดับกฎจนถูกต้อง
9. เมื่อผ่าน ระบบสรุปว่า “การเรียงเงื่อนไขสำคัญมาก”

### 6.2.6 Feedback ที่ควรมี

กรณีผู้เรียนตรวจดินก่อนฝน:

- “ระบบเห็นว่าดินแห้งจึงรดน้ำทันที แต่ลืมดูว่าฝนกำลังตก ทำให้แปลง A น้ำท่วม”
- “ลองตรวจเงื่อนไขฝนตกก่อน แล้วค่อยตรวจดินแห้ง”

กรณีไม่ใส่ Else:

- “บางสถานการณ์ยังไม่มีกฎรองรับ ระบบจึงไม่รู้ว่าต้องทำอะไร”
- “ควรมีคำสั่งสำรองในกรณีที่ไม่เข้าเงื่อนไขใดเลย”

กรณีถูกต้อง:

- “เยี่ยมมาก! คุณจัดลำดับเงื่อนไขได้ถูกต้อง ระบบจึงไม่รดน้ำซ้ำตอนฝนตก”

### 6.2.7 เกณฑ์ผ่านด่าน

ผ่านด่านเมื่อ

1. แปลงฝนตกไม่ถูกรดน้ำเพิ่ม
2. แปลงไม่มีฝนและดินแห้งได้รับน้ำ
3. แปลงไม่มีฝนแต่ดินชื้นหยุดรดน้ำ
4. เงื่อนไขฝนตกถูกตรวจเป็นลำดับแรก

## 6.3 Stage 9: ระบบน้ำครอบจักรวาล

### 6.3.1 แนวคิดการเรียนรู้

ด่านนี้เป็นด่านสรุปของบทเรียน Condition โดยให้ผู้เรียนใช้หลายเงื่อนไขร่วมกัน และคำนึงถึงทรัพยากรจำกัด เช่น น้ำในถัง

แนวคิดหลัก:

```text
ถ้าถังน้ำหมด → แจ้งเตือนเติมน้ำ
มิฉะนั้นถ้าฝนตก → หยุดรดน้ำ
มิฉะนั้นถ้าดินแห้ง → รดน้ำ
มิฉะนั้น → สังเกตต่อ
```

### 6.3.2 ภารกิจในเกม

ผู้เรียนต้องตั้งค่าระบบรดน้ำอัจฉริยะเพื่อดูแลฟาร์มหลายแปลงให้รอดภายในจำนวนรอบที่กำหนด เช่น 5 รอบ โดยต้องไม่ให้น้ำหมด ไม่ให้น้ำท่วม และไม่ให้พืชเหี่ยว

### 6.3.3 สถานการณ์ตัวอย่าง

ระบบอาจสุ่มหรือกำหนดสถานการณ์ เช่น

| แปลง | สถานะดิน | ฝน | ถังน้ำ | ผลที่ควรเกิด |
|---|---|---|---|---|
| A | แห้ง | ไม่มีฝน | พร้อม | รดน้ำ |
| B | ชื้น | ฝนตก | พร้อม | หยุดรดน้ำ |
| C | แห้ง | ไม่มีฝน | หมด | แจ้งเตือนเติมน้ำ |
| D | ชื้น | ไม่มีฝน | พร้อม | สังเกตต่อ |

### 6.3.4 องค์ประกอบใหม่ที่เพิ่มความเป็นเกม

ด่านนี้ควรเพิ่มระบบต่อไปนี้

1. **ค่าพลังชีวิตของพืช**
   - เริ่มต้น 70–100
   - ถ้าดินแห้งแล้วไม่รดน้ำ พลังชีวิตลด
   - ถ้ารดน้ำถูกต้อง พลังชีวิตเพิ่ม
   - ถ้ารดน้ำตอนฝนตกหรือดินชื้นมาก พลังชีวิตลดจากน้ำท่วม

2. **ระดับน้ำในถัง**
   - รดน้ำแต่ละครั้งใช้น้ำ
   - ถังน้ำหมดแล้วรดน้ำไม่ได้
   - ต้องแจ้งเตือนเติมน้ำก่อน

3. **จำนวนรอบจำลอง**
   - เช่น 5 รอบ
   - แต่ละรอบเปลี่ยนสภาพอากาศและความชื้น
   - ผู้เรียนต้องออกแบบกฎให้ใช้ได้กับหลายสถานการณ์

4. **คะแนนประสิทธิภาพ**
   - พืชรอดครบทุกแปลง
   - ใช้น้ำน้อย
   - แก้กฎน้อยครั้ง
   - ไม่ใช้คำใบ้

### 6.3.5 ลำดับการเล่น

1. ระบบแสดงสถานการณ์ฟาร์มหลายแปลง
2. ผู้เรียนสร้างกฎหลายชั้น
3. ผู้เรียนกด “รันระบบ”
4. ระบบจำลองทีละแปลงหรือทีละรอบ
5. ระบบแสดงผลลัพธ์ เช่น
   - พืชฟื้น
   - น้ำท่วม
   - ถังน้ำหมด
   - ระบบแจ้งเตือนเติมน้ำ
6. ผู้เรียนดูรายงานผลหลังรัน
7. ถ้าผิด ผู้เรียนปรับลำดับเงื่อนไขหรือคำสั่ง
8. เมื่อทุกแปลงรอดครบตามเงื่อนไข ระบบให้ผ่านด่าน

### 6.3.6 Feedback ที่ควรมี

ตัวอย่าง feedback:

- “ถังน้ำหมด แต่ระบบยังพยายามรดน้ำ จึงไม่สามารถช่วยพืชได้”
- “ควรตรวจถังน้ำก่อน เพราะถ้าไม่มีน้ำ ระบบไม่ควรสั่งรดน้ำ”
- “ฝนตกแล้ว แต่ระบบยังรดน้ำเพิ่ม ทำให้แปลง B น้ำมากเกินไป”
- “กฎของคุณครอบคลุมทุกสถานการณ์แล้ว ระบบดูแลฟาร์มได้สำเร็จ”

### 6.3.7 เกณฑ์ผ่านด่าน

ผ่านด่านเมื่อ

1. พืชทุกแปลงมีสุขภาพไม่ต่ำกว่าเกณฑ์ เช่น 70%
2. ไม่มีแปลงใดน้ำท่วม
3. ไม่มีแปลงใดเหี่ยวตาย
4. ระบบแจ้งเตือนเติมน้ำเมื่อถังน้ำหมด
5. กฎทำงานถูกต้องกับทุกสถานการณ์ทดสอบ

# 9. ระบบคะแนนและดาว

## 9.1 คะแนนที่ควรเก็บ

ระบบควรเก็บคะแนนจาก

1. จำนวนครั้งที่กดทดสอบ
2. จำนวนข้อผิดพลาด
3. เวลาในการผ่านด่าน
4. การใช้คำใบ้
5. ความสมบูรณ์ของกฎ
6. ประสิทธิภาพของระบบ เช่น ใช้น้ำน้อยและพืชรอด

## 9.2 เกณฑ์ดาว

### 3 ดาว

ได้เมื่อ

- ผ่านภารกิจ
- กฎถูกต้องครบถ้วน
- ไม่เกิดน้ำท่วมหรือพืชเหี่ยว
- ใช้คำใบ้ 0 ครั้ง
- กดทดสอบไม่เกิน 2 ครั้ง

### 2 ดาว

ได้เมื่อ

- ผ่านภารกิจ
- มีข้อผิดพลาดเล็กน้อยระหว่างทดสอบ
- ใช้คำใบ้ไม่เกิน 1 ครั้ง
- กดทดสอบไม่เกิน 4 ครั้ง

### 1 ดาว

ได้เมื่อ

- ผ่านภารกิจ
- ใช้คำใบ้หลายครั้ง
- กดทดสอบหลายครั้ง
- ผ่านหลังจากเกิดข้อผิดพลาดหลายรอบ

## 9.3 ตัวอย่างการคำนวณดาว

```javascript
function calculateStars({ attempts, mistakes, hintUsed, waterWaste, allPlantsSafe }) {
    if (!allPlantsSafe) return 0;
    if (!hintUsed && attempts <= 2 && mistakes === 0 && waterWaste === 0) return 3;
    if (attempts <= 4 && mistakes <= 2) return 2;
    return 1;
}
```

# 11. Logic การจำลองระบบ

## 11.1 วิธีอ่านกฎของผู้เรียน

เมื่อผู้เรียนสร้างกฎ ระบบควรแปลงบล็อกเป็นชุดข้อมูล เช่น

```javascript
[
    { condition: 'tank_empty', action: 'refill' },
    { condition: 'rain', action: 'stop' },
    { condition: 'soil_dry', action: 'water' },
    { condition: 'else', action: 'observe' }
]
```

## 11.2 วิธีรันระบบ

ระบบควรนำกฎไปตรวจทีละแปลง โดยอ่านจากบนลงล่าง

```javascript
function evaluateRules(plot, rules) {
    for (const rule of rules) {
        if (rule.condition === 'else' || checkCondition(plot, rule.condition)) {
            return rule.action;
        }
    }
    return 'no_action';
}
```

## 11.3 วิธีตรวจเงื่อนไข

```javascript
function checkCondition(plot, condition) {
    if (condition === 'soil_dry') return plot.soil === 'dry';
    if (condition === 'soil_wet') return plot.soil === 'wet';
    if (condition === 'rain') return plot.rain === true;
    if (condition === 'tank_empty') return plot.tank === 'empty';
    if (condition === 'tank_ready') return plot.tank === 'ready';
    return false;
}
```

## 11.4 วิธีคำนวณผลลัพธ์

```javascript
function applyAction(plot, action) {
    if (action === 'water') {
        if (plot.rain || plot.soil === 'wet') {
            plot.result = 'flood';
            plot.health -= 20;
        } else if (plot.tank === 'empty') {
            plot.result = 'no_water';
            plot.health -= 10;
        } else {
            plot.result = 'healthy';
            plot.health += 20;
        }
    }

    if (action === 'stop') {
        if (plot.soil === 'dry' && !plot.rain) {
            plot.result = 'dry';
            plot.health -= 15;
        } else {
            plot.result = 'safe';
        }
    }

    if (action === 'refill') {
        if (plot.tank === 'empty') {
            plot.result = 'refill_needed';
        } else {
            plot.result = 'unnecessary_refill';
        }
    }

    if (action === 'observe') {
        plot.result = 'observe';
    }

    plot.health = Math.max(0, Math.min(100, plot.health));
    return plot;
}
```

# 20. สรุปแนวทางการพัฒนา

บทเรียนที่ 3 ควรถูกพัฒนาให้เป็นเกม “วิศวกรระบบน้ำอัจฉริยะ” ที่ผู้เรียนได้สร้างกฎ If-Then-Else เพื่อควบคุมเครื่องรดน้ำอัตโนมัติ ไม่ใช่เพียงลากคำตอบไปจับคู่กับสถานการณ์ การเปลี่ยนแปลงนี้จะทำให้บทเรียนมีความเป็น Game-Based Learning ชัดเจนขึ้น เพราะผู้เรียนมีเป้าหมาย มีภารกิจ มีความท้าทาย ได้ทดลอง ได้เห็นผลลัพธ์ ได้รับ feedback และได้ปรับปรุงวิธีคิดของตนเอง

แกนสำคัญของการพัฒนาคือ

> “ให้ผู้เรียนสร้างกฎ แล้วให้เกมแสดงผลของกฎนั้น”

หากทำได้ตามแผนนี้ บทเรียนที่ 3 จะกลายเป็นบทเรียนที่มีพลังมากขึ้น ทั้งในด้านความสนุก ความเข้าใจเชิงนามธรรม และความสอดคล้องกับเป้าหมายของวิทยาการคำนวณระดับประถมศึกษา
