## 1. บทสรุปแนวทางปรับปรุง

บทเรียนที่ 3 จะใช้ชื่อรวมว่า

> **บทที่ 3: ผู้จัดการฟาร์มอัจฉริยะ**  
> **Smart Farm Manager**

แนวคิดหลักคือให้ผู้เรียนเป็นผู้จัดการฟาร์มที่ต้องเขียนกฎเงื่อนไขเพื่อควบคุมระบบอัตโนมัติในฟาร์มผ่าน **สายพานลำเลียงสิ่งของ/เหตุการณ์** และ **บล็อกคำสั่งแบบลากวาง**

ระบบใหม่ต้องเน้นว่า

> เด็กไม่ได้กดตอบคำถาม แต่กำลังลากบล็อกเพื่อเขียนโปรแกรมควบคุมฟาร์มให้ทำงานจริง

โครงสร้างใหม่ของบทเรียนที่ 3 คือ

| Stage | เกม | ตรรกะหลัก | บทบาทของด่านย่อย |
|---|---|---|---|
| Stage 7 | โรงคัดแยกผลผลิตของพี่สมปอง | If | ไต่ระดับด้วยชนิดวัตถุ ตัวหลอก และความเร็วสายพาน |
| Stage 8 | หุ่นยนต์เลี้ยงสัตว์อัจฉริยะ | If / Else | ไต่ระดับด้วยประเภทสัตว์ ผลของ Else และลักษณะสังเกตที่ซับซ้อนขึ้น |
| Stage 9 | ผู้ดูแลโรงเรือนไฮเทค | If / Else If / Else | ไต่ระดับด้วยตัวเลข ช่วงค่า สภาพแวดล้อม และลำดับเงื่อนไข |

หมายเหตุสำคัญ: ยังใช้ `stage_id` เดิมคือ 7, 8, 9 แต่ในแต่ละ stage จะมี **ด่านย่อย 3 ด่านภายในเกมเดียวกัน** โดยระบบจะส่งคะแนนผ่าน `window.sendResult()` เมื่อผู้เรียนผ่านด่านย่อยครบทั้งหมดของ stage นั้น หรือเมื่อจบ stage ตามกติกาที่กำหนด

## 2. ปัญหาที่พบจากการทดสอบและแนวทางแก้ไข

### 2.1 ชื่อด่านและชื่อเกมไม่สอดคล้องกับตัวเกมจริง

**ปัญหา:**  
ชื่อที่แสดงตั้งแต่ Dashboard, หน้าเลือกบทเรียน, หน้าเลือกด่าน และหน้าเล่นเกมยังไม่สอดคล้องกับคอนเซ็ปต์ใหม่ ทำให้ผู้เรียนสับสนว่าเกมกำลังสอนเรื่องใด

**แนวทางแก้ไข:**  
เปลี่ยนชื่อบทเรียนที่ 3 ทั้งระบบให้เป็นชุดเดียวกัน ดังนี้

- ชื่อบทหลัก: **บทที่ 3: ผู้จัดการฟาร์มอัจฉริยะ**
- ชื่ออังกฤษ: **Smart Farm Manager**
- คำอธิบายสั้น: **ฝึกใช้เงื่อนไขควบคุมระบบฟาร์มอัตโนมัติ**

ชื่อเกมย่อย:

| Stage | ชื่อเดิมที่ควรเลิกใช้ | ชื่อใหม่ |
|---|---|---|
| Stage 7 | เครื่องรดน้ำอัจฉริยะ / ชื่อไม่ตรงระบบ | เกมที่ 1: โรงคัดแยกผลผลิตของพี่สมปอง |
| Stage 8 | ชื่อด่านที่ไม่สะท้อนเกม | เกมที่ 2: หุ่นยนต์เลี้ยงสัตว์อัจฉริยะ |
| Stage 9 | ชื่อด่านที่ไม่สะท้อนเกม | เกมที่ 3: ผู้ดูแลโรงเรือนไฮเทค |

### 2.2 โครงสร้างความรู้เดิมยังไม่ชัดเจนพอ

**ปัญหา:**  
เดิมแต่ละเกมมี If, If/Else, If/Else If/Else อยู่ภายในเกมเดียวกัน ทำให้แกนการเรียนรู้ของแต่ละเกมไม่ชัด

**แนวทางแก้ไข:**  
ปรับใหม่เป็น

- เกมที่ 1 ทั้งเกมสอน **If**
- เกมที่ 2 ทั้งเกมสอน **If / Else**
- เกมที่ 3 ทั้งเกมสอน **If / Else If / Else**

ส่วนด่านย่อย 3 ด่านของแต่ละเกมใช้เพิ่มความยากด้วยปัจจัยอื่น เช่น จำนวนวัตถุ ตัวหลอก ความเร็วสายพาน ความคล้ายกันของภาพ ตัวเลข และช่วงค่า

### 2.3 การแสดงผลวัตถุคลาดเคลื่อนเล็กน้อยเมื่อเข้าเครื่องสแกน

**ปัญหา:**  
วัตถุที่วิ่งบนสายพานดูเหมือนเข้าเครื่องสแกนไม่ตรงตำแหน่ง หรือคลาดเคลื่อนจากจุดตรวจ ทำให้ภาพไม่เนียนและลดความรู้สึกเป็นเกม

**แนวทางแก้ไขเชิงเทคนิค:**

1. กำหนดตำแหน่งกลางของเครื่องสแกนเป็นค่าคงที่ เช่น `SCAN_POINT.x`, `SCAN_POINT.y`
2. วัตถุทุกชิ้นต้องใช้ `origin = 0.5, 0.5`
3. เมื่อวัตถุเข้าสู่โซนสแกน ต้อง tween ไป snap ที่จุดสแกนก่อนเสมอ
4. หยุดวัตถุ 300–500 ms เพื่อเล่น animation สแกน
5. ประเมินเงื่อนไขหลัง snap เสร็จเท่านั้น
6. หลังประเมินผลแล้วจึง tween ไปยังปลายทาง

ตัวอย่าง logic:

```javascript
// SCAN ALIGNMENT NOTE:
// ก่อนตรวจเงื่อนไข ต้องจัดตำแหน่งวัตถุให้ตรงกลางเครื่องสแกนเสมอ
// เพื่อป้องกัน sprite คลาดเคลื่อนหรือดูเหมือนยังไม่เข้าเครื่องสแกน
async function processItemAtScanner(item) {
  item.setOrigin(0.5, 0.5);
  await moveItemTo(item, SCAN_POINT.x, SCAN_POINT.y, 180);
  await playScanAnimation(item);
  const action = evaluateRules(item.getData('props'), currentRules);
  await routeItemByAction(item, action);
}
```

### 2.4 ต้องรองรับการเปลี่ยนจากไอคอนเป็นภาพกราฟิกในอนาคต

**ปัญหา:**  
ถ้าเขียน logic ผูกกับ emoji หรือ icon โดยตรง ในอนาคตจะเปลี่ยนเป็นภาพ PNG หรือ Sprite Sheet ได้ยาก

**แนวทางแก้ไข:**  
ออกแบบระบบ asset abstraction โดยทุกวัตถุต้องมีข้อมูลดังนี้

- `fallbackIcon`
- `asset.key`
- `asset.path`
- `asset.width`
- `asset.height`
- `asset.description`
- `futureSpriteSheet`
- `props` สำหรับ logic

ตัวอย่าง config:

```javascript
{
  id: 'dirty_carrot',
  label: 'แครอทเปื้อนโคลน',
  fallbackIcon: '🥕',
  asset: {
    key: 'dirty_carrot',
    path: '../assets/img/conveyor/produce/dirty_carrot.png',
    width: 96,
    height: 96,
    description: 'ภาพแครอทมีคราบโคลน ใช้ในเกมโรงคัดแยกผลผลิต ด่าน If',
    futureSpriteSheet: {
      path: '../assets/img/conveyor/produce/dirty_carrot_sheet.png',
      frameWidth: 96,
      frameHeight: 96,
      animations: ['idle', 'move', 'bounce_correct', 'drop_wrong']
    }
  },
  props: {
    type: 'carrot',
    dirty: true
  }
}
```

คอมเมนต์ที่ต้องใส่ในโค้ด:

```javascript
// ASSET NOTE:
// ตอนนี้ระบบสามารถใช้ fallbackIcon เพื่อให้ prototype ทำงานได้ทันที
// ในอนาคตสามารถเปลี่ยนเป็น PNG หรือ Sprite Sheet ได้โดยแก้เฉพาะ levelConfig.asset
// ห้ามผูก logic กับ emoji โดยตรง เพราะ emoji เป็นเพียง fallback ด้านการแสดงผล
// Logic ต้องอ่านจาก props เท่านั้น เช่น type, dirty, size, ripeness, weight, moisture, temperature
```

### 2.5 คะแนน คอมโบ และเหรียญไม่จำเป็นในรอบนี้

**ปัญหา:**  
ระบบคะแนน คอมโบ เหรียญ ซ้ำซ้อนกับดาวซึ่งเป็นคะแนนหลัก และอาจทำให้ UI รกเกินไป

**แนวทางแก้ไข:**  
ตัดระบบต่อไปนี้ออกจากเวอร์ชันปรับปรุงรอบนี้

- คะแนนตัวเลขแบบละเอียด
- Combo
- เหรียญ
- ร้านค้าแต่งฟาร์ม

ให้เหลือเฉพาะ

1. ดาว 1–3 ดวง
2. จำนวนถูก/ผิด
3. จำนวนวัตถุเสียหาย
4. เวลา หากจำเป็นต่อการให้ดาว

เกณฑ์ดาวที่เสนอ:

```text
3 ดาว = ถูกต้องอย่างน้อย 90% และเสียหายไม่เกิน 1 รายการ
2 ดาว = ถูกต้องอย่างน้อย 75%
1 ดาว = ถูกต้องอย่างน้อย 60%
0 ดาว = ต่ำกว่า 60% หรือกฎไม่ครบ
```

### 2.6 Drag & Drop ต้องทำให้ได้จริง

**ปัญหา:**  
การแก้หลายรอบยังไม่ทำให้ Drag & Drop ใช้งานได้จริง ผู้เรียนยังไม่สามารถลากบล็อกเงื่อนไขและบล็อกคำสั่งไปวางในแผงสร้างโปรแกรมได้อย่างเป็นธรรมชาติ

**แนวทางแก้ไข:**  
กำหนดให้ Drag & Drop เป็น requirement หลัก ไม่ใช่ optional

สิ่งที่ต้องลากได้:

1. บล็อกเงื่อนไข → ช่องเงื่อนไข
2. บล็อกคำสั่ง → ช่องคำสั่ง
3. บล็อกที่วางแล้ว → ลากออกเพื่อลบ
4. บล็อกที่วางแล้ว → ลากไปแทนที่ช่องอื่น
5. ในเกมที่ 3 → ลากสลับลำดับ If / Else If ได้

พฤติกรรมที่ต้องมี:

- ขณะลาก ช่องที่วางได้ต้องเรืองแสง
- วางถูกช่องแล้ว snap เข้า slot
- วางผิดช่องแล้วเด้งกลับ
- ถ้าช่องมีบล็อกเดิม ให้แทนที่หรือสลับตามกติกา
- หลังวางแล้ว rule preview ต้องอัปเดตทันที
- ก่อนเริ่มสายพาน ถ้ากฎไม่ครบต้องเตือน

โครงสร้าง HTML ตัวอย่าง:

```html
<div class="rule-row" data-rule-type="if">
  <div class="rule-label">ถ้า</div>
  <div class="drop-slot condition-slot" data-accept="condition"></div>
  <div class="rule-arrow">→</div>
  <div class="drop-slot action-slot" data-accept="action"></div>
</div>
```

แนะนำให้ใช้ HTML/CSS สำหรับบล็อกคำสั่งและ rule panel ส่วน Phaser ใช้สำหรับสายพานและ animation วิธีนี้จะทำให้ Drag & Drop เสถียรกว่าการลากวัตถุ UI ทั้งหมดใน Phaser

โครงสร้าง JS ที่แนะนำ:

```javascript

  constructor({ blockContainer, rulePanel, onRulesChanged }) {
    this.blockContainer = blockContainer;
    this.rulePanel = rulePanel;
    this.onRulesChanged = onRulesChanged;
    this.rules = [];
  }

  createDraggableBlock(blockData) {
    // สร้าง block ด้วยข้อมูล type: condition/action
    // ผูก dragstart, dragend และ accessibility fallback
  }

  registerDropSlot(slotElement) {
    // slot ต้องมี data-accept="condition" หรือ "action"
    // ขณะ dragover ให้ highlight ถ้ารับ block ได้
  }

  handleDrop(blockData, slotElement) {
    // ตรวจว่า slot รับ block ประเภทนี้หรือไม่
    // ถ้าถูกต้อง ให้ snap block เข้า slot
    // อัปเดต rule model
    // เรียก onRulesChanged(this.getRules())
  }

  getRules() {
    // คืนค่า rules array สำหรับ engine
    return this.rules;
  }
}
```

### 2.8 Responsive Web Design ต้องแก้จริง

**ปัญหา:**  
เมื่อย่อหน้าจอ เนื้อหาบางส่วนจมกรอบหรือหายไป แสดงว่า container บางส่วนใช้ `overflow: hidden` หรือ fix height ไม่เหมาะสม

**แนวทางแก้ไข:**

1. ห้ามใช้ `overflow: hidden` กับส่วนที่มีเนื้อหาสำคัญ เช่น sidebar, toolbox, program panel
2. ใช้ `overflow-y: auto` กับ sidebar เมื่อเนื้อหาเกิน
3. ใช้ `overflow-x: auto` กับ program builder เมื่อแถวกฎยาว
4. Canvas wrapper สามารถใช้ `overflow: hidden` ได้ แต่ต้องไม่ซ่อน UI
5. ใช้ CSS Grid และ media query เพื่อเปลี่ยน layout เป็นแนวตั้งบนจอเล็ก
6. Phaser canvas ใช้ `Scale.FIT`

CSS ที่เสนอ:

```css
.smart-farm-shell {
  width: min(1180px, calc(100vw - 24px));
  margin: 0 auto;
  display: grid;
  grid-template-columns: minmax(0, 1fr) 340px;
  grid-template-areas:
    "mission toolbox"
    "board toolbox"
    "program program";
  gap: 16px;
  align-items: start;
}

.mission-card {
  grid-area: mission;
}

.board-panel {
  grid-area: board;
  min-width: 0;
  overflow: hidden;
}

.toolbox-panel {
  grid-area: toolbox;
  max-height: 680px;
  overflow-y: auto;
}

.program-panel {
  grid-area: program;
  overflow-x: auto;
  min-width: 0;
}
```

## 3. โครงสร้างเกมใหม่แบบละเอียด

## 3.1 เกมที่ 1: โรงคัดแยกผลผลิตของพี่สมปอง

**Stage ID:** 7  
**ตรรกะหลัก:** If  
**แนวคิด:** หากพบวัตถุที่เข้าเงื่อนไข ให้ส่งไปดำเนินการพิเศษ หากไม่เข้าเงื่อนไขให้ปล่อยผ่านอัตโนมัติ  
**โครงสร้างกฎ:**

```text
ถ้า [เงื่อนไข] → [คำสั่ง]
ไม่เข้าเงื่อนไข → ปล่อยผ่านอัตโนมัติ
```

### ด่าน 1-1: แครอทเปื้อนโคลน

**เป้าหมาย:** ส่งแครอทเปื้อนโคลนเข้าเครื่องล้าง  
**เงื่อนไข:** แครอทเปื้อนโคลน  
**คำสั่ง:** ส่งเข้าเครื่องล้าง  
**ตัวหลอก:** แครอทสะอาด  
**ความยาก:** ง่ายที่สุด ใช้ภาพต่างกันชัดเจน

กฎที่คาดหวัง:

```text
ถ้า [แครอทเปื้อนโคลน] → [ส่งเข้าเครื่องล้าง]
```

Animation:

- แครอทวิ่งบนสายพาน
- เครื่องสแกนส่องคราบโคลน
- ประตูเปิดไปเครื่องล้าง
- เครื่องล้างพ่นน้ำ
- แครอทสะอาดเด้งลงตะกร้า

### ด่าน 1-2: ไข่ร้าวต้องแยกออก

**เป้าหมาย:** ส่งไข่ร้าวเข้าถาดคัดทิ้ง  
**เงื่อนไข:** ไข่มีรอยร้าว  
**คำสั่ง:** ส่งเข้าถาดคัดทิ้ง  
**ตัวหลอก:** ไข่ดีที่มีสีและขนาดใกล้เคียง  
**ความยาก:** ต้องสังเกตรอยแตก

กฎที่คาดหวัง:

```text
ถ้า [ไข่ร้าว] → [ส่งเข้าถาดคัดทิ้ง]
```

Animation:

- ไข่กลิ้งบนสายพานอย่างระมัดระวัง
- เครื่องสแกนแสดงรอยแตก
- ไข่ร้าวถูกวางลงถาดคัดทิ้ง
- หากผิด ไข่ร้าวลงกล่องขายแล้วกล่องสั่นเตือน

### ด่าน 1-3: ผักมีหนอนต้องตรวจพิเศษ

**เป้าหมาย:** ส่งผักที่มีหนอนไปโต๊ะตรวจพิเศษ  
**เงื่อนไข:** ผักมีหนอน  
**คำสั่ง:** ส่งไปโต๊ะตรวจพิเศษ  
**ตัวหลอก:** ผักหลายชนิดที่ไม่มีหนอน  
**ความยาก:** มีผักหลายชนิด และสายพานเร็วขึ้นเล็กน้อย

กฎที่คาดหวัง:

```text
ถ้า [ผักมีหนอน] → [ส่งไปโต๊ะตรวจพิเศษ]
```

Animation:

- ผักหลายชนิดวิ่งบนสายพาน
- หนอนขยับเล็ก ๆ บนผักบางชิ้น
- โต๊ะตรวจมีแว่นขยาย
- หากผิด ผักมีหนอนลงตะกร้าแล้วหนอนโผล่มาตลก ๆ

## 3.2 เกมที่ 2: หุ่นยนต์เลี้ยงสัตว์อัจฉริยะ

**Stage ID:** 8  
**ตรรกะหลัก:** If / Else  
**แนวคิด:** หากเข้าเงื่อนไขให้ทำคำสั่ง A หากไม่เข้าเงื่อนไขให้ทำคำสั่ง B  
**โครงสร้างกฎ:**

```text
ถ้า [เงื่อนไข] → [คำสั่ง A]
นอกเหนือจากนี้ → [คำสั่ง B]
```

### ด่าน 2-1: วัวกินหญ้า สัตว์อื่นกินรำ

**เป้าหมาย:** แจกอาหารให้สัตว์ถูกชนิด  
**เงื่อนไข:** เป็นวัว  
**คำสั่ง A:** จ่ายหญ้าเนเปียร์  
**คำสั่ง B:** จ่ายรำข้าว  
**ตัวหลอก:** ไก่ เป็ด หมู

กฎที่คาดหวัง:

```text
ถ้า [เป็นวัว] → [จ่ายหญ้าเนเปียร์]
นอกเหนือจากนี้ → [จ่ายรำข้าว]
```

Animation:

- สัตว์เดินเข้าจุดให้อาหาร
- วัวได้หญ้าแล้วมีหัวใจขึ้น
- ไก่/เป็ด/หมูได้รำแล้วดีใจ
- ถ้าวัวได้รำ วัวทำหน้างง
- ถ้าไก่ได้หญ้า ไก่ลากหญ้าไม่ไหว

### ด่าน 2-2: แกะขนยาวต้องตัดขน ที่เหลือไปกินหญ้า

**เป้าหมาย:** ตัดขนแกะที่ขนฟูมาก  
**เงื่อนไข:** แกะขนฟูมาก  
**คำสั่ง A:** ตัดขน  
**คำสั่ง B:** ปล่อยไปกินหญ้า  
**ตัวหลอก:** แกะขนสั้นและสัตว์อื่นที่เดินผ่าน

กฎที่คาดหวัง:

```text
ถ้า [แกะขนฟูมาก] → [ตัดขน]
นอกเหนือจากนี้ → [ปล่อยไปกินหญ้า]
```

Animation:

- แกะขนฟูเดินดุ๊กดิ๊ก
- เครื่องตัดขนทำงาน
- ขนแกะฟุ้งออกเป็นก้อนเมฆ
- แกะขนสั้นถูกตัดซ้ำแล้วหนาวสั่นถ้าผิด

### ด่าน 2-3: สัตว์ป่วยต้องเข้าคอกพยาบาล

**เป้าหมาย:** แยกสัตว์ป่วยไปคอกพยาบาล ที่เหลือไปคอกปกติ  
**เงื่อนไข:** สัตว์ป่วย  
**คำสั่ง A:** ส่งคอกพยาบาล  
**คำสั่ง B:** ส่งคอกปกติ  
**ตัวหลอก:** สัตว์หลากหลายชนิด ทั้งป่วยและไม่ป่วย

กฎที่คาดหวัง:

```text
ถ้า [สัตว์ป่วย] → [ส่งคอกพยาบาล]
นอกเหนือจากนี้ → [ส่งคอกปกติ]
```

Animation:

- สัตว์ป่วยมีไอคอนปรอทหรือผ้าพันแผล
- หุ่นยนต์พาเข้าคอกพยาบาล
- สัตว์ปกติไปคอกสีเขียว
- ถ้าสัตว์ป่วยเข้าคอกปกติ คอกปกติมีสัญญาณเตือน

## 3.3 เกมที่ 3: ผู้ดูแลโรงเรือนไฮเทค

**Stage ID:** 9  
**ตรรกะหลัก:** If / Else If / Else  
**แนวคิด:** ตรวจเงื่อนไขตามลำดับ หากไม่เข้าเงื่อนไขแรก ให้ตรวจเงื่อนไขถัดไป หากไม่เข้าเงื่อนไขใดเลย ให้ทำคำสั่ง Else  
**โครงสร้างกฎ:**

```text
ถ้า [เงื่อนไข A] → [คำสั่ง A]
นอกเหนือจากนี้ถ้า [เงื่อนไข B] → [คำสั่ง B]
นอกเหนือจากนี้ → [คำสั่ง C]
```

### ด่าน 3-1: ควบคุมความชื้นในโรงเรือน

**เป้าหมาย:** ปรับความชื้นให้เหมาะสม  
**เงื่อนไข A:** ความชื้นต่ำกว่า 30%  
**คำสั่ง A:** เปิดสปริงเกอร์  
**เงื่อนไข B:** ความชื้นสูงกว่า 80%  
**คำสั่ง B:** เปิดพัดลมลดความชื้น  
**Else:** ปิดระบบ ประหยัดพลังงาน

กฎที่คาดหวัง:

```text
ถ้า [ความชื้น < 30%] → [เปิดสปริงเกอร์]
นอกเหนือจากนี้ถ้า [ความชื้น > 80%] → [เปิดพัดลมลดความชื้น]
นอกเหนือจากนี้ → [ปิดระบบ]
```

Animation:

- กระถางผ่านเซ็นเซอร์ความชื้น
- ตัวเลขความชื้นแสดงเหนือกระถาง
- สปริงเกอร์ทำงานเมื่อชื้นต่ำ
- พัดลมทำงานเมื่อชื้นสูง
- ระบบปิดเมื่อค่าปกติ

### ด่าน 3-2: ควบคุมหลังคาโรงเรือนเมลอน

**เป้าหมาย:** ป้องกันฝนและจัดการแสงให้เหมาะกับเมลอน  
**เงื่อนไข A:** ฝนตก  
**คำสั่ง A:** กางหลังคา  
**เงื่อนไข B:** แดดแรงมาก  
**คำสั่ง B:** กางสแลนพรางแสง  
**Else:** เปิดรับแสงธรรมชาติ

กฎที่คาดหวัง:

```text
ถ้า [ฝนตก] → [กางหลังคา]
นอกเหนือจากนี้ถ้า [แดดแรงมาก] → [กางสแลนพรางแสง]
นอกเหนือจากนี้ → [เปิดรับแสงธรรมชาติ]
```

Animation:

- เมฆฝนหรือแสงแดดแสดงบนโรงเรือน
- หลังคากางเมื่อฝนตก
- สแลนพรางแสงกางเมื่อแดดแรง
- เปิดหลังคาเมื่ออากาศปกติ

### ด่าน 3-3: ควบคุมอุณหภูมิโรงเรือนเห็ด

**เป้าหมาย:** ควบคุมอุณหภูมิให้เหมาะสมกับเห็ด  
**เงื่อนไข A:** อุณหภูมิสูงกว่า 35°C  
**คำสั่ง A:** เปิดพัดลม  
**เงื่อนไข B:** อุณหภูมิต่ำกว่า 15°C  
**คำสั่ง B:** เปิดหลอดไฟให้ความอบอุ่น  
**Else:** ปิดระบบทั้งหมด

กฎที่คาดหวัง:

```text
ถ้า [อุณหภูมิ > 35°C] → [เปิดพัดลม]
นอกเหนือจากนี้ถ้า [อุณหภูมิ < 15°C] → [เปิดหลอดไฟอุ่น]
นอกเหนือจากนี้ → [ปิดระบบทั้งหมด]
```

Animation:

- เทอร์โมมิเตอร์แสดงตัวเลข
- ไอร้อนเมื่ออุณหภูมิสูง
- พัดลมหมุน
- ไอเย็นเมื่ออุณหภูมิต่ำ
- หลอดไฟอุ่นเปิด
- ระบบปิดเมื่ออุณหภูมิปกติ

## 4. ระบบให้ดาวและสรุปผล

หลังจบแต่ละ stage ให้แสดงผลดังนี้

- จำนวนรายการทั้งหมด
- จำนวนที่ถูกต้อง
- จำนวนที่ผิดพลาด
- จำนวนสินค้าหรือเหตุการณ์เสียหาย
- ดาวที่ได้รับ
- ปุ่มเล่นซ้ำ
- ปุ่มกลับหน้าเลือกด่าน

เกณฑ์ดาว:

| ดาว | เงื่อนไข |
|---|---|
| 3 ดาว | ถูกต้องอย่างน้อย 90% และเสียหายไม่เกิน 1 รายการ |
| 2 ดาว | ถูกต้องอย่างน้อย 75% |
| 1 ดาว | ถูกต้องอย่างน้อย 60% |
| 0 ดาว | ต่ำกว่า 60% หรือกฎไม่ครบ |

การส่งผลคะแนน:

```javascript
window.sendResult(window.STAGE_ID, stars, duration, attempts);
```

หมายเหตุ: ถ้าในหนึ่ง stage มี 3 ด่านย่อย ให้คิดดาวจากผลรวมของด่านย่อยทั้งหมด หรือกำหนดให้ต้องผ่านด่านย่อยครบก่อนส่งผลรวมครั้งเดียว

```javascript
window.FarmMissions.conveyorLogic = function(config) {
  // สร้างเกมสายพานตรรกะจาก level config
```

หน้าที่ของ Base Engine:

1. โหลด asset หรือ fallback icon

4. สร้างจุดสแกน
5. spawn item ตาม queue
6. snap item เข้าจุดสแกน
7. อ่าน rules จาก DragDropManager
8. evaluate condition
9. route item ไปปลายทาง
10. play animation ถูก/ผิด
11. นับถูก/ผิด/เสียหาย
12. คำนวณดาว
13. ส่งผลผ่าน `window.sendResult()`

```javascript
class DragDropManager {

  handleDrop(blockData, slotElement) {}
  removeBlock(slotElement) {}
  swapRows(fromRow, toRow) {}
  getRules() {}
  validateRules() {}
}
```

## 7. Prompt สำหรับสั่ง Codex พัฒนา

```text
ปรับปรุงบทเรียนที่ 3 Smart Farm Manager ตามสเปก v2 ต่อไปนี้

1. เปลี่ยนชื่อบทเรียนที่ 3 ให้สอดคล้องทั้งระบบเป็น “ผู้จัดการฟาร์มอัจฉริยะ / Smart Farm Manager”
2. Stage 7 ต้องเป็นเกม “โรงคัดแยกผลผลิตของพี่สมปอง” และใช้ตรรกะ If เท่านั้น
   - 7-1 แครอทเปื้อนโคลน
   - 7-2 ไข่ร้าวต้องแยกออก
   - 7-3 ผักมีหนอนต้องตรวจพิเศษ
3. Stage 8 ต้องเป็นเกม “หุ่นยนต์เลี้ยงสัตว์อัจฉริยะ” และใช้ตรรกะ If / Else เท่านั้น
   - 8-1 วัวกินหญ้า สัตว์อื่นกินรำ
   - 8-2 แกะขนยาวต้องตัดขน ที่เหลือไปกินหญ้า
   - 8-3 สัตว์ป่วยต้องเข้าคอกพยาบาล ที่เหลือไปคอกปกติ
4. Stage 9 ต้องเป็นเกม “ผู้ดูแลโรงเรือนไฮเทค” และใช้ตรรกะ If / Else If / Else เท่านั้น
   - 9-1 ควบคุมความชื้น
   - 9-2 ควบคุมหลังคาโรงเรือนเมลอน
   - 9-3 ควบคุมอุณหภูมิโรงเรือนเห็ด
5. สร้างหรือปรับ Base Engine เป็น FarmMissions.conveyorLogic(config)
6. ใช้ Level Config ในการกำหนด items, conditions, actions, expectedLogic, assets และ scoring ห้าม hardcode logic ใน scene
7. แก้การแสดงผลวัตถุเมื่อเข้าเครื่องสแกน โดย snap item ไปที่ SCAN_POINT ก่อน play scan animation และก่อน evaluateRules เสมอ
8. ทุก item ต้องมี fallbackIcon และ asset metadata เช่น key, path, width, height, description, futureSpriteSheet เพื่อรองรับการเปลี่ยนเป็นภาพกราฟิกในอนาคต
9. ห้ามผูก logic กับ emoji หรือ icon โดยตรง ให้ logic อ่านจาก props เท่านั้น
10. ตัดระบบคะแนนตัวเลข Combo เหรียญ และร้านค้าออกในรอบนี้ ใช้ดาวเป็นคะแนนหลักเท่านั้น
11. ทำ Drag & Drop ให้ใช้งานได้จริง โดยแยก DragDropManager เป็น module สำหรับบล็อกเงื่อนไขและบล็อกคำสั่ง
12. ผู้เรียนต้องลากบล็อกเงื่อนไขและคำสั่งไปวางในแผงสร้างโปรแกรมได้โดยตรง
13. ขณะลาก ช่องที่วางได้ต้อง highlight วางถูกต้อง snap วางผิดต้องเด้งกลับ
14. ใน Stage 9 ต้องสลับลำดับ If / Else If ได้
15. ย้ายคำแนะนำจากแถบด้านล่างไปเป็น Mission Card เหนือพื้นที่สายพาน
16. ปรับ UI ให้ปุ่มควบคุมแยกจากบล็อกคำสั่งชัดเจน
17. แก้ Responsive Web Design ไม่ให้เนื้อหาจมกรอบหรือหายเมื่อย่อจอ
18. ใช้ CSS Grid layout และ Phaser Scale.FIT เพื่อไม่ให้ canvas ล้นกรอบ
19. หลังจบ stage ให้แสดงดาว จำนวนถูกผิด และจำนวนเสียหาย แล้วเรียก window.sendResult(window.STAGE_ID, stars, duration, attempts)
20. หลีกเลี่ยงการเปลี่ยน database ถ้าไม่จำเป็น
```

## 9. ข้อควรระวัง

1. อย่ากลับไปใช้ชื่อ “เครื่องรดน้ำอัจฉริยะ” เพราะไม่ตรงกับเกมใหม่
2. อย่าให้แต่ละเกมปนโครงสร้างตรรกะหลายแบบในรอบนี้
3. อย่าผูก logic กับ emoji
4. อย่าใส่คะแนน คอมโบ เหรียญกลับมาในเวอร์ชันนี้
5. อย่าทำ Drag & Drop แบบครึ่ง ๆ กลาง ๆ
6. อย่าให้ Mission หรือคำแนะนำไปอยู่เป็นแถบล่างที่ทำให้ layout ไม่สมดุล
7. อย่าใช้ `overflow: hidden` จนเนื้อหาสำคัญหาย
8. อย่าให้ Phaser canvas ล้น container
9. อย่า hardcode ด่านจนแก้ config ยาก
10. อย่าเปลี่ยนฐานข้อมูลถ้าไม่จำเป็น

## 10. สรุป

แผนปรับปรุง **Smart Farm Manager v2** มุ่งแก้ปัญหาที่พบจากการทดสอบจริง โดยจัดโครงสร้างบทเรียนใหม่ให้ชัดเจนกว่าเดิม

- เกมที่ 1 = If
- เกมที่ 2 = If / Else
- เกมที่ 3 = If / Else If / Else

พร้อมปรับปรุงระบบหลัก ได้แก่

- ชื่อเกมให้สอดคล้องทั้งระบบ
- Drag & Drop ที่ต้องใช้งานได้จริง
- การจัดตำแหน่งวัตถุเข้าเครื่องสแกน
- โครงสร้าง asset รองรับภาพกราฟิกในอนาคต
- ตัดระบบคะแนนที่ไม่จำเป็น
- ย้าย Mission Card ไปตำแหน่งที่สมดุลกว่า
- แก้ Responsive Web Design ไม่ให้เนื้อหาจมกรอบ
