# แผนการพัฒนาเกมบทเรียนที่ 3
# Water Hero: ศึกป้องกันสวนอัจฉริยะ

**บทเรียน:** บทที่ 3 เครื่องรดน้ำอัจฉริยะ (Condition)  
**แนวเกมใหม่:** Action Puzzle + Tower Defense + Condition Cards  
**กลุ่มเป้าหมาย:** นักเรียนชั้นประถมศึกษาปีที่ 4  
**เป้าหมายหลัก:** เปลี่ยนบทเรียนจากแบบฝึกจับคู่คำตอบให้เป็นเกมที่ผู้เรียนรู้สึกสนุก ตื่นเต้น มีภารกิจ มีตัวละคร มีสถานการณ์คุกคาม และต้องใช้เงื่อนไข If-Then-Else เพื่อเอาชนะเกม  
**ชื่อเกมที่เสนอ:** Water Hero: ศึกป้องกันสวนอัจฉริยะ  
**ชื่อไทยทางการในระบบ:** ภารกิจฮีโร่หยดน้ำ ป้องกันสวนด้วยเงื่อนไข

## 1. บทนำและเหตุผลในการพัฒนา

บทเรียนที่ 3 เดิมมีเป้าหมายทางการเรียนรู้ที่ดี คือการสอนแนวคิดเรื่อง **เงื่อนไข (Condition)** ผ่านสถานการณ์ “เครื่องรดน้ำอัจฉริยะ” โดยให้ผู้เรียนเข้าใจหลักการ `ถ้า...แล้ว...มิฉะนั้น` หรือ `If-Then-Else` เช่น ถ้าดินแห้งให้รดน้ำ ถ้าฝนตกให้หยุดรดน้ำ และถ้าถังน้ำหมดให้แจ้งเตือนเติมน้ำ

อย่างไรก็ตาม เมื่อนำมาพิจารณาในมุมของ “เกม” พบว่ารูปแบบเดิมยังไม่สนุกเพียงพอ เพราะยังมีลักษณะเป็นแบบฝึกมากกว่าเกม ผู้เรียนต้องอ่านข้อความสถานการณ์ แล้วลากคำตอบไปวางให้ตรงกับสถานการณ์ จากนั้นระบบตรวจว่าถูกหรือผิด กิจกรรมลักษณะนี้แม้ช่วยให้ฝึกเนื้อหาได้ แต่ยังขาดองค์ประกอบสำคัญของเกม เช่น ความตื่นเต้น เป้าหมายที่เร้าใจ ความกดดันจากเวลา ตัวละคร เหตุการณ์ไม่คาดคิด การเอาชนะศัตรู การลองผิดลองถูก และความรู้สึกอยากเล่นซ้ำเพื่อให้ได้คะแนนดีขึ้น

ดังนั้น แผนการพัฒนาฉบับนี้จึงเสนอให้ปรับบทเรียนที่ 3 ใหม่ทั้งหมดในเชิงประสบการณ์การเล่น โดยยังคงเนื้อหาวิชาการเรื่อง If-Then-Else ไว้ แต่เปลี่ยนวิธีนำเสนอให้กลายเป็นเกมแนว **Action Puzzle + Tower Defense + Condition Cards** ที่ผู้เรียนต้องควบคุมตัวละคร เก็บการ์ดเงื่อนไข ติดตั้งกฎให้ระบบรดน้ำ และป้องกันสวนจากภัยคุกคาม เช่น แดดเผา ฝนถล่ม ถังน้ำรั่ว และดินแห้ง

แนวคิดสำคัญของเกมใหม่คือ

> ผู้เรียนไม่ได้แค่ “ตอบเงื่อนไข” แต่ต้อง “ใช้เงื่อนไขเป็นเครื่องมือเอาชนะสถานการณ์ในเกม”

## 2. ปัญหาของรูปแบบเดิม

### 2.1 เกมเดิมยังมีลักษณะเหมือนแบบฝึก

กิจกรรมเดิมของบทเรียนที่ 3 ใช้การลากคำสั่งไปวางตามสถานการณ์ เช่น ดินแห้งให้รดน้ำ ดินชื้นให้หยุดรดน้ำ หรือฝนตกให้หยุดรดน้ำ แม้มีการใช้ภาพและการโต้ตอบ แต่ความรู้สึกของผู้เรียนยังคล้ายกับการทำใบงานออนไลน์มากกว่าเล่นเกม

ปัญหาหลักคือผู้เรียนไม่ได้มีบทบาทชัดเจน ไม่ได้มีภัยคุกคาม ไม่ได้มีเป้าหมายที่เร้าใจ และไม่ได้มีความรู้สึกว่าต้องรีบคิดหรือวางแผนเพื่อให้ฟาร์มรอด

### 2.2 ขาดแรงจูงใจในการเล่นซ้ำ

เกมที่ดีควรทำให้ผู้เรียนอยากลองใหม่ เช่น อยากได้ 3 ดาว อยากผ่านโดยไม่พลาด อยากเอาชนะบอส หรืออยากทำคะแนนให้ดีกว่าเดิม แต่รูปแบบเดิมเมื่อผู้เรียนตอบถูกแล้วก็จบ ไม่มีแรงกระตุ้นให้กลับมาเล่นซ้ำ

### 2.3 ผลลัพธ์ยังไม่เร้าอารมณ์

เมื่อผู้เรียนตอบผิด ระบบควรทำให้เห็นผลของความผิดพลาด เช่น พืชเหี่ยว น้ำท่วม ถังน้ำหมด หรือระบบพัง แต่รูปแบบเดิมมักเน้นการแจ้งว่าคำตอบผิดมากกว่าการจำลองผลลัพธ์ที่มีความหมาย

### 2.4 ไม่เห็นภาพว่าเงื่อนไขคือ “ระบบควบคุม”

การเรียน If-Then-Else ควรทำให้ผู้เรียนเห็นว่าเงื่อนไขเป็นกฎที่ควบคุมระบบให้ตัดสินใจได้เอง แต่รูปแบบเดิมเป็นการเลือกคำตอบเฉพาะหน้า จึงยังไม่ทำให้ผู้เรียนรู้สึกว่าตนเองกำลังสร้างระบบอัจฉริยะจริง ๆ

### 2.5 ขาดความตื่นเต้นตามวัย

นักเรียนระดับประถมศึกษาชอบกิจกรรมที่มีตัวละคร ภารกิจ ความท้าทาย การเคลื่อนไหว เสียง เอฟเฟกต์ และผลลัพธ์ทันที หากเกมนิ่งเกินไปหรือใช้เพียงการอ่านและลากวาง ผู้เรียนอาจหมดความสนใจเร็ว

## 3. แนวคิดเกมใหม่

## Water Hero: ศึกป้องกันสวนอัจฉริยะ

เกมใหม่นี้ให้ผู้เรียนรับบทเป็น **ฮีโร่หยดน้ำ** หรือ **หุ่นยนต์ดูแลสวน** ที่ต้องป้องกันสวนผักจากภัยคุกคามต่าง ๆ โดยใช้ความรู้เรื่องเงื่อนไข If-Then-Else

ผู้เรียนจะต้องเดินหรือเคลื่อนตัวละครในสนามเกม เก็บการ์ดเงื่อนไขและการ์ดคำสั่ง แล้วนำไปติดตั้งใน “ตู้ควบคุมระบบน้ำอัจฉริยะ” จากนั้นกดเริ่ม Wave เพื่อให้ระบบทำงาน หากผู้เรียนวางเงื่อนไขถูกต้อง สวนจะรอด พืชจะฟื้น และผู้เรียนจะได้คะแนน แต่หากวางผิด ระบบอาจรดน้ำผิดเวลา ทำให้พืชเหี่ยว น้ำท่วม หรือถังน้ำหมด

## 4. เป้าหมายการเรียนรู้

เมื่อผู้เรียนเล่นบทเรียนที่ 3 รูปแบบใหม่นี้จบ ควรสามารถ

1. อธิบายความหมายของเงื่อนไขแบบ `ถ้า...แล้ว...`
2. เข้าใจการทำงานของ `If`
3. เข้าใจการทำงานของ `If-Else`
4. เข้าใจการทำงานของ `If-Else If-Else`
5. เข้าใจว่าลำดับของเงื่อนไขมีผลต่อผลลัพธ์
6. เลือกเงื่อนไขที่เหมาะสมกับสถานการณ์ได้
7. เลือกคำสั่งที่เหมาะสมกับเงื่อนไขได้
8. ทดสอบผลลัพธ์จากกฎที่ตนเองสร้าง
9. แก้ไขกฎเมื่อผลลัพธ์ไม่ถูกต้อง
10. อธิบายเหตุผลของการจัดลำดับเงื่อนไขได้
11. เชื่อมโยงแนวคิด If-Then-Else กับสถานการณ์ในชีวิตจริงได้

## 5. เป้าหมายด้านความเป็นเกม

เกมใหม่ต้องทำให้ผู้เรียนรู้สึกว่า “กำลังเล่นเกม” ไม่ใช่ “กำลังทำแบบฝึกหัด” โดยต้องมีองค์ประกอบต่อไปนี้

1. มีตัวละครหลักให้ควบคุมหรือมีตัวแทนผู้เล่นชัดเจน
2. มีภารกิจเร่งด่วน เช่น ป้องกันสวนก่อนพืชเหี่ยว
3. มีภัยคุกคาม เช่น แดดแรง ฝนตก ถังน้ำรั่ว
4. มีระบบ Wave หรือรอบการโจมตี
5. มีพลังชีวิตของสวนหรือพืช
6. มีคะแนน ดาว เหรียญ หรือ Badge
7. มีการเก็บการ์ดหรือเลือกบล็อกเงื่อนไข
8. มีผลลัพธ์เชิงภาพและเสียง
9. มีความท้าทายเพิ่มขึ้นตามด่าน
10. มีด่านบอสเพื่อสร้างความรู้สึกสำเร็จ
11. มีระบบ Combo หรือ Bonus เพื่อกระตุ้นให้เล่นดีขึ้น
12. มี feedback ที่อธิบายเหตุผลของความผิดพลาด

## 6. รูปแบบการเล่นโดยรวม

### 6.1 วงจรการเล่นหลัก

```text
เริ่มภารกิจ
→ อ่านสถานการณ์สวน
→ ควบคุมตัวละครหรือเลือกการ์ด
→ เก็บ/เลือกการ์ดเงื่อนไขและคำสั่ง
→ ติดตั้งกฎในตู้ควบคุม
→ เริ่ม Wave
→ ระบบรดน้ำทำงานตามกฎ
→ แสดงผลลัพธ์
→ ได้คะแนนหรือเกิดความเสียหาย
→ ปรับปรุงกฎ
→ ผ่านด่านหรือเล่นซ้ำ
```

### 6.2 แกนการตัดสินใจ

ผู้เรียนต้องตัดสินใจ 3 เรื่องหลัก

1. **จะใช้เงื่อนไขอะไร**
   - ดินแห้ง
   - ฝนตก
   - ถังน้ำหมด
   - ดินชื้น

2. **จะสั่งให้ระบบทำอะไร**
   - รดน้ำ
   - หยุดรดน้ำ
   - แจ้งเติมน้ำ
   - สังเกตต่อ

3. **จะเรียงเงื่อนไขลำดับใด**
   - ตรวจฝนก่อนดินแห้งหรือไม่
   - ตรวจถังน้ำก่อนรดน้ำหรือไม่
   - ใช้ Else เป็นคำสั่งสำรองหรือไม่

## 7. ธีมและเนื้อเรื่อง

### 7.1 เรื่องย่อ

โรงเรียนมีสวนผักอัจฉริยะที่นักเรียนช่วยกันดูแล แต่ช่วงนี้สภาพอากาศแปรปรวนมาก บางวันแดดแรงจนดินแห้ง บางวันฝนตกหนัก บางครั้งถังน้ำก็รั่วจนน้ำหมด ทำให้ระบบรดน้ำเดิมตัดสินใจผิดพลาด

ผู้เรียนจึงต้องรับบทเป็น **ฮีโร่หยดน้ำ** ผู้เชี่ยวชาญด้านเงื่อนไข เพื่อสร้างระบบรดน้ำอัจฉริยะให้สามารถตัดสินใจได้เองอย่างถูกต้อง

### 7.2 ตัวละคร

#### ตัวละครหลัก

**น้องหยดน้ำ (Water Hero)**  
ตัวละครหยดน้ำสีฟ้าขนาดเล็ก เคลื่อนไหวได้ มีท่าทางน่ารัก เช่น กระโดด ดีใจ ตกใจ และเหนื่อย

#### ตัวละครเสริม

**ครูโรบอทสวนผัก**  
เป็นตัวละครช่วยสอน อธิบายภารกิจและให้คำใบ้

#### ศัตรู/ภัยคุกคาม

1. **แดดจอมเผา**
   - ทำให้ดินแห้งและพืชเหี่ยว

2. **เมฆดำจอมป่วน**
   - ทำให้ฝนตกหนักและเกิดน้ำท่วมถ้ารดน้ำซ้ำ

3. **ถังน้ำรั่ว**
   - ทำให้น้ำหมด ถ้าระบบไม่ตรวจถังน้ำก่อนจะสั่งรดน้ำผิด

4. **บั๊กเงื่อนไข**
   - เป็นสัญลักษณ์ข้อผิดพลาดเมื่อผู้เรียนจัดลำดับกฎผิด

## 8. องค์ประกอบของเกม

### 8.1 สนามเกม

สนามเกมควรเป็นสวนผัก 2D มุมมองด้านบนหรือกึ่งด้านข้าง มีองค์ประกอบดังนี้

1. แปลงผัก 2–5 แปลง
2. หัวรดน้ำของแต่ละแปลง
3. ถังน้ำกลางสวน
4. ตู้ควบคุมระบบน้ำ
5. การ์ดเงื่อนไขที่ให้ผู้เรียนเก็บหรือเลือก
6. ตัวละครน้องหยดน้ำ
7. แถบพลังชีวิตของสวน
8. แถบเวลาเตรียมระบบก่อน Wave
9. ปุ่มเริ่ม Wave
10. พื้นที่แสดงผลลัพธ์หลังทดสอบ

### 8.2 การ์ดเงื่อนไข

การ์ดเงื่อนไขควรมีสีและไอคอนชัดเจน เช่น

| การ์ด | สี | ไอคอน | ความหมาย |
|---|---|---|---|
| ดินแห้ง | น้ำตาล | 🟫 | ตรวจว่าดินแห้งหรือไม่ |
| ดินชื้น | เขียว | 🌱 | ตรวจว่าดินชื้นพอดีหรือไม่ |
| ฝนตก | น้ำเงินเข้ม | ⛈️ | ตรวจว่าฝนกำลังตกหรือไม่ |
| ถังน้ำหมด | แดง | 🪣 | ตรวจว่าน้ำในถังหมดหรือไม่ |
| มิฉะนั้น | เทา | 🔁 | กรณีที่ไม่เข้าเงื่อนไขใดเลย |

### 8.3 การ์ดคำสั่ง

| การ์ด | สี | ไอคอน | ความหมาย |
|---|---|---|---|
| รดน้ำ | ฟ้า | 💧 | เปิดน้ำให้แปลงผัก |
| หยุดรดน้ำ | เหลือง | ✋ | ปิดน้ำหรือไม่เปิดน้ำเพิ่ม |
| แจ้งเติมน้ำ | ส้ม | 🚨 | แจ้งให้เติมน้ำในถัง |
| สังเกตต่อ | ม่วง | 👀 | ยังไม่ทำอะไร รอดูสถานการณ์ |

### 8.4 ตู้ควบคุมระบบน้ำ

ตู้ควบคุมเป็นพื้นที่ให้ผู้เรียนวางการ์ดเป็นกฎ เช่น

```text
ถ้า [________] แล้ว [________]
มิฉะนั้นถ้า [________] แล้ว [________]
มิฉะนั้น [________]
```

ในด่านต้นอาจมีช่องน้อย ด่านท้ายมีช่องมากขึ้น

- แปลง B: ดินชื้น พืชยังสดชื่น

แดดจอมเผากำลังทำให้พืชแห้งลงเรื่อย ๆ ผู้เรียนต้องตั้งระบบรดน้ำให้ทันก่อนพืชเหี่ยวหมด

### 9.1.4 วิธีเล่น

1. ผู้เรียนเห็นสวน 2 แปลง
2. ระบบแสดงเวลาเตรียมตัว เช่น 30 วินาที
3. ผู้เรียนควบคุมน้องหยดน้ำเดินไปเก็บการ์ด
4. ผู้เรียนนำการ์ดไปติดตั้งในตู้ควบคุม
5. ผู้เรียนกด “เริ่ม Wave แดดแรง”
6. ระบบตรวจแต่ละแปลงตามกฎ
7. ถ้ากฎถูก แปลง A ได้รับน้ำ แปลง B ไม่ถูกรดน้ำเพิ่ม
8. ถ้ากฎผิด พืชเหี่ยวหรือน้ำล้น
9. ผู้เรียนผ่านด่านเมื่อสวนปลอดภัย

### 9.1.5 ความสนุกที่เพิ่มเข้ามา

- มีตัวจับเวลา
- พืชค่อย ๆ เหี่ยวถ้าช้า
- มีเสียงเตือนแดดแรง
- เมื่อรดน้ำถูก พืชเด้งฟื้นและมีเอฟเฟกต์ประกาย
- มีคำว่า “Water Save!” เมื่อดูแลถูก
- ถ้าผ่านโดยไม่พลาด ได้ 3 ดาว

### 9.1.6 เงื่อนไขผ่านด่าน

- แปลงดินแห้งได้รับน้ำ
- แปลงดินชื้นไม่ถูกรดน้ำเพิ่ม
- สวนเหลือพลังชีวิตมากกว่า 70%

### 9.1.7 Feedback ตัวอย่าง

กรณีถูก:

```text
ยอดเยี่ยม! คุณใช้ If-Else ได้ถูกต้อง ดินแห้งจึงได้รับน้ำ ส่วนดินชื้นไม่ถูกรดน้ำเพิ่ม
```

กรณีผิด:

```text
แปลง A ดินแห้งแต่ระบบไม่ได้รดน้ำ พืชจึงเหี่ยวลง ลองตั้งกฎว่า ถ้าดินแห้ง → รดน้ำ
```

## 9.2 Stage 8: พายุฝนถล่มสวน

### 9.2.1 เป้าหมายด่าน

สอนลำดับความสำคัญของเงื่อนไข

### 9.2.2 แนวคิดหลัก

```text
ถ้าฝนตก → หยุดรดน้ำ
มิฉะนั้นถ้าดินแห้ง → รดน้ำ
มิฉะนั้น → หยุดรดน้ำ
```

### 9.2.3 ฉาก

เมฆดำกำลังเคลื่อนเข้ามาในสวน บางแปลงดินแห้ง แต่ฝนกำลังตก ถ้าระบบตรวจดินแห้งก่อนตรวจฝน ระบบจะรดน้ำซ้ำจนเกิดน้ำท่วม

### 9.2.4 สถานการณ์

- แปลง A: ฝนตกและดินแห้ง
- แปลง B: ไม่มีฝนและดินแห้ง
- แปลง C: ไม่มีฝนและดินชื้น

### 9.2.5 วิธีเล่น

1. ผู้เรียนเห็นเมฆดำค่อย ๆ เคลื่อนเข้าหาสวน
2. ผู้เรียนต้องเก็บการ์ด `ฝนตก`, `ดินแห้ง`, `รดน้ำ`, `หยุดรดน้ำ`
3. ผู้เรียนติดตั้งกฎในตู้ควบคุม
4. เกมอนุญาตให้ผู้เรียนวางกฎผิดลำดับได้
5. เมื่อกดเริ่ม Wave ฝนถล่ม ระบบจะทำงานตามลำดับกฎ
6. ถ้าตรวจฝนก่อน สวนปลอดภัย
7. ถ้าตรวจดินแห้งก่อน แปลง A น้ำท่วม
8. ผู้เรียนต้องแก้ลำดับเงื่อนไขให้ถูกต้อง

### 9.2.6 ความสนุกที่เพิ่มเข้ามา

- เมฆดำเคลื่อนที่เป็นตัวจับเวลา
- ถ้าจัดลำดับผิด น้ำจะท่วมจริงในฉาก
- มีเสียงฟ้าร้อง
- มีเอฟเฟกต์น้ำล้น
- มีคำว่า “Storm Guard!” เมื่อป้องกันฝนสำเร็จ
- มี Combo เมื่อดูแลถูกทุกแปลง

### 9.2.7 เงื่อนไขผ่านด่าน

- แปลงฝนตกไม่ถูกรดน้ำเพิ่ม
- แปลงไม่มีฝนและดินแห้งได้รับน้ำ
- แปลงดินชื้นไม่ถูกรดน้ำ
- เงื่อนไขฝนตกถูกจัดไว้ก่อนดินแห้ง

### 9.2.8 Feedback ตัวอย่าง

กรณีถูก:

```text

```

### 9.3.3 ฉาก

ด่านสุดท้ายเป็นด่านบอส เมฆดำจอมป่วนร่วมมือกับถังน้ำรั่วโจมตีสวน ผู้เรียนต้องสร้างระบบรดน้ำอัจฉริยะที่รับมือได้ทุกสถานการณ์

### 9.3.4 ระบบ Boss Wave

ด่านนี้มี 3 Wave

#### Wave 1: แดดแรง

- ดินแห้งหลายแปลง
- ต้องรดน้ำให้ถูกแปลง

#### Wave 2: ฝนถล่ม

- ฝนตกบางแปลง
- ต้องหยุดรดน้ำแม้บางแปลงดินแห้ง

#### Wave 3: ถังน้ำรั่ว

- ถังน้ำหมด
- ต้องแจ้งเติมน้ำก่อนสั่งรดน้ำ

### 9.3.5 วิธีเล่น

1. ผู้เรียนเริ่มด้วยสวน 4 แปลง
2. บอสปรากฏพร้อมแถบ HP
3. ผู้เรียนเก็บหรือเลือกการ์ดเงื่อนไขหลายใบ
4. ผู้เรียนจัดลำดับกฎในตู้ควบคุม
5. เมื่อเริ่ม Wave ระบบจะประมวลผลตามกฎ
6. ถ้ากฎถูก สวนปล่อยพลังหยดน้ำโจมตีบอส
7. ถ้ากฎผิด บอสโจมตีสวน ทำให้ HP สวนลด
8. ผู้เรียนมีโอกาสแก้กฎระหว่าง Wave
9. ผ่านด่านเมื่อบอส HP เหลือ 0 และสวนยังไม่พัง

### 9.3.6 ความสนุกที่เพิ่มเข้ามา

- มีบอสพร้อมแถบ HP
- มี HP ของสวน
- มี 3 Wave
- มีเอฟเฟกต์โจมตีบอสเมื่อเงื่อนไขถูก
- มีเอฟเฟกต์บอสโจมตีเมื่อเงื่อนไขผิด
- มีเสียงเตือนถังน้ำรั่ว
- มีรางวัลหลังชนะ
- มี Badge “วิศวกรระบบน้ำอัจฉริยะ”

### 9.3.7 เงื่อนไขผ่านด่าน

- สวนยังมี HP มากกว่า 0
- บอสถูกกำจัด
- ไม่มีแปลงใดน้ำท่วมจนเสียหายหนัก
- ระบบแจ้งเติมน้ำเมื่อถังน้ำหมด
- ระบบหยุดรดน้ำเมื่อฝนตก
- ระบบรดน้ำเมื่อดินแห้งและมีน้ำพร้อม

### 9.3.8 Feedback ตัวอย่าง

```text
ถังน้ำหมดแล้ว แต่ระบบยังพยายามรดน้ำ ทำให้ช่วยพืชไม่ได้ ลองตรวจถังน้ำหมดเป็นเงื่อนไขแรก
```

กรณีไม่ตรวจฝนก่อนดินแห้ง:

```text
บอสปล่อยฝนใส่สวน แต่ระบบยังรดน้ำเพิ่มเพราะตรวจดินแห้งก่อน ทำให้แปลงน้ำท่วม
```

กรณีถูก:

```text
สุดยอด! ระบบของคุณตรวจถังน้ำ ฝน และดินแห้งได้ถูกลำดับ สวนปล่อยพลังหยดน้ำกำจัดบอสได้สำเร็จ
```

# 16. Algorithm การทำงานของเกม

## 16.1 การประเมินกฎ

```javascript
function evaluateRules(plot, rules) {
    for (const rule of rules) {
        if (rule.type === 'else') {
            return rule.action;
        }

        if (checkCondition(plot, rule.condition)) {
            return rule.action;
        }
    }

    return 'no_action';
}
```

## 16.2 การตรวจเงื่อนไข

```javascript
function checkCondition(plot, condition) {
    switch (condition) {
        case 'soil_dry':
            return plot.soil === 'dry';
        case 'soil_wet':
            return plot.soil === 'wet';
        case 'rain':
            return plot.rain === true;
        case 'tank_empty':
            return plot.tank === 'empty';
        case 'tank_ready':
            return plot.tank === 'ready';
        default:
            return false;
    }
}
```

## 16.3 การใช้คำสั่งกับแปลงผัก

```javascript
function applyAction(plot, action) {
    let result = {
        status: 'safe',
        hpChange: 0,
        message: ''
    };

    if (action === 'water') {
        if (plot.tank === 'empty') {
            result.status = 'fail';
            result.hpChange = -15;
            result.message = 'ถังน้ำหมด แต่ระบบพยายามรดน้ำ';
        } else if (plot.rain) {
            result.status = 'flood';
            result.hpChange = -25;
            result.message = 'ฝนตกอยู่แล้ว แต่ระบบยังรดน้ำเพิ่ม';
        } else if (plot.soil === 'wet') {
            result.status = 'overwater';
            result.hpChange = -15;
            result.message = 'ดินชื้นแล้ว แต่ระบบยังรดน้ำเพิ่ม';
        } else {
            result.status = 'healthy';
            result.hpChange = 20;
            result.message = 'ดินแห้งได้รับน้ำ พืชฟื้นแล้ว';
        }
    }

    if (action === 'stop') {
        if (plot.soil === 'dry' && !plot.rain) {
            result.status = 'dry';
            result.hpChange = -20;
            result.message = 'ดินแห้งแต่ระบบหยุดรดน้ำ';
        } else {
            result.status = 'safe';
            result.hpChange = 5;
            result.message = 'ระบบหยุดรดน้ำได้เหมาะสม';
        }
    }

    if (action === 'refill') {
        if (plot.tank === 'empty') {
            result.status = 'refill';
            result.hpChange = 5;
            result.message = 'ระบบแจ้งเติมน้ำได้ถูกต้อง';
        } else {
            result.status = 'unneeded_refill';
            result.hpChange = -5;
            result.message = 'ถังน้ำยังไม่หมด แต่ระบบแจ้งเติมน้ำ';
        }
    }

    if (action === 'observe') {
        if (plot.soil === 'wet' && !plot.rain) {
            result.status = 'safe';
            result.hpChange = 5;
            result.message = 'สถานการณ์ปลอดภัย ระบบสังเกตต่อได้';
        } else {
            result.status = 'miss';
            result.hpChange = -10;
            result.message = 'ระบบสังเกตต่อ ทั้งที่ควรตัดสินใจทำอย่างอื่น';
        }
    }

    return result;
}
```

# 24. สรุป

การพัฒนาบทเรียนที่ 3 ควรเปลี่ยนจากเกมจับคู่คำตอบเป็นเกม **Water Hero: ศึกป้องกันสวนอัจฉริยะ** ซึ่งเป็นเกมแนว Action Puzzle + Tower Defense + Condition Cards ให้ผู้เรียนใช้เงื่อนไข If-Then-Else เพื่อป้องกันสวนจากภัยคุกคาม

จุดเด่นของแนวใหม่นี้คือ

- เด็กมีบทบาทเป็นฮีโร่
- มีภารกิจช่วยสวน
- มีภัยคุกคามและความตื่นเต้น
- มี Wave และด่านบอส
- มีการ์ดเงื่อนไขและตู้ควบคุม
- เห็นผลลัพธ์จากกฎที่สร้าง
- มี feedback เชิงเหตุผล
- ยังสอน If-Then-Else ได้ครบถ้วน

หัวใจสำคัญของการพัฒนาคือ

> ทำให้ผู้เรียนรู้สึกว่า “ฉันกำลังใช้เงื่อนไขเพื่อชนะเกม” ไม่ใช่ “ฉันกำลังตอบคำถามเรื่องเงื่อนไข”

หากพัฒนาได้ตามแผนนี้ บทเรียนที่ 3 จะมีความเป็นเกมสูงขึ้นมาก น่าเล่นขึ้น สนุกขึ้น และยังคงตอบโจทย์การเรียนรู้วิทยาการคำนวณเรื่องการใช้เงื่อนไขได้อย่างชัดเจน
