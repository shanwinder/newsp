## 1. ภาพรวมแนวคิด

เกมบทเรียนที่ 3 จะเปลี่ยนเป็นเกมชื่อ **Smart Farm Manager: สายพานตรรกะฟาร์มอัจฉริยะ** ผู้เรียนรับบทเป็นผู้จัดการฟาร์มอัจฉริยะที่ต้องตั้งโปรแกรมควบคุมเครื่องจักรในฟาร์ม ได้แก่ โรงคัดแยกผลผลิต หุ่นยนต์เลี้ยงสัตว์ และระบบโรงเรือนไฮเทค โดยใช้บล็อกคำสั่งแบบ Scratch-like เพื่อสร้างตรรกะเงื่อนไข

กลไกหลักคือ

```text
ตั้งกฎ → ปล่อยสายพาน → วัตถุ/เหตุการณ์เคลื่อนผ่านจุดตรวจ → ระบบตัดสินใจตามกฎ → แสดงผลด้วย Animation → ได้คะแนน เหรียญ ดาว หรือเกิดความเสียหาย
```

เด็กไม่ได้ตอบคำถามแบบแบบฝึก แต่กำลัง “เขียนโปรแกรมควบคุมฟาร์ม” ให้ทำงานอัตโนมัติจริง ๆ

## 2. เหตุผลที่เลือกแนว Conveyor Belt Logic

แนวสายพานลำเลียงเหมาะกับบทเรียน Condition เพราะทำให้เด็กเห็นลำดับการตัดสินใจอย่างเป็นรูปธรรม วัตถุเคลื่อนผ่านจุดตรวจ แล้วระบบจะตรวจเงื่อนไขจากบนลงล่าง หากเข้าเงื่อนไขก็ทำคำสั่งนั้น หากไม่เข้าเงื่อนไขก็ไปเงื่อนไขถัดไป หรือไป Else

จุดแข็งของแนวนี้คือ

1. **มีภาพเคลื่อนไหวตลอดเวลา** วัตถุ สัตว์ หรือเหตุการณ์เคลื่อนผ่านหน้าจอ ทำให้รู้สึกว่าเป็นเกมจริง
2. **สื่อ If / Else ได้ชัดเจน** เพราะจุดตรวจทำหน้าที่เหมือนเครื่องประมวลผลเงื่อนไข
3. **ใช้ Engine เดียวสร้างได้หลายด่าน** เปลี่ยนเพียง Sprite, เงื่อนไข, คำสั่ง, ฉาก และ Level Config
4. **เหมาะกับเด็ก ป.4** เพราะเข้าใจภาพสายพาน ประตู เครื่องจักร และผลลัพธ์ได้ง่าย
5. **เกิด Feedback ทันที** ถูกก็เด้งเข้ากล่อง ผิดก็ตกพื้น เครื่องช็อต หรือสัตว์งง
6. **ต่อยอด Gamification ได้ดี** เช่น Combo, เหรียญ, ดาว, ร้านแต่งฟาร์ม, ความเร็วสายพาน

## 3. โครงสร้างบทเรียนใหม่

บทที่ 3 ประกอบด้วย 3 Stage หลัก และแต่ละ Stage มี 3 ด่านย่อย รวมทั้งหมด 9 ด่านย่อย

| Stage | ชื่อเกม | เนื้อหาหลัก | ลักษณะการเรียนรู้ |
|---|---|---|---|
| Stage 7 | โรงคัดแยกผลผลิตของพี่สมปอง | เงื่อนไขจากลักษณะสิ่งของ | สี ขนาด ความสะอาด ความสุก |
| Stage 8 | หุ่นยนต์เลี้ยงสัตว์อัจฉริยะ | เงื่อนไขจากประเภทและค่าสถานะสัตว์ | ประเภทสัตว์ น้ำหนัก ขน |

| ด่านที่ 3 | If / Else If / Else | เข้าใจการตรวจหลายเงื่อนไขตามลำดับ |

## 4. Core Game Loop

### 4.1 Briefing: รับภารกิจ

ก่อนเริ่มเกม ระบบแสดงโจทย์สั้น ๆ เช่น

```text
วันนี้พี่สมปองต้องคัดแยกมะม่วง 20 ลูก
ช่วยตั้งกฎให้สายพานส่งมะม่วงไปถูกที่นะ!
```

โจทย์ต้องสั้น ชัด และบอกเป้าหมาย เช่น ต้องคัดถูกกี่ชิ้น ห้ามเสียหายเกินกี่ชิ้น หรือมีเงื่อนไขอะไรที่ควรระวัง

### 4.2 Programming: สร้างกฎ

ผู้เรียนลากบล็อกเงื่อนไขและบล็อกคำสั่งไปใส่ในแผงโปรแกรม เช่น

```text
ถ้า [มะม่วงสีเหลืองสุก] → [ส่งไปขายแพ็คคู่]
นอกเหนือจากนี้ถ้า [มะม่วงสีเขียวดิบ] → [ส่งไปทำมะม่วงดอง]
นอกเหนือจากนี้ → [ส่งเข้าถังปุ๋ยหมัก]
```

### 4.3 Run: ปล่อยสายพาน

เมื่อกดปุ่ม “เริ่มสายพาน” วัตถุจะเคลื่อนที่บนสายพานทีละชิ้น ผ่านจุดตรวจสอบ แล้วระบบจะประเมินกฎตามลำดับจากบนลงล่าง

### 4.4 Feedback: เห็นผลทันที

ถ้าตัดสินใจถูก จะมีผลลัพธ์เชิงบวก เช่น ผลไม้เด้งเข้ากล่อง เครื่องจักรทำงาน เสียง “ชิ้ง!” คะแนนเพิ่ม และ Combo ต่อเนื่อง

ถ้าตัดสินใจผิด จะมีผลลัพธ์เชิงตลกและสื่อความหมาย เช่น ผลไม้ตกพื้น เครื่องจักรมีควัน หุ่นยนต์ช็อต สัตว์ทำหน้างง หรือต้นไม้เหี่ยว

### 4.5 Result: สรุปผล

เมื่อจบจำนวนวัตถุที่กำหนด ระบบสรุปผล เช่น

```text
ถูกต้อง 17/20 รายการ
เสียหาย 3 รายการ
Combo สูงสุด x5
ได้รับ 2 ดาว
ได้รับ 45 เหรียญ
```

## 5. UI/UX ที่เสนอ

### 5.1 โครงสร้างหน้าจอหลัก

```text
┌─────────────────────────────────────────────────────┐
│ คะแนน | Combo | เหรียญ | จำนวนสินค้า | ดาว          │
├─────────────────────────────────────────────────────┤
│                                                     │
│       พื้นที่สายพาน + จุดสแกน + ประตูแยกทาง        │
│                                                     │
│  วัตถุวิ่ง → จุดตรวจ → เครื่องจักร/ประตู → ปลายทาง │
│                                                     │
├───────────────────────────────┬─────────────────────┤
│ แผงสร้างโปรแกรม              │ คลังบล็อก            │
│ ถ้า [____] → [____]           │ - เงื่อนไข            │
│ นอกเหนือจากนี้ถ้า [____] →... │ - คำสั่ง              │
│ นอกเหนือจากนี้ → [____]       │ - ปุ่มควบคุม          │
└───────────────────────────────┴─────────────────────┘
```

### 5.2 การแยกหมวด UI

ต้องแยกเป็น 4 โซนชัดเจน

1. **พื้นที่เกมหลัก** แสดงสายพาน วัตถุ เครื่องจักร ประตู และ Animation
2. **แผงสร้างโปรแกรม** แสดง If / Else If / Else ที่ผู้เรียนลากบล็อกมาใส่
3. **คลังบล็อก** แยกบล็อกเงื่อนไขและบล็อกคำสั่งด้วยสีต่างกัน
4. **แผงควบคุมเกม** ปุ่มเริ่มสายพาน หยุดชั่วคราว ล้างกฎ ย้อนกลับ และคำใบ้ ต้องแยกจากบล็อก

### 5.3 สีของบล็อก

| ประเภทบล็อก | สีที่เสนอ |
|---|---|
| If / Else If / Else | เหลือง/ส้ม |
| เงื่อนไข | น้ำเงิน |
| คำสั่ง | เขียว |
| ปุ่มควบคุม | ม่วง/แดง/เขียวเด่น |
| Feedback ถูก | เขียว |
| Feedback ผิด | แดง |
| Combo / เหรียญ | ทอง |

### 5.4 Drag & Drop

Drag & Drop เป็นแกนหลักของการเล่น

- ลากบล็อกเงื่อนไขใส่ช่องเงื่อนไข
- ลากบล็อกคำสั่งใส่ช่องคำสั่ง
- ลากเพื่อสลับลำดับกฎในด่านที่อนุญาต
- ลากบล็อกออกจากช่องเพื่อลบ
- วางผิดช่องแล้วเด้งกลับ
- ช่องที่รับบล็อกได้ต้องเรืองแสงเมื่อกำลังลาก

สำหรับมือถือรองรับ Tap-to-place เป็นทางเลือกเสริม

## 6. ระบบเกมหลัก

### 6.1 Conveyor Belt

สายพานต้องมี Animation เคลื่อนที่ตลอดเวลาเมื่อเริ่มเกม เช่น พื้นสายพานขยับวน ลูกกลิ้งหมุน วัตถุเลื่อนผ่านจุดสแกน และมีเสียงเครื่องจักรเบา ๆ

### 6.2 Scan Point

เมื่อวัตถุผ่านจุดสแกน เกมต้อง

1. ชะลอวัตถุสั้น ๆ
2. เล่น Animation สแกน เช่น แสงสีฟ้ากวาดผ่านวัตถุ
3. อ่านคุณสมบัติของวัตถุ
4. ประเมินกฎจากบนลงล่าง
5. เปิดประตูหรือสั่งเครื่องจักรทำงาน

### 6.3 Logic Gate / Sorting Gate

ประตูตรรกะแยกวัตถุไปยังปลายทางต่าง ๆ เช่น เครื่องล้าง กล่องพรีเมียม กล่องแปรรูป ถังปุ๋ยหมัก เครื่องตัดขน เครื่องให้อาหาร สปริงเกอร์ หลังคา พัดลม หรือหลอดไฟ

### 6.4 Rule Evaluation

```javascript
function evaluateRules(item, rules) {
  for (const rule of rules) {
    if (rule.type === 'else') {
      return rule.action;
    }
    if (checkCondition(item, rule.condition)) {
      return rule.action;
    }
  }
  return 'pass';
}
```

### 6.5 Condition Checker

```javascript
function checkCondition(item, conditionId) {
  switch (conditionId) {
    case 'carrot_dirty':
      return item.type === 'carrot' && item.dirty === true;
    case 'orange_big':
      return item.type === 'orange' && item.size === 'big';
    case 'mango_ripe':
      return item.type === 'mango' && item.ripeness === 'ripe';
    case 'mango_raw':
      return item.type === 'mango' && item.ripeness === 'raw';
    case 'sheep_woolly':
      return item.type === 'sheep' && item.wool === 'long';
    case 'animal_cow':
      return item.type === 'cow';
    case 'pig_under_40':
      return item.type === 'pig' && item.weight < 40;
    case 'pig_over_80':
      return item.type === 'pig' && item.weight > 80;
    case 'soil_low_moisture':
      return item.type === 'plant_pot' && item.moisture < 30;
    case 'rain':
      return item.weather === 'rain';
    case 'temp_over_35':
      return item.temperature > 35;
    case 'temp_under_15':
      return item.temperature < 15;
    default:
      return false;
  }
}
```

## 7. รายละเอียด Stage 7: โรงคัดแยกผลผลิตของพี่สมปอง

**เป้าหมายหลัก:** สอนเงื่อนไขจากลักษณะของผลผลิต เช่น ความสะอาด ขนาด สี และความสุก  
**โทนเกม:** สดใส เข้าใจง่าย เหมาะสำหรับเริ่มต้นบทเรียน  
**ตัวละครประกอบ:** พี่สมปอง เกษตรกรเจ้าของโรงคัดแยกผลผลิต

### 7.1 ด่าน 7-1: แครอทเปื้อนโคลน

**แนวคิด:** If  
**โจทย์:** คัดแยกแครอทเปื้อนโคลนไปล้างน้ำ  
**ตรรกะ:**

```text
ถ้า [แครอทเปื้อนโคลน] → [ส่งไปเครื่องล้างน้ำ]
```

ถ้าแครอทสะอาดอยู่แล้ว ให้ปล่อยผ่านลงตะกร้า

**วัตถุในด่าน:** แครอทสะอาด, แครอทเปื้อนโคลน

**บล็อกที่มีให้:**

เงื่อนไข:

- แครอทเปื้อนโคลน

คำสั่ง:

- ส่งไปเครื่องล้างน้ำ
- ปล่อยผ่าน

**Animation ที่ต้องมี:**

- แครอทเคลื่อนบนสายพาน
- จุดสแกนส่องไฟตรวจคราบโคลน
- ประตูเปิดส่งแครอทเข้าเครื่องล้าง
- เครื่องล้างหมุนและพ่นน้ำ
- แครอทสะอาดเด้งลงตะกร้า
- ถ้าผิด แครอทเลอะลงตะกร้าและตะกร้ามีคราบโคลน

**Feedback ตัวอย่าง:**

```text
แครอทเปื้อนโคลนหลุดลงตะกร้า ทำให้ตะกร้าเลอะ ลองใช้เงื่อนไข “แครอทเปื้อนโคลน”
```

**เงื่อนไขผ่านด่าน:**

- คัดแครอทถูกต้องอย่างน้อย 80%
- สินค้าเสียหายไม่เกิน 3 ชิ้น

---

### 7.2 ด่าน 7-2: คัดขนาดส้มแสนอร่อย

**แนวคิด:** If / Else  
**โจทย์:** คัดแยกส้มลูกใหญ่ไปกล่องพรีเมียม และส้มอื่น ๆ ไปกล่องแปรรูป  
**ตรรกะ:**

```text
ถ้า [ส้มลูกใหญ่] → [ใส่กล่องพรีเมียม]
นอกเหนือจากนี้ → [ใส่กล่องทำน้ำส้มสายชู]
```

**วัตถุในด่าน:** ส้มลูกใหญ่, ส้มลูกเล็ก

**Animation ที่ต้องมี:**

- ส้มกลิ้งบนสายพาน
- เครื่องวัดขนาดมีไม้บรรทัดหรือวงแหวนสแกน
- ส้มลูกใหญ่เด้งเข้ากล่องพรีเมียม
- ส้มลูกเล็กเด้งเข้ากล่องแปรรูป
- กล่องปิดฝาเมื่อรับถูกต้อง
- ถ้าผิด กล่องสั่นและมีเครื่องหมายตกใจ

**Feedback ตัวอย่าง:**

```text
ส้มลูกเล็กไม่ควรเข้ากล่องพรีเมียม ควรส่งไปกล่องทำน้ำส้มสายชู
```

**เงื่อนไขผ่านด่าน:**

- แยกส้มถูกต้องอย่างน้อย 85%
- ใช้ Else ได้ถูกต้อง

---

### 7.3 ด่าน 7-3: คัดเกรดมะม่วงตามความสุก

**แนวคิด:** If / Else If / Else  
**โจทย์:** คัดเกรดมะม่วงตามความสุก  
**ตรรกะ:**

```text
ถ้า [มะม่วงสีเหลืองสุก] → [ส่งไปขายแพ็คคู่]
นอกเหนือจากนี้ถ้า [มะม่วงสีเขียวดิบ] → [ส่งไปทำมะม่วงดอง]
นอกเหนือจากนี้ → [ทิ้งถังปุ๋ยหมัก]
```

**วัตถุในด่าน:** มะม่วงสีเหลืองสุก, มะม่วงสีเขียวดิบ, มะม่วงเน่าดำ

**Animation ที่ต้องมี:**

- เครื่องสแกนสีมะม่วง
- ประตูสามทาง
- มะม่วงสุกเข้ากล่องแพ็คคู่
- มะม่วงดิบเข้าถังทำมะม่วงดอง
- มะม่วงเน่าเด้งลงถังปุ๋ยหมัก
- ถังปุ๋ยหมักมีใบไม้และไอน้ำตลก ๆ

**Feedback ตัวอย่าง:**

```text
มะม่วงเน่าดำไม่เข้าเงื่อนไขสุกหรือดิบ จึงควรไปที่ Else คือถังปุ๋ยหมัก
```

**เงื่อนไขผ่านด่าน:**

- คัดถูกต้องอย่างน้อย 85%
- ใช้ If / Else If / Else ครบถ้วน

## 8. รายละเอียด Stage 8: หุ่นยนต์เลี้ยงสัตว์อัจฉริยะ

**เป้าหมายหลัก:** สอนเงื่อนไขจากประเภทวัตถุและค่าสถานะที่ซับซ้อนขึ้น  
**โทนเกม:** น่ารัก ตลก มีตัวละครสัตว์เคลื่อนไหว  
**ตัวละครประกอบ:** หุ่นยนต์ฟาร์ม “บอทบัวบาน”

### 8.1 ด่าน 8-1: ตัดขนแกะเมื่อขนยาว

**แนวคิด:** If  
**โจทย์:** ตัดขนแกะเมื่อขนฟูมาก  
**ตรรกะ:**

```text
ถ้า [แกะขนฟูมาก] → [กดปุ่มตัดขน]
```

ถ้าแกะเพิ่งตัดขนมาแล้ว ให้ปล่อยผ่านไปกินหญ้า

**Animation ที่ต้องมี:**

- แกะเดินดุ๊กดิ๊ก
- แกะขนฟูตัวกลมมาก
- เครื่องตัดขนทำงาน “ซูม!”
- ขนแกะฟุ้งเป็นก้อนเมฆ
- แกะหลังตัดขนยิ้ม
- ถ้าตัดแกะขนสั้น แกะหนาวสั่นตลก ๆ

**Feedback ตัวอย่าง:**

```text
แกะขนสั้นไม่ควรถูกตัดซ้ำ ลองใช้เงื่อนไขเฉพาะ “แกะขนฟูมาก”
```

---

### 8.2 ด่าน 8-2: แจกจ่ายอาหารให้สัตว์ในฟาร์ม

**แนวคิด:** If / Else  
**โจทย์:** แจกอาหารให้สัตว์แต่ละประเภท  
**ตรรกะ:**

```text
ถ้า [เป็นวัว] → [จ่ายหญ้าเนเปียร์]
นอกเหนือจากนี้ → [จ่ายรำข้าว]
```

สัตว์อื่น เช่น ไก่ เป็ด หมู ให้รำข้าว

**Animation ที่ต้องมี:**

- สัตว์เดินบนทางฟาร์ม
- เครื่องจ่ายอาหารหมุน
- วัวได้หญ้าแล้วหัวใจขึ้น
- ไก่/เป็ด/หมูได้รำแล้วดีใจ
- ถ้าวัวได้รำ วัวทำหน้างง
- ถ้าไก่ได้หญ้า ไก่ลากหญ้าไม่ไหว

**Feedback ตัวอย่าง:**

```text
วัวควรได้หญ้าเนเปียร์ ส่วนสัตว์อื่นใช้ Else เพื่อจ่ายรำข้าว
```

---

### 8.3 ด่าน 8-3: เติมสารอาหารให้แม่หมูตามน้ำหนัก

**แนวคิด:** If / Else If / Else และเงื่อนไขตัวเลข  
**โจทย์:** เติมอาหารให้แม่หมูตามน้ำหนัก  
**ตรรกะ:**

```text
ถ้า [น้ำหนัก < 40 กิโล] → [ให้สูตรเร่งโต]
นอกเหนือจากนี้ถ้า [น้ำหนัก > 80 กิโล] → [ให้สูตรคุมน้ำหนัก]
นอกเหนือจากนี้ → [ให้สูตรอาหารปกติ]
```

**Animation ที่ต้องมี:**

- หมูเดินขึ้นแท่นชั่ง
- ตัวเลขน้ำหนักเด้งขึ้น
- เครื่องจ่ายอาหารสูตรต่าง ๆ สีต่างกัน
- หมูผอมได้สูตรเร่งโตแล้วยิ้ม
- หมูอ้วนได้สูตรคุมน้ำหนักแล้ววิ่งออกกำลัง
- หมูปกติได้อาหารปกติแล้วเดินสบาย

**Feedback ตัวอย่าง:**

```text
หมูน้ำหนัก 85 กิโล มากกว่า 80 กิโล ควรได้รับสูตรคุมน้ำหนัก
```

## 9. รายละเอียด Stage 9: ผู้ดูแลโรงเรือนไฮเทค

**เป้าหมายหลัก:** ใช้เงื่อนไขจากสิ่งแวดล้อมและตัวเลข  
**โทนเกม:** ไฮเทคขึ้น เป็นด่านปราบเซียน แต่ยังต้องเข้าใจง่ายสำหรับ ป.4  
**ตัวละครประกอบ:** ระบบ AI โรงเรือน “น้องเซนเซอร์”

### 9.1 ด่าน 9-1: ระบบรดน้ำอัตโนมัติ

**แนวคิด:** If  
**โจทย์:** เปิดสปริงเกอร์เมื่อความชื้นในดินต่ำ  
**ตรรกะ:**

```text
ถ้า [ความชื้นในดินต่ำ] → [เปิดสปริงเกอร์รดน้ำ]
```

ถ้าความชื้นดีอยู่แล้ว ไม่ต้องทำอะไร

**Animation ที่ต้องมี:**

- กระถางไหลผ่านสายพานในโรงเรือน
- เซ็นเซอร์วัดความชื้นโชว์ตัวเลข
- สปริงเกอร์หมุนพ่นน้ำ
- ต้นไม้ฟื้นเมื่อได้รดน้ำ
- ถ้าดินชื้นแต่รดน้ำ จะมีน้ำล้นกระถาง

**Feedback ตัวอย่าง:**

```text
ค่าความชื้นต่ำ จึงควรเปิดสปริงเกอร์ แต่ถ้าความชื้นดีแล้วไม่ต้องรดน้ำเพิ่ม
```

---

### 9.2 ด่าน 9-2: ระบบเปิด-ปิดหลังคากันฝนให้ต้นเมลอน

**แนวคิด:** If / Else  
**โจทย์:** กางหลังคาเมื่อฝนตก และเปิดรับแสงแดดเมื่อไม่ตก  
**ตรรกะ:**

```text
ถ้า [ฝนตก] → [กางหลังคาผ้าใบ]
นอกเหนือจากนี้ → [เปิดรับแสงแดด]
```

**Animation ที่ต้องมี:**

- เมฆฝนลอยมา
- ฝนตกลงบนโรงเรือน
- หลังคาผ้าใบกางออก
- แดดส่องเมื่อเปิดหลังคา
- เมลอนโตขึ้นเล็กน้อยเมื่อได้รับแสง
- ถ้าปิดหลังคาตอนแดดดี เมลอนหน้าหงอย

**Feedback ตัวอย่าง:**

```text
ตอนนี้ไม่มีฝน ควรเปิดหลังคาให้เมลอนได้รับแสงแดด
```

---

### 9.3 ด่าน 9-3: ควบคุมอุณหภูมิโรงเรือนเห็ด

**แนวคิด:** If / Else If / Else และตัวเลขช่วงค่า  
**โจทย์:** ควบคุมอุณหภูมิโรงเรือนเห็ด  
**ตรรกะ:**

```text
ถ้า [อุณหภูมิ > 35°C] → [เปิดพัดลมระบายอากาศ]
นอกเหนือจากนี้ถ้า [อุณหภูมิ < 15°C] → [เปิดหลอดไฟให้ความอบอุ่น]
นอกเหนือจากนี้ → [ปิดระบบปรับอากาศทั้งหมด]
```

**Animation ที่ต้องมี:**

- เทอร์โมมิเตอร์แสดงตัวเลข
- ไอร้อนเมื่ออุณหภูมิสูง
- พัดลมหมุนเมื่อเปิดระบายอากาศ
- ไอเย็นเมื่ออุณหภูมิต่ำ
- หลอดไฟสีอุ่นเปิดส่อง
- อุณหภูมิปกติ ระบบปิดและขึ้น Bonus ประหยัดไฟ

**Feedback ตัวอย่าง:**

```text
อุณหภูมิ 38°C สูงกว่า 35°C จึงควรเปิดพัดลมระบายอากาศ
```

## 10. ระบบคะแนน ดาว เหรียญ และ Combo

### 10.1 คะแนน

ผู้เรียนได้คะแนนจาก

- คัดแยกถูกต้อง
- ใช้กฎครบถ้วน
- ไม่ทำให้สินค้าเสียหาย
- ทำ Combo ได้ต่อเนื่อง
- ผ่านด่านโดยใช้เวลาน้อย
- ไม่ใช้คำใบ้

### 10.2 เหรียญ

ตัวอย่างการให้เหรียญ

```text
ถูก 1 ครั้ง = 5 เหรียญ
Combo x3 = +10 เหรียญ
ผ่านด่าน = +20 เหรียญ
ได้ 3 ดาว = +30 เหรียญ
```

### 10.3 ดาว

| ดาว | เงื่อนไข |
|---|---|
| 3 ดาว | ถูกต้องอย่างน้อย 95%, ไม่ใช้คำใบ้, เสียหายไม่เกิน 1 |
| 2 ดาว | ถูกต้องอย่างน้อย 80%, เสียหายไม่เกิน 4 |
| 1 ดาว | ผ่านขั้นต่ำตามเป้าหมาย |
| 0 ดาว | สินค้าเสียหายมากเกินไป หรือกฎไม่ครบ |

### 10.4 Combo

ถ้าคัดถูกต่อเนื่อง จะเพิ่ม Combo เช่น

```text
Correct x1
Combo x2
Combo x3
Super Farm Combo!
Perfect Logic!
```

Combo ต้องแสดงด้วยข้อความเด้งขึ้นกลางจอ พร้อมเสียงและเอฟเฟกต์
