# Game Design Specification: บทที่ 2, 3 และ 4

> เอกสารนี้เป็นข้อกำหนดการออกแบบเกม (Game Design Specification) สำหรับพัฒนาเกมแบบฝึกทักษะออนไลน์ เรื่อง “การแก้ปัญหาอย่างเป็นขั้นตอน” สำหรับนักเรียนชั้นประถมศึกษาปีที่ 4  
> ใช้สำหรับสั่งงาน Codex หรือผู้พัฒนาระบบให้ปรับปรุงบทที่ 2, 3 และ 4 ให้มี “ความเป็นเกม” เทียบเท่าบทที่ 1 โดยมีฉาก ภาพกราฟิก กลไกเกม กฎการเล่น เงื่อนไขชนะ/แพ้ ระบบดาว เวลา จำนวนครั้งที่พยายาม และการส่งผลลัพธ์กลับระบบหลัก

---

## 1. เป้าหมายของเอกสาร

เอกสารนี้กำหนดรายละเอียดการออกแบบเกมบทที่ 2, 3 และ 4 ให้ชัดเจนในระดับที่ผู้พัฒนาสามารถนำไปสร้างไฟล์เกมได้จริง โดยแยกแต่ละด่านว่า

- ฉากคืออะไร
- วัตถุในเกมมีอะไร
- ผู้เล่นต้องทำอะไร
- วิธีการเล่นเป็นอย่างไร
- กฎการเล่นคืออะไร
- ชนะอย่างไร
- แพ้อย่างไรหรือผิดพลาดอย่างไร
- ให้ดาวอย่างไร
- ใช้ asset อะไร
- ต้องส่งผลคะแนนกลับระบบอย่างไร

## 2. มาตรฐานความเป็นเกมที่ต้องใช้ทุกด่าน

ทุกด่านในบทที่ 2, 3 และ 4 ต้องเป็นเกมที่สร้างด้วย Phaser เหมือนบทที่ 1 ไม่ใช่แบบฝึกหัดแบบ static, quiz ธรรมดา, ตารางเลือกคำตอบ หรือหน้าจออ่านข้อความแล้วกดตอบ

ทุกด่านต้องมีองค์ประกอบต่อไปนี้

| องค์ประกอบ | รายละเอียดที่ต้องมี |
|---|---|
| Phaser Scene | ใช้ `new Phaser.Game(config)` และแสดงผลใน `#game-container` |
| ฉาก/พื้นหลัง | มีภาพพื้นหลังหรือฉากที่สื่อถึงฟาร์ม แปลงนา ระบบรดน้ำ หรือวิกฤตฟาร์ม |
| Sprite / Object | มีวัตถุในเกม เช่น รถไถ กองฟาง หิน ตะกร้าผลผลิต ถังน้ำ เมฆฝน แปลงผัก บล็อกคำสั่ง |
| Goal | ผู้เล่นต้องรู้ว่าภารกิจคืออะไร |
| Interaction | ผู้เล่นต้องลาก วาง คลิก สลับ แก้ไข หรือกดรัน ไม่ใช่แค่กดตอบ |
| Rule | มีข้อกำหนดว่าทำอะไรถูก ทำอะไรผิด |
| Feedback | มีเสียง/ข้อความ/animation เมื่อถูกหรือผิด |
| Win Condition | มีเงื่อนไขจบเกมที่ชัดเจน |
| Mistake / Attempt | มีการนับจำนวนครั้งที่ผิดพลาดหรือพยายาม |
| Time | มีการจับเวลา |
| Star Rating | คำนวณดาว 1–3 ดาว |
| Result Submission | เมื่อจบด่านต้องเรียก `window.sendResult(window.STAGE_ID, stars, duration, attempts)` |

## 4. โครงสร้างโค้ดมาตรฐานของทุกด่าน

ทุกไฟล์ควรมีโครงหลักคล้ายกัน เพื่อให้ดูแลรักษาง่าย

```javascript
const config = {
    type: Phaser.AUTO,
    width: 800,
    height: 600,
    backgroundColor: '#87CEEB',
    parent: 'game-container',
    physics: {
        default: 'arcade',
        arcade: { debug: false }
    },
    scene: {
        preload: preload,
        create: create,
        update: update
    }
};

const game = new Phaser.Game(config);

let startTime;
let attempts = 0;
let mistakes = 0;
let isGameEnded = false;

function preload() {
    // load assets
}

function create() {
    startTime = Date.now();
    // create scene
}

function update() {
    // game loop if needed
}

function endGame(success) {
    if (isGameEnded) return;
    isGameEnded = true;

    const duration = Math.floor((Date.now() - startTime) / 1000);
    const stars = calculateStars(success);

    setTimeout(() => {
        if (typeof window.sendResult === 'function') {
            window.sendResult(window.STAGE_ID, stars, duration, attempts);
        }
    }, 2000);
}

function calculateStars(success) {
    if (!success) return 0;
    if (attempts <= 1 && mistakes === 0) return 3;
    if (attempts <= 3 || mistakes <= 2) return 2;
    return 1;
}
```

หมายเหตุ: อาจปรับเกณฑ์ดาวให้เหมาะกับแต่ละด่าน แต่ต้องมีมาตรฐานใกล้เคียงกันทั้งระบบ

# บทที่ 2: เส้นทางเดินรถไถ (Sequence)

## 5. แนวคิดหลักของบทที่ 2

บทที่ 2 เป็นเกมฝึกการเรียงลำดับขั้นตอน ผู้เรียนต้องวางคำสั่งเพื่อพารถไถไปยังเป้าหมายบนแผนที่ โดยต้องคิดล่วงหน้า จัดลำดับคำสั่ง และกดรันเพื่อดูผลลัพธ์

### รูปแบบเกมหลักของบทที่ 2

เกมแนว **Command Sequencing / Grid Movement Game**

ผู้เล่นต้อง

1. มองแผนที่
2. วางแผนเส้นทาง
3. ลากบล็อกคำสั่ง
4. กดรัน
5. ดูรถไถเคลื่อนที่จริง
6. แก้ไขคำสั่งหากผิด
7. ผ่านด่านเมื่อรถไถถึงเป้าหมาย

### Asset หลักของบทที่ 2

| Asset | ใช้สำหรับ |
|---|---|
| `bg_farm.webp` หรือ `bg_field.webp` | พื้นหลังแปลงนา |
| `tractor.webp` | ตัวละครหลัก รถไถ |
| `hay.webp` หรือ `haystack.webp` | กองฟาง สิ่งกีดขวาง |
| `rock.webp` | หิน สิ่งกีดขวาง |
| `basket.webp` | ตะกร้าผลผลิต |
| `barn.webp` | โรงนา/จุดเริ่มต้นหรือจุดหมาย |
| `tile_grass.webp` หรือใช้ graphics วาด grid | ช่องแผนที่ |
| `arrow_up.webp`, `arrow_down.webp`, `arrow_left.webp`, `arrow_right.webp` หรือใช้ emoji/text | บล็อกคำสั่ง |
| `correct.mp3` | เสียงถูก |
| `wrong.mp3` | เสียงผิด |
| `success.mp3` ถ้ามี | เสียงผ่านด่าน |

หาก asset บางไฟล์ยังไม่มี ให้ใช้ Phaser graphics หรือ emoji ชั่วคราวได้ แต่ต้องแสดงเป็นภาพ/วัตถุในเกม ไม่ใช่ข้อความล้วน

## 6. Stage 4: เดินหน้าสู่แปลงนา

### 6.1 ฉาก

ฉากเป็นแปลงนาขนาดเล็ก มีตาราง 4x4 หรือ 5x5 อยู่กลางหน้าจอ พื้นหลังเป็นฟาร์มหรือท้องนา

บนแผนที่มี

- รถไถอยู่ที่จุดเริ่มต้น
- เป้าหมายเป็นแปลงนา/ตะกร้าผลผลิต
- ไม่มีสิ่งกีดขวางหรือมีน้อยมาก
- มีป้ายภารกิจด้านบน

### 6.2 วัตถุในเกม

| วัตถุ | รายละเอียด |
|---|---|
| รถไถ | ตัวละครหลัก เคลื่อนที่ทีละช่อง |
| เป้าหมาย | ตะกร้าหรือแปลงนา |
| Grid | ช่องทางเดิน |
| บล็อกคำสั่ง | ปุ่มลูกศรให้ลากหรือคลิกเพิ่ม |
| พื้นที่คำสั่ง | แถบด้านล่างสำหรับเรียงคำสั่ง |
| ปุ่มรัน | ใช้สั่งให้รถไถเคลื่อนที่ |
| ปุ่มล้างคำสั่ง | ใช้เริ่มเรียงใหม่ |

### 6.3 ผู้เล่นทำอะไร

ผู้เล่นต้องเลือกหรือลากบล็อกคำสั่ง เช่น `ขวา`, `ขวา`, `ขวา` ลงในช่องคำสั่ง แล้วกด “รันรถไถ” เพื่อให้รถไถเคลื่อนที่ไปถึงเป้าหมาย

### 6.4 วิธีการเล่น

1. อ่านภารกิจด้านบน
2. ดูตำแหน่งรถไถและเป้าหมาย
3. เลือกบล็อกคำสั่งที่เหมาะสม
4. เรียงคำสั่งในแถบด้านล่าง
5. กดปุ่ม “รันรถไถ”
6. รถไถเคลื่อนที่ตามคำสั่งทีละช่อง
7. หากถึงเป้าหมายจะผ่านด่าน

### 6.5 กฎการเล่น

- รถไถเคลื่อนที่ตามลำดับคำสั่งที่ผู้เล่นวาง
- รถไถต้องอยู่ในขอบเขตแผนที่
- หากคำสั่งทำให้รถไถออกนอกแผนที่ ถือว่าผิด
- ผู้เล่นสามารถล้างคำสั่งและลองใหม่ได้
- ทุกครั้งที่กดรันให้นับเป็น 1 attempt

### 6.6 เงื่อนไขชนะ

ชนะเมื่อรถไถเคลื่อนที่ไปถึงเป้าหมายพอดี

### 6.7 เงื่อนไขผิดพลาด/แพ้

ในด่านนี้ไม่ควร Game Over ทันที แต่ให้เกิด feedback แล้วให้ลองใหม่

ผิดพลาดเมื่อ

- รถไถออกนอกแผนที่
- รถไถหยุดไม่ถึงเป้าหมาย
- คำสั่งเกินจนเลยเป้าหมาย

### 6.8 Feedback

- ถ้าถูก: รถไถมี animation ดีใจ หรือมีดาวลอยขึ้น พร้อมเสียง correct
- ถ้าผิด: รถไถสั่น มีเสียง wrong และข้อความ “ลองตรวจลำดับคำสั่งอีกครั้ง”

### 6.9 การให้ดาว

| เงื่อนไข | ดาว |
|---|---:|
| ผ่านใน attempt แรก | 3 |
| ผ่านใน attempt ที่ 2–3 | 2 |
| ผ่านตั้งแต่ attempt ที่ 4 ขึ้นไป | 1 |

### 6.10 Asset ที่ต้องใช้

- พื้นหลังฟาร์ม
- รถไถ
- เป้าหมาย/ตะกร้า
- ลูกศรคำสั่ง
- เสียง correct/wrong
- effect ดาวหรือข้อความสำเร็จ

## 7. Stage 5: หลบหลีกกองฟาง

### 7.1 ฉาก

ฉากเป็นแปลงนาที่มีเส้นทางซับซ้อนกว่าเดิม มีตาราง 5x5 หรือ 6x6 พร้อมกองฟางหรือหินเป็นสิ่งกีดขวาง

### 7.2 วัตถุในเกม

| วัตถุ | รายละเอียด |
|---|---|
| รถไถ | เริ่มต้นที่มุมหนึ่งของแผนที่ |
| เป้าหมาย | แปลงนาหรือโรงนา |
| กองฟาง | สิ่งกีดขวาง ห้ามชน |
| หิน | สิ่งกีดขวางเสริม |
| Grid | พื้นที่เดิน |
| บล็อกคำสั่ง | ขึ้น ลง ซ้าย ขวา |
| ปุ่มรัน/ล้าง | ควบคุมการทดลอง |

### 7.3 ผู้เล่นทำอะไร

ผู้เล่นต้องวางคำสั่งเพื่อพารถไถไปถึงเป้าหมาย โดยหลบกองฟางและหิน

### 7.4 วิธีการเล่น

1. มองตำแหน่งรถไถ เป้าหมาย และสิ่งกีดขวาง
2. วางแผนเส้นทาง
3. เรียงบล็อกคำสั่ง
4. กดรัน
5. ถ้าชนสิ่งกีดขวาง ให้แก้ไขคำสั่งแล้วลองใหม่

### 7.5 กฎการเล่น

- รถไถห้ามชนกองฟางหรือหิน
- รถไถห้ามออกนอกแผนที่
- หากชนสิ่งกีดขวาง ให้หยุดทันทีและนับ mistake
- กดรันแต่ละครั้งนับเป็น 1 attempt
- ผู้เล่นสามารถแก้ชุดคำสั่งได้หลังรันผิด

### 7.6 เงื่อนไขชนะ

ชนะเมื่อรถไถไปถึงเป้าหมายโดยไม่ชนสิ่งกีดขวาง

### 7.7 เงื่อนไขผิดพลาด/แพ้

- ชนกองฟาง
- ชนหิน
- ออกนอกแผนที่
- ใช้คำสั่งแล้วไปไม่ถึงเป้าหมาย

ไม่ต้องจบเกมทันที ให้ลองใหม่ได้ แต่จะเสียดาว

### 7.8 Feedback

- ชนกองฟาง: มีเสียงผิด รถไถสั่น ควันหรือเครื่องหมาย ❌
- ออกนอกแผนที่: ข้อความ “รถไถออกนอกเส้นทาง”
- ไปไม่ถึงเป้าหมาย: ข้อความ “ยังไม่ถึงแปลงนา ลองเพิ่มหรือเปลี่ยนคำสั่ง”
- ผ่าน: รถไถถึงเป้าหมาย มีแสง/ดาวและเสียงสำเร็จ

### 7.9 การให้ดาว

| เงื่อนไข | ดาว |
|---|---:|
| ผ่านโดยไม่ชนและ attempt แรก | 3 |
| ผ่านโดยผิด 1–2 ครั้ง | 2 |
| ผ่านโดยผิด 3 ครั้งขึ้นไป | 1 |

### 7.10 Asset ที่ต้องใช้

- พื้นหลังแปลงนา
- รถไถ
- กองฟาง
- หิน
- เป้าหมาย
- ลูกศรคำสั่ง
- เสียง correct/wrong
- ควันหรือ effect ชน

## 8. Stage 6: เก็บเกี่ยวผลผลิต

### 8.1 ฉาก

ฉากเป็นแปลงนาขนาด 6x6 มีผลผลิตหลายจุด รถไถต้องเก็บให้ครบก่อนกลับโรงนา

### 8.2 วัตถุในเกม

| วัตถุ | รายละเอียด |
|---|---|
| รถไถ | ตัวละครหลัก |
| ตะกร้าผลผลิต 2–3 จุด | เป้าหมายย่อยที่ต้องเก็บ |
| โรงนา | จุดหมายสุดท้าย |
| กองฟาง/หิน | สิ่งกีดขวาง |
| Grid | แผนที่ |
| บล็อกคำสั่ง | ลูกศร |
| ตัวนับผลผลิต | แสดงเก็บแล้วกี่จุด |

### 8.3 ผู้เล่นทำอะไร

ผู้เล่นต้องวางลำดับคำสั่งให้รถไถเก็บผลผลิตครบทุกจุด แล้วกลับไปยังโรงนา

### 8.4 วิธีการเล่น

1. ตรวจดูตำแหน่งผลผลิตทั้งหมด
2. วางแผนว่าจะไปเก็บจุดใดก่อน
3. เรียงคำสั่ง
4. กดรัน
5. รถไถเคลื่อนที่และเก็บผลผลิตเมื่อเดินผ่าน
6. ผ่านเมื่อเก็บครบและถึงโรงนา

### 8.5 กฎการเล่น

## 10. Stage 7: เช็คดินแห้งหรือเปียก

### 10.1 ฉาก

### 10.2 วัตถุในเกม

| วัตถุ | รายละเอียด |
|---|---|
| แปลงผัก | พื้นที่หลัก |

| ต้นไม้ | แสดงผลลัพธ์หลังรดน้ำ |
| หัวฉีดน้ำ | ทำงานเมื่อคำสั่งถูก |

### 10.4 วิธีการเล่น

1. ดูสถานะดิน
2. ลากบล็อกเงื่อนไขไปวางในช่อง “ถ้า”
3. ลากบล็อกคำสั่งไปวางในช่อง “แล้ว”
4. กด “ทดสอบระบบ”
5. ระบบแสดงว่าน้ำทำงานถูกหรือไม่

### 10.5 กฎการเล่น

- ถ้าดินแห้ง ต้องรดน้ำ
- ถ้าดินชื้น ไม่ต้องรดน้ำ
- ถ้ารดน้ำผิดสถานะ จะนับ mistake
- กดทดสอบแต่ละครั้งนับเป็น attempt

### 10.6 เงื่อนไขชนะ

ชนะเมื่อระบบทำงานถูกต้องตามสถานะดินที่กำหนด

### 10.7 เงื่อนไขผิดพลาด

- ดินแห้งแต่ไม่รดน้ำ → ต้นไม้เหี่ยว
- ดินชื้นแต่รดน้ำ → น้ำขัง
- เลือกบล็อกไม่ครบ → แจ้งให้วางคำสั่งให้สมบูรณ์

### 10.8 Feedback

- ถูก: น้ำไหล ต้นไม้สดชื่น มีเสียง correct
- ผิด: ต้นไม้เหี่ยวหรือน้ำขัง มีเสียง wrong
- ข้อความช่วย: “ลองดูว่าดินแห้งหรือชื้นก่อนเลือกคำสั่ง”

### 11.3 ผู้เล่นทำอะไร

```text
ถ้าฝนตก → หยุดรดน้ำ
มิฉะนั้น ถ้าดินแห้ง → รดน้ำ
มิฉะนั้น → ไม่ต้องรดน้ำ
```

### 11.4 วิธีการเล่น

1. ดูสถานะฝนและดิน
2. ลากบล็อก “ถ้าฝนตก” ไปไว้เงื่อนไขแรก
3. ลากบล็อก “หยุดรดน้ำ” ไปเป็นผลลัพธ์
4. ลากบล็อก “ถ้าดินแห้ง” ไปไว้ในช่องมิฉะนั้น
5. กดทดสอบ
6. ดูผลลัพธ์ว่าระบบหยุดหรือรดน้ำถูกต้องหรือไม่

### 11.5 กฎการเล่น

- หากฝนตก ระบบต้องหยุดรดน้ำเสมอ
- หากฝนไม่ตกและดินแห้ง ระบบจึงรดน้ำ
- หากฝนไม่ตกและดินชื้น ระบบไม่ต้องรดน้ำ
- หากตรวจดินก่อนฝน อาจทำให้เกิดน้ำท่วมในกรณีฝนตกและดินแห้ง
- การวางเงื่อนไขผิดลำดับนับเป็น mistake

### 11.6 เงื่อนไขชนะ

ชนะเมื่อระบบทำงานถูกต้องในสถานการณ์ทดสอบอย่างน้อย 2–3 สถานการณ์ เช่น

- ฝนตก + ดินแห้ง
- ฝนไม่ตก + ดินแห้ง
- ฝนไม่ตก + ดินชื้น

### 11.7 เงื่อนไขผิดพลาด

- ฝนตกแล้วยังรดน้ำ
- ดินแห้งแต่ไม่รดน้ำเมื่อฝนไม่ตก
- ดินชื้นแล้วยังรดน้ำ
- วางบล็อกไม่ครบ

### 11.8 Feedback

- “ฝนตกอยู่ ระบบควรหยุดรดน้ำก่อน”
- “เงื่อนไขฝนควรมาก่อนเงื่อนไขดิน”
- “ดินแห้งและฝนไม่ตก จึงควรรดน้ำ”
- “ระบบทำงานถูกต้องแล้ว”

|---|---:|

| ผิด 1–2 ครั้ง | 2 |
| ผิดมากกว่า 2 ครั้งแต่ผ่านได้ | 1 |

## 14. Stage 10: ขั้นตอนที่ผิดเพี้ยน

### 14.1 ฉาก

ฉากเป็นฟาร์มที่มี “สายพานขั้นตอนการทำงาน” หรือกระดานลำดับขั้นตอนอยู่กลางจอ มีภาพขั้นตอนการปลูกข้าวหรือดูแลพืชเรียงผิดลำดับ

### 14.2 วัตถุในเกม

| วัตถุ | รายละเอียด |
|---|---|
| การ์ดขั้นตอน | เช่น พรวนดิน, หว่านเมล็ด, รดน้ำ, เก็บเกี่ยว |
| กระดานลำดับ | ช่องวางขั้นตอน 1–4 |
| ปุ่มตรวจสอบ | ตรวจว่าลำดับถูกหรือไม่ |
| ตัวละครครู/หุ่นยนต์ฟาร์ม | ให้คำแนะนำ |
| เครื่องหมายถูก/ผิด | feedback |
| เสียงถูก/ผิด | ผลตอบสนอง |

### 14.3 ผู้เล่นทำอะไร

ผู้เล่นต้องลากการ์ดขั้นตอนที่เรียงผิดให้กลับมาอยู่ในลำดับที่ถูกต้อง

### 14.4 วิธีการเล่น

1. อ่านสถานการณ์ “ขั้นตอนการทำงานผิด”
2. ดูการ์ดขั้นตอนที่เรียงสลับกัน
3. ลากการ์ดไปวางในช่องลำดับที่ถูกต้อง
4. กดตรวจสอบ
5. หากผิด ระบบบอกว่าบางขั้นตอนยังไม่ถูก
6. แก้ไขจนถูกครบ

### 14.5 กฎการเล่น

- ต้องเรียงการ์ดให้ครบทุกช่อง
- ต้องเรียงตามลำดับเหตุผล
- กดตรวจสอบแต่ละครั้งนับเป็น attempt
- หากลำดับผิด ให้เพิ่ม mistake
- สามารถสลับการ์ดได้ไม่จำกัด แต่จะเสียดาวตามจำนวน attempt/mistake

### 14.6 เงื่อนไขชนะ

ชนะเมื่อลำดับถูกต้องครบทุกขั้นตอน

ตัวอย่างลำดับถูก

```text
พรวนดิน → หว่านเมล็ด → รดน้ำ → เก็บเกี่ยว
```

### 14.7 เงื่อนไขผิดพลาด

- เรียงขั้นตอนผิด
- ช่องว่างไม่ครบ
- เก็บเกี่ยวมาก่อนปลูก
- รดน้ำก่อนหว่านเมล็ด

### 14.8 Feedback

- “ขั้นตอนเก็บเกี่ยวควรอยู่หลังจากต้นพืชเติบโตแล้ว”
- “ลองคิดว่าต้องเตรียมดินก่อนหรือหลังหว่านเมล็ด”
- “ลำดับนี้ยังไม่ถูก ลองสลับขั้นตอนอีกครั้ง”
- “ถูกต้อง! ขั้นตอนการทำงานกลับมาเป็นปกติแล้ว”

### 14.9 การให้ดาว

| เงื่อนไข | ดาว |
|---|---:|
| เรียงถูกใน attempt แรก | 3 |
| เรียงถูกใน attempt 2–3 | 2 |
| เรียงถูกตั้งแต่ attempt 4 ขึ้นไป | 1 |

### 14.10 Asset ที่ต้องใช้

- พื้นหลังฟาร์ม
- การ์ดพรวนดิน
- การ์ดหว่านเมล็ด
- การ์ดรดน้ำ
- การ์ดเก็บเกี่ยว
- ช่องลำดับ
- ตัวละคร/หุ่นยนต์ฟาร์ม
- เสียงถูก/ผิด
- effect ดาว

## 15. Stage 11: รถไถหลงทาง

### 15.1 ฉาก

ฉากเป็นแผนที่รถไถคล้ายบทที่ 2 แต่มีชุดคำสั่งเดิมที่ผิด ผู้เล่นต้องกดรันเพื่อดูว่ารถไถผิดอย่างไร แล้วแก้ไขคำสั่ง

### 15.2 วัตถุในเกม

| วัตถุ | รายละเอียด |
|---|---|
| รถไถ | เคลื่อนที่ตามคำสั่ง |
| Grid | แผนที่ |
| กองฟาง/หิน | สิ่งกีดขวาง |
| เป้าหมาย | จุดหมาย |
| ชุดคำสั่งเดิม | มีบั๊ก 1–2 จุด |
| บล็อกคำสั่งสำรอง | ใช้เปลี่ยนคำสั่งผิด |
| ปุ่มรันระบบเดิม | แสดงผลลัพธ์ผิด |
| ปุ่มทดสอบหลังแก้ | ตรวจคำตอบ |

### 15.3 ผู้เล่นทำอะไร

ผู้เล่นต้องค้นหาคำสั่งที่ทำให้รถไถหลงทางหรือชนสิ่งกีดขวาง แล้วแก้ชุดคำสั่งให้ถูกต้อง

### 15.4 วิธีการเล่น

1. กด “รันระบบเดิม”
2. ดูว่ารถไถเริ่มผิดที่ขั้นตอนใด
3. เลือกบล็อกคำสั่งที่น่าจะเป็นบั๊ก
4. เปลี่ยนหรือสลับคำสั่ง
5. กด “ทดสอบใหม่”
6. ผ่านเมื่อรถไถถึงเป้าหมายโดยไม่ชน

### 15.5 กฎการเล่น

- ต้องแก้จากชุดคำสั่งเดิม ไม่ใช่สร้างใหม่ทั้งหมด
- คำสั่งที่ผิดมี 1–2 จุด
- กดทดสอบแต่ละครั้งนับเป็น attempt
- หากรถไถชนหรือไม่ถึงเป้าหมาย นับ mistake
- ใช้คำใบ้ได้ แต่ถ้าใช้คำใบ้ควรลดดาว

### 15.6 เงื่อนไขชนะ

ชนะเมื่อรถไถถึงเป้าหมายโดยไม่ชนสิ่งกีดขวางหลังจากแก้ชุดคำสั่ง

### 15.7 เงื่อนไขผิดพลาด

- แก้แล้วรถไถยังชน
- แก้แล้วรถไถออกนอกแผนที่
- แก้แล้วรถไถไม่ถึงเป้าหมาย
- ลบคำสั่งสำคัญจนเส้นทางไม่สมบูรณ์

### 15.8 Feedback

- “รถไถเริ่มผิดที่คำสั่งลำดับที่ 3”
- “ลองดูว่าควรเลี้ยวก่อนหรือเดินหน้าก่อน”
- “คำสั่งนี้ยังทำให้รถไถชนกองฟาง”
- “แก้ไขถูกต้อง! รถไถถึงเป้าหมายแล้ว”

### 15.9 การให้ดาว

| เงื่อนไข | ดาว |
|---|---:|
| แก้ถูกใน attempt แรกหลังดูระบบเดิม | 3 |
| แก้ถูกโดยผิด 1–2 ครั้ง | 2 |
| แก้ถูกหลังผิดมากกว่า 2 ครั้งหรือใช้คำใบ้ | 1 |

### 15.10 Asset ที่ต้องใช้

- รถไถ
- แผนที่ grid
- กองฟาง/หิน
- เป้าหมาย
- บล็อกคำสั่ง
- ไอคอน bug
- เครื่องมือแก้ไข
- เสียงถูก/ผิด
- effect ชน

## 16. Stage 12: วิกฤตน้ำท่วมฟาร์ม

### 16.1 ฉาก

ฉากเป็นฟาร์มที่กำลังเกิดวิกฤตน้ำท่วม แปลงผักบางส่วนมีน้ำขัง มีสัญญาณเตือน และระบบรดน้ำทำงานผิดพลาด

### 16.2 วัตถุในเกม

| วัตถุ | รายละเอียด |
|---|---|
| แปลงผัก | จุดที่ได้รับผลกระทบ |
| น้ำท่วม | effect แสดงความเสียหาย |
| เมฆฝน | แสดงว่าฝนตก |
| หัวฉีดน้ำ | ทำงานผิดพลาด |
| แผงควบคุมระบบ | แสดงเงื่อนไขที่ผิด |
| บล็อกเงื่อนไข | ถ้าฝนตก, ถ้าดินแห้ง, ถังน้ำเต็ม |
| บล็อกคำสั่ง | รดน้ำ, หยุดรดน้ำ, แจ้งเตือน |
| ปุ่มรันระบบ | จำลองระบบ |
| ปุ่มทดสอบหลังแก้ | ตรวจผล |

### 16.3 ผู้เล่นทำอะไร

ผู้เล่นต้องตรวจสอบเงื่อนไขที่ทำให้ระบบรดน้ำผิดพลาด แล้วแก้ไขให้ระบบหยุดรดน้ำเมื่อฝนตก และทำงานตามเงื่อนไขที่ถูกต้อง

### 16.4 วิธีการเล่น

1. กด “รันระบบเดิม”
2. ดูผลลัพธ์ เช่น ฝนตกแต่หัวฉีดยังรดน้ำ ทำให้น้ำท่วม
3. ดูแผงเงื่อนไขที่ผิด
4. สลับหรือเปลี่ยนบล็อกเงื่อนไข
5. กด “ทดสอบระบบ”
6. ผ่านเมื่อฟาร์มหยุดน้ำท่วมและระบบทำงานถูกต้อง

### 16.5 กฎการเล่น

- หากฝนไม่ตกและดินแห้ง ระบบจึงรดน้ำ
- หากถังน้ำหมด ต้องแจ้งเตือน ไม่ใช่รดน้ำ

- กดทดสอบแต่ละครั้งนับเป็น attempt
- หากระบบยังทำให้น้ำท่วมหรือพืชเหี่ยว นับ mistake

### 16.6 เงื่อนไขชนะ

ชนะเมื่อระบบผ่านสถานการณ์ทดสอบครบ เช่น

- ฝนตก → หยุดรดน้ำ

- ถังน้ำหมด → แจ้งเตือน

### 16.7 เงื่อนไขผิดพลาด

- ฝนตกแล้วยังรดน้ำ
- ดินแห้งแต่ไม่รดน้ำเมื่อไม่มีฝน
- ถังน้ำหมดแล้วยังสั่งรดน้ำ
- ดินชื้นแล้วยังรดน้ำ
- ลำดับเงื่อนไขผิด

### 16.8 Feedback

- “สำเร็จ! ฟาร์มหยุดน้ำท่วมแล้ว”

- ห้ามไม่บันทึก attempts

- ห้ามทำ gameplay ที่ไม่เกี่ยวกับ Sequence, Condition หรือ Debugging
- ห้ามใช้ชื่อบทผิด เช่น บทที่ 3 เป็น Debugging หรือบทที่ 4 เป็น Loop
