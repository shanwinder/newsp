# Drone Farm Tactics: วางแผนโดรน พิชิตภารกิจฟาร์ม

**บทเรียน:** บทที่ 3 เงื่อนไข (Condition)  
**Stage ที่เกี่ยวข้อง:** Stage 7, Stage 8, Stage 9  
**รูปแบบเกมใหม่:** Grid Strategy + Resource Management + Conditional Programming  
**ธีม:** เกษตรกร / ไร่พืช / โดรนเกษตร / การจัดการทรัพยากร  
**กลุ่มเป้าหมาย:** นักเรียนชั้นประถมศึกษาปีที่ 4  
**เป้าหมายหลัก:** เปลี่ยนบทเรียนที่ 3 จากเกมที่ยังคล้ายแบบฝึก ให้เป็นเกมวางแผนเชิงกลยุทธ์ที่มีภาพเคลื่อนไหว มีการลากวาง มีทรัพยากรจำกัด มีตารางพื้นที่เล่น และใช้ If / Else If / Else เป็นกลไกหลักในการควบคุมโดรน

## 1. สรุปแนวคิดใหม่

บทเรียนที่ 3 จะถูกออกแบบใหม่เป็นเกมชื่อ

> **Drone Farm Tactics: วางแผนโดรน พิชิตภารกิจฟาร์ม**

ผู้เล่นรับบทเป็นผู้ควบคุมโดรนเกษตร ต้องวางแผนให้โดรนออกจากฐานไปดูแลไร่พืชในตารางแนวนอน โดยแต่ละช่องของตารางมีสถานะต่างกัน เช่น ดินแห้ง พืชขาดปุ๋ย มีแมลง พืชสุก วัชพืช ฝนตก หรือช่องว่าง

ผู้เล่นต้องทำ 2 เรื่องหลัก

1. **จัดทรัพยากรให้โดรน**  
   เช่น น้ำ ปุ๋ย น้ำยากำจัดศัตรูพืช ตะกร้าเก็บเกี่ยว โดยโดรนมีน้ำหนักบรรทุกจำกัด และถ้าบรรทุกหนัก แบตเตอรี่จะลดเร็วขึ้น

2. **สร้างกฎ If / Else If / Else ให้โดรนตัดสินใจ**  
   เช่น

```text
ถ้า [มีแมลง] → [กำจัดศัตรูพืช]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้นถ้า [พืชขาดปุ๋ย] → [ใส่ปุ๋ย]
มิฉะนั้นถ้า [พืชสุก] → [เก็บเกี่ยว]
มิฉะนั้น → [บินต่อ]
```

เมื่อกด “ปล่อยโดรน” โดรนจะบินออกจากฐาน เคลื่อนที่ทีละช่อง สแกนสถานะในช่องนั้น ตรวจเงื่อนไขจากบนลงล่าง และทำคำสั่งตามกฎที่ผู้เล่นสร้าง ผลลัพธ์จะแสดงด้วยภาพเคลื่อนไหว เช่น โดรนพ่นน้ำ ใส่ปุ๋ย ไล่แมลง เก็บเกี่ยว พืชฟื้น พืชเสียหาย แบตเตอรี่ลด หรือโดรนกลับฐานไม่ทัน

หัวใจของเกมคือ

> **ผู้เล่นไม่ได้ตอบคำถามเรื่องเงื่อนไข แต่กำลังใช้เงื่อนไขเพื่อควบคุมโดรนภายใต้ทรัพยากรจำกัด**

## 2. ปัญหาจากการทดสอบที่ต้องแก้ไข

จากผลการทดสอบ พบปัญหาหลัก 4 ประเด็น ดังนี้

### 2.1 รูปแบบการเล่นแบบคลิกเลือกคำสั่งยังล้าสมัย

การคลิกเลือกคำสั่งทีละอันให้ความรู้สึกคล้ายแบบฝึกมากกว่าเกม ไม่เหมาะกับเด็กยุคใหม่ที่คุ้นเคยกับการลาก วาง ขยับ จัดเรียง และมีปฏิสัมพันธ์กับวัตถุบนหน้าจอ

**แนวทางแก้ไข:**  
เปลี่ยนระบบหลักเป็น **Drag & Drop** ได้แก่

- ลากทรัพยากรใส่ช่องบรรทุกโดรน
- ลากบล็อกเงื่อนไขใส่ช่อง If / Else If
- ลากบล็อกคำสั่งใส่ช่อง Action
- ลากสลับลำดับกฎได้
- ลากบล็อกออกเพื่อทิ้งได้

บนมือถือควรรองรับ **Tap-to-place** เป็นทางเลือกเสริม แต่บนคอมพิวเตอร์ควรให้ Drag & Drop เป็นประสบการณ์หลัก

---

### 2.2 ปุ่มควบคุมกลมกลืนกับการ์ดเงื่อนไขและคำสั่ง

ปุ่ม “ปล่อยโดรน”, “คำใบ้”, “ย้อนกลับ”, “ล้างกฎ” ไม่ควรอยู่รวมกับการ์ดเงื่อนไขและคำสั่งจนผู้เรียนแยกไม่ออกว่าอะไรคือบล็อกที่ใช้สร้างกฎ และอะไรคือปุ่มควบคุมเกม

**แนวทางแก้ไข:**  
ออกแบบ UI ด้านข้างเป็น 4 โซนที่แยกชัดเจน

```text
┌──────────────────────────────┐
│ 1. คลังบล็อกเงื่อนไข          │
│ [ดินแห้ง] [มีแมลง] [พืชสุก]   │
├──────────────────────────────┤
│ 2. คลังบล็อกคำสั่ง            │
│ [รดน้ำ] [ใส่ปุ๋ย] [เก็บเกี่ยว] │
├──────────────────────────────┤
│ 3. ช่องบรรทุกโดรน             │
│ [น้ำ] [ปุ๋ย] [น้ำยา] [ตะกร้า]  │
├──────────────────────────────┤
│ 4. แผงควบคุมเกม               │
│ [ปล่อยโดรน]                  │
│ [ย้อนกลับ] [ล้างกฎ] [คำใบ้]   │
└──────────────────────────────┘
```

ปุ่ม “ปล่อยโดรน” ต้องเป็นปุ่มใหญ่ที่สุด สีเด่นที่สุด และอยู่แยกจากบล็อกชัดเจน

### 2.3 พื้นที่เกมล้นออกจากกรอบเส้นประ

ปัญหาพื้นที่เล่นล้นกรอบเป็นปัญหาเชิง layout และ scaling ต้องแก้ในระดับโครงสร้าง ไม่ใช่แค่ลดขนาด canvas แบบเฉพาะหน้า

**แนวทางแก้ไข:**  
สร้าง layout ใหม่ที่คุมขนาดชัดเจน

```css
.game-shell {
  width: min(1180px, calc(100vw - 32px));
  max-width: 100%;
  margin: 0 auto;
  display: grid;
  grid-template-columns: minmax(0, 1fr) 340px;
  gap: 16px;
  overflow: hidden;
}

.board-panel {
  min-width: 0;
  overflow: hidden;
}

#phaser-canvas {
  width: 100%;
  max-width: 100%;
  overflow: hidden;
}

#phaser-canvas canvas {
  width: 100% !important;
  height: auto !important;
  max-width: 100%;
  display: block;
}
```

ใน Phaser ควรใช้ scale mode ที่เหมาะสม เช่น

```javascript
scale: {
  mode: Phaser.Scale.FIT,
  autoCenter: Phaser.Scale.CENTER_BOTH,
  parent: 'phaser-canvas',
  width: 900,
  height: 600
}
```

หรือคำนวณขนาดกระดานจาก container จริง เพื่อไม่ให้ canvas ล้นกรอบ

หาก Stage 9 ผ่านได้ด้วยคำสั่งแพทเทิร์นเดียว เกมจะหมดความท้าทาย เพราะผู้เรียนไม่จำเป็นต้องวิเคราะห์ภารกิจหรือปรับเงื่อนไขตามโจทย์

**แนวทางแก้ไข:**

## 3. เป้าหมายการออกแบบใหม่

### 3.1 เป้าหมายด้านความเป็นเกม

เกมใหม่ต้องมีองค์ประกอบดังนี้

- มีพื้นที่เล่นเป็นตารางชัดเจน
- มีฐานปล่อยโดรน
- มีโดรนเคลื่อนไหวจริง
- มีการลากวางทรัพยากรและบล็อกคำสั่ง
- มีข้อจำกัดด้านแบตเตอรี่ น้ำหนักบรรทุก และทรัพยากร
- มีภารกิจเฉพาะด่าน
- มีด่านย่อย 3 ด่านต่อ Stage
- มีภาพเคลื่อนไหวจากผลการตัดสินใจ
- มีคะแนน ดาว และเงื่อนไขผ่านด่าน
- มี replay หรือ log หลังปล่อยโดรน
- มีความท้าทายเพิ่มขึ้นทีละระดับ

### 3.2 เป้าหมายด้านการเรียนรู้

ผู้เรียนควรเข้าใจว่า

- If คือการตัดสินใจเมื่อพบเงื่อนไข
- Else If คือการตรวจเงื่อนไขถัดไปเมื่อเงื่อนไขก่อนหน้าไม่จริง
- Else คือทางเลือกสุดท้าย
- ลำดับของเงื่อนไขมีผลต่อผลลัพธ์
- เงื่อนไขเดียวกันอาจต้องจัดลำดับต่างกันตามภารกิจ
- การแก้ปัญหาไม่ได้มีแค่คำสั่งถูก แต่ต้องคำนึงถึงทรัพยากรด้วย
- การเขียนกฎที่ดีต้องเหมาะกับโจทย์และข้อจำกัด

## 4. โครงสร้างพื้นที่เล่น

### 4.1 ตารางฟาร์ม

พื้นที่เล่นเป็นตารางแนวนอน เช่น 3 แถว × 6 คอลัมน์ หรือ 4 แถว × 6 คอลัมน์

```text
┌──────┬────────┬────────┬────────┬────────┬────────┐
│ ฐาน  │ ช่อง 1 │ ช่อง 2 │ ช่อง 3 │ ช่อง 4 │ ช่อง 5 │
├──────┼────────┼────────┼────────┼────────┼────────┤
│ ฐาน  │ ช่อง 1 │ ช่อง 2 │ ช่อง 3 │ ช่อง 4 │ ช่อง 5 │
├──────┼────────┼────────┼────────┼────────┼────────┤
│ ฐาน  │ ช่อง 1 │ ช่อง 2 │ ช่อง 3 │ ช่อง 4 │ ช่อง 5 │
└──────┴────────┴────────┴────────┴────────┴────────┘
```

ฝั่งซ้ายสุดเป็นฐานปล่อยโดรน  
ช่องที่เหลือเป็นไร่พืช  
โดรนเริ่มจากฐาน บินไปตามเส้นทางที่กำหนด หรือบินตรวจแปลงตามลำดับที่เกมกำหนด

### 4.2 สถานะของช่องตาราง

แต่ละช่องอาจมีสถานะเดียวหรือหลายสถานะซ้อนกัน เช่น

| สถานะ | ความหมาย | คำสั่งที่เหมาะสม |
|---|---|---|
| ดินแห้ง | พืชต้องการน้ำ | รดน้ำ |
| พืชขาดปุ๋ย | พืชโตช้า | ใส่ปุ๋ย |
| มีแมลง | พืชถูกทำลาย | กำจัดศัตรูพืช |
| พืชสุก | พร้อมเก็บเกี่ยว | เก็บเกี่ยว |
| ฝนตก | ไม่ควรรดน้ำ | ข้าม / กลับฐาน |
| วัชพืช | รบกวนการเติบโต | กำจัดวัชพืช |
| ช่องว่าง | ไม่มีงานต้องทำ | บินต่อ |

ตัวอย่างช่องซ้อนสถานะ:

```text
ช่อง A: พืชสุก + มีแมลง
ช่อง B: ดินแห้ง + ฝนตก
ช่อง C: พืชขาดปุ๋ย + ดินแห้ง
```

ช่องซ้อนสถานะจะใช้สอนเรื่องลำดับเงื่อนไข

## 5. ระบบทรัพยากรโดรน

### 5.1 ทรัพยากรหลัก

โดรนมีทรัพยากรจำกัด ดังนี้

| ทรัพยากร | ใช้ทำอะไร | ข้อจำกัด |
|---|---|---|
| แบตเตอรี่ | ใช้ในการบินและทำงาน | ลดตามระยะทางและน้ำหนัก |
| น้ำ | ใช้รดน้ำ | พกได้จำกัด |
| ปุ๋ย | ใช้ใส่ปุ๋ย | พกได้จำกัด |
| น้ำยากำจัดศัตรูพืช | ใช้กำจัดแมลง | พกได้จำกัด |
| ตะกร้า | ใช้เก็บเกี่ยว | มีความจุจำกัด |
| น้ำหนักบรรทุก | รวมทุกสิ่งที่โดรนพก | ยิ่งหนัก แบตยิ่งลดเร็ว |

### 5.2 ตัวอย่างค่าน้ำหนัก

```text
ความจุโดรน: 6 หน่วย

น้ำ 1 ถัง = 2 หน่วย
ปุ๋ย 1 ถุง = 2 หน่วย
น้ำยากำจัดแมลง 1 ขวด = 1 หน่วย
ตะกร้าเก็บเกี่ยว 1 ใบ = 2 หน่วย
```

### 5.3 ผลของน้ำหนักต่อแบตเตอรี่

```text
น้ำหนัก 0–2 หน่วย: บิน 1 ช่อง ใช้แบต 5
น้ำหนัก 3–4 หน่วย: บิน 1 ช่อง ใช้แบต 7
น้ำหนัก 5–6 หน่วย: บิน 1 ช่อง ใช้แบต 10
```

### 5.4 ผลต่อการเล่น

ผู้เล่นต้องคิดว่า

- จะพกน้ำเยอะหรือปุ๋ยเยอะ
- จะพกตะกร้าเก็บเกี่ยวหรือไม่
- จะยอมบรรทุกหนักเพื่อทำงานหลายอย่างหรือไม่
- จะให้กฎสั่งกลับฐานเมื่อแบตต่ำหรือไม่
- ถ้าไม่พกทรัพยากรที่จำเป็น จะทำภารกิจล้มเหลวหรือไม่

## 6. ระบบ Drag & Drop

### 6.1 จุดที่ต้องลากวางได้

1. ลากทรัพยากรใส่ช่องบรรทุกโดรน
2. ลากบล็อกเงื่อนไขใส่ช่องเงื่อนไข
3. ลากบล็อกคำสั่งใส่ช่องคำสั่ง
4. ลากสลับลำดับกฎ
5. ลากบล็อกออกเพื่อลบ
6. ลากทรัพยากรออกจากโดรนเพื่อปรับน้ำหนัก

### 6.2 พฤติกรรม Drag & Drop ที่ควรมี

- เมื่อเริ่มลาก บล็อกขยายเล็กน้อย
- ช่องที่รับได้ต้องเรืองแสง
- เมื่อวางถูกช่อง บล็อก snap เข้า slot
- เมื่อวางผิดช่อง บล็อกเด้งกลับ
- ถ้าวางซ้ำในช่องเดิม ให้แทนที่ของเดิมหรือเตือน
- ถ้าลากสลับกฎ ให้แถวกฎเลื่อนแบบ animation
- มีเสียง “คลิก” หรือ “ป๊อบ” เมื่อวางสำเร็จ

### 6.3 รองรับมือถือ

แม้บนคอมจะใช้ Drag & Drop เป็นหลัก แต่ควรมี fallback สำหรับมือถือ

- แตะบล็อกเพื่อเลือก
- แตะช่องเพื่อวาง
- แตะบล็อกที่วางแล้วเพื่อลบ
- แตะค้างเพื่อดูคำอธิบาย

  { type: 'if', condition: 'pest', action: 'spray' },
  { type: 'else_if', condition: 'soil_dry', action: 'water' },
  { type: 'else_if', condition: 'need_fertilizer', action: 'fertilize' },

### 8.3 การประเมินกฎ

```javascript
function evaluateDroneRules(cell, drone, rules) {
  for (const rule of rules) {
    if (rule.type === 'else') {
      return rule.action;
    }

    if (checkCondition(cell, drone, rule.condition)) {
      return rule.action;
    }
  }

  return 'move_next';
}
```

### 8.4 เงื่อนไขที่รองรับ

```javascript
function checkCondition(cell, drone, condition) {
  switch (condition) {
    case 'soil_dry':
      return cell.soil === 'dry';
    case 'need_fertilizer':
      return cell.needFertilizer === true;
    case 'pest':
      return cell.pest === true;
    case 'ripe':
      return cell.ripe === true;
    case 'rain':
      return cell.rain === true;
    case 'weed':
      return cell.weed === true;
    case 'battery_low':
      return drone.battery <= 25;
    case 'water_empty':
      return drone.water <= 0;
    case 'basket_full':
      return drone.harvest >= drone.basketCapacity;
    default:
      return false;
  }
}
```

## 9. ระบบคำสั่งโดรน

### 9.1 คำสั่งพื้นฐาน

| คำสั่ง | ผลลัพธ์ | ใช้ทรัพยากร |
|---|---|---|
| รดน้ำ | ดินแห้งดีขึ้น | น้ำ |
| ใส่ปุ๋ย | พืชโตดีขึ้น | ปุ๋ย |
| กำจัดศัตรูพืช | แมลงหาย | น้ำยากำจัดแมลง |
| เก็บเกี่ยว | ได้ผลผลิต | ความจุตะกร้า |
| กำจัดวัชพืช | วัชพืชหาย | แบตเตอรี่ |
| บินต่อ | ข้ามช่อง | แบตเตอรี่ |
| กลับฐาน | จบภารกิจ/เติมทรัพยากร | แบตเตอรี่ |
| รอ/สังเกต | ไม่ทำอะไร | แบตเตอรี่เล็กน้อย |

### 9.2 ตัวอย่าง logic คำสั่ง

```javascript
function applyDroneAction(cell, drone, action) {
  switch (action) {
    case 'water':
      if (drone.water <= 0) return fail('ไม่มีน้ำพอสำหรับรดน้ำ');
      if (cell.rain) return fail('ฝนตกอยู่แล้ว การรดน้ำทำให้น้ำท่วม');
      if (cell.soil !== 'dry') return warn('ดินไม่แห้ง ใช้น้ำเกินจำเป็น');
      drone.water -= 1;
      cell.soil = 'wet';
      cell.hp += 15;
      return success('รดน้ำสำเร็จ พืชฟื้นตัว');

    case 'fertilize':
      if (drone.fertilizer <= 0) return fail('ไม่มีปุ๋ยพอ');
      if (!cell.needFertilizer) return warn('พืชไม่ได้ขาดปุ๋ย ใช้ปุ๋ยเกินจำเป็น');
      drone.fertilizer -= 1;
      cell.needFertilizer = false;
      cell.hp += 10;
      return success('ใส่ปุ๋ยสำเร็จ');

    case 'spray':
      if (drone.pesticide <= 0) return fail('ไม่มีน้ำยากำจัดศัตรูพืช');
      if (!cell.pest) return warn('ไม่มีศัตรูพืชในช่องนี้');
      drone.pesticide -= 1;
      cell.pest = false;
      cell.hp += 10;
      return success('กำจัดศัตรูพืชสำเร็จ');

    case 'harvest':
      if (!cell.ripe) return fail('ผลผลิตยังไม่สุก');
      if (cell.pest) return fail('ผลผลิตมีแมลง ต้องกำจัดก่อนเก็บเกี่ยว');
      if (drone.harvest >= drone.basketCapacity) return fail('ตะกร้าเต็ม');
      cell.ripe = false;
      drone.harvest += 1;
      return success('เก็บเกี่ยวสำเร็จ');

    case 'return_base':
      drone.returning = true;
      return success('โดรนกลับฐาน');

    default:
      return neutral('บินต่อ');
  }
}
```

# 10. โครงสร้าง Stage 7, 8, 9 แบบ 3 ด่านย่อย

## Stage 7: พื้นฐานการตัดสินใจของโดรน

**แนวคิดหลัก:** If / Else  
**เป้าหมาย:** ให้ผู้เรียนเข้าใจการตรวจเงื่อนไขเดียวและการใช้ Else  
**ทรัพยากร:** น้ำและแบตเตอรี่เป็นหลัก  
**จำนวนด่านย่อย:** 3 ด่าน

---

### Stage 7-1: รดน้ำให้ถูกแปลง

**เป้าหมายการเรียนรู้:** If แบบพื้นฐาน  
**ภารกิจ:** รดน้ำเฉพาะช่องที่ดินแห้ง  
**ตาราง:** 2 แถว × 5 คอลัมน์  
**ทรัพยากรเริ่มต้น:** น้ำ 2 ถัง, แบตเตอรี่ 100  
**เงื่อนไขที่ใช้:** ดินแห้ง  
**คำสั่งที่ใช้:** รดน้ำ, บินต่อ

กฎที่คาดหวัง:

```text
ถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินต่อ]
```

**เงื่อนไขผ่านด่าน:**

- รดน้ำช่องดินแห้งครบอย่างน้อย 2 ช่อง
- ใช้น้ำไม่เกินที่กำหนด
- โดรนกลับฐานได้

**Animation สำคัญ:**

- โดรนบินออกจากฐาน
- สแกนดิน
- พ่นน้ำ
- พืชเหี่ยวฟื้นตัว

---

### Stage 7-2: อย่ารดน้ำมั่ว

**เป้าหมายการเรียนรู้:** Else และการหลีกเลี่ยงการทำงานผิดกรณี  
**ภารกิจ:** รดน้ำเฉพาะดินแห้ง ห้ามรดน้ำดินชื้น  
**ตาราง:** 3 แถว × 5 คอลัมน์  
**ทรัพยากรเริ่มต้น:** น้ำ 2 ถัง, แบตเตอรี่ 100  
**เงื่อนไขที่ใช้:** ดินแห้ง  
**คำสั่งที่ใช้:** รดน้ำ, บินต่อ

กฎที่คาดหวัง:

```text
ถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินต่อ]
```

**ความท้าทายที่เพิ่มขึ้น:**

- มีช่องดินชื้นหลายช่อง
- ถ้ารดน้ำช่องดินชื้น จะเกิดน้ำขังและเสียคะแนน
- น้ำมีจำกัด หากรดน้ำผิดจะไม่พอสำหรับช่องที่จำเป็น

**เงื่อนไขผ่านด่าน:**

- ไม่มีช่องน้ำขัง
- รดน้ำช่องดินแห้งครบ
- น้ำเหลือหรือใช้พอดี

---

### Stage 7-3: น้ำกับแบตพอดีเป๊ะ

**เป้าหมายการเรียนรู้:** If / Else + การจัดทรัพยากรเบื้องต้น  
**ภารกิจ:** เลือกปริมาณน้ำที่เหมาะสม และรดน้ำช่องดินแห้งให้ครบก่อนแบตหมด  
**ตาราง:** 3 แถว × 6 คอลัมน์  
**ทรัพยากรเริ่มต้น:** ผู้เล่นเลือกน้ำ 1–3 ถังเอง  
**เงื่อนไขที่ใช้:** ดินแห้ง  
**คำสั่งที่ใช้:** รดน้ำ, บินต่อ, กลับฐาน

กฎที่คาดหวัง:

```text
ถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินต่อ]
```

หรือถ้ามีเงื่อนไขแบตต่ำ:

```text
ถ้า [แบตต่ำ] → [กลับฐาน]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินต่อ]
```

**ความท้าทายที่เพิ่มขึ้น:**

- ถ้าพกน้ำมากเกินไป แบตลดเร็ว
- ถ้าพกน้ำน้อยเกินไป รดน้ำไม่ครบ
- ผู้เรียนต้องลองปรับทรัพยากร

**เงื่อนไขผ่านด่าน:**

- รดน้ำครบตามเป้าหมาย
- โดรนไม่แบตหมดกลางทาง
- กลับฐานได้สำเร็จ

---

## Stage 8: การจัดลำดับเงื่อนไข

**แนวคิดหลัก:** Else If และ Priority  
**เป้าหมาย:** ให้ผู้เรียนเข้าใจว่าการเรียงลำดับเงื่อนไขมีผลต่อผลลัพธ์  
**ทรัพยากร:** น้ำ ปุ๋ย น้ำยากำจัดแมลง  
**จำนวนด่านย่อย:** 3 ด่าน

---

### Stage 8-1: ปุ๋ยก่อนน้ำหรืออะไรก่อนดี

**เป้าหมายการเรียนรู้:** เงื่อนไขซ้อนและลำดับความสำคัญ  
**ภารกิจ:** ดูแลพืชที่ขาดปุ๋ยและดินแห้ง  
**ตาราง:** 3 แถว × 6 คอลัมน์  
**ทรัพยากรเริ่มต้น:** น้ำ 2, ปุ๋ย 2, แบตเตอรี่ 100  
**เงื่อนไขที่ใช้:** พืชขาดปุ๋ย, ดินแห้ง  
**คำสั่งที่ใช้:** ใส่ปุ๋ย, รดน้ำ, บินต่อ

กฎที่เหมาะสม:

```text
ถ้า [พืชขาดปุ๋ย] → [ใส่ปุ๋ย]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินต่อ]
```

**จุดออกแบบสำคัญ:**

- บางช่องมีทั้ง “ขาดปุ๋ย” และ “ดินแห้ง”
- ถ้ารดน้ำก่อน พืชฟื้นน้อย
- ถ้าใส่ปุ๋ยก่อน ได้คะแนนสูงกว่า
- ทำให้เห็นว่า priority ขึ้นกับภารกิจ

**เงื่อนไขผ่านด่าน:**

- พืชขาดปุ๋ยได้รับปุ๋ยอย่างน้อย 2 ช่อง
- ดินแห้งได้รับน้ำอย่างน้อย 2 ช่อง
- พืช HP เฉลี่ยไม่ต่ำกว่าเกณฑ์

---

### Stage 8-2: แมลงบุก ต้องจัดการก่อน

**เป้าหมายการเรียนรู้:** การจัดลำดับเงื่อนไขที่มีผลเร่งด่วน  
**ภารกิจ:** กำจัดแมลงก่อนพืชเสียหาย แล้วจึงรดน้ำหรือใส่ปุ๋ย  
**ตาราง:** 3 แถว × 6 คอลัมน์  
**ทรัพยากรเริ่มต้น:** น้ำยา 2, น้ำ 2, แบตเตอรี่ 100  
**เงื่อนไขที่ใช้:** มีแมลง, ดินแห้ง  
**คำสั่งที่ใช้:** กำจัดศัตรูพืช, รดน้ำ, บินต่อ

กฎที่เหมาะสม:

```text
ถ้า [มีแมลง] → [กำจัดศัตรูพืช]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินต่อ]
```

**จุดออกแบบสำคัญ:**

- ถ้าเจอช่องที่ทั้งมีแมลงและดินแห้ง แล้วกฎตรวจดินแห้งก่อน แมลงจะยังทำลายพืชต่อ
- แมลงมี animation ขยับและกัดพืช
- หากปล่อยไว้นาน พืช HP ลด

**เงื่อนไขผ่านด่าน:**

- กำจัดแมลงครบตามเป้าหมาย
- พืชเสียหายไม่เกิน 2 ช่อง
- โดรนยังมีแบตเหลือเมื่อจบด่าน

---

### Stage 8-3: ฝนกำลังมา อย่าทำน้ำท่วม

**เป้าหมายการเรียนรู้:** การตรวจเงื่อนไขป้องกันความเสียหาย  
**ภารกิจ:** หลีกเลี่ยงการรดน้ำเมื่อฝนตก และดูแลพืชที่จำเป็น  
**ตาราง:** 4 แถว × 6 คอลัมน์  
**ทรัพยากรเริ่มต้น:** น้ำ 2, ปุ๋ย 1, แบตเตอรี่ 100  
**เงื่อนไขที่ใช้:** ฝนตก, ดินแห้ง, พืชขาดปุ๋ย  
**คำสั่งที่ใช้:** บินต่อ/ข้าม, รดน้ำ, ใส่ปุ๋ย

กฎที่เหมาะสม:

```text
ถ้า [ฝนตก] → [บินต่อ]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้นถ้า [พืชขาดปุ๋ย] → [ใส่ปุ๋ย]
มิฉะนั้น → [บินต่อ]
```

**จุดออกแบบสำคัญ:**

- บางช่องมี “ดินแห้ง + ฝนตก”
- ถ้าตรวจดินแห้งก่อน จะรดน้ำและเกิดน้ำท่วม
- น้ำท่วมมี animation น้ำค่อย ๆ ขัง พืชเสีย HP
- ด่านนี้เป็นสะพานไปสู่ Stage 9

**เงื่อนไขผ่านด่าน:**

- ไม่มีน้ำท่วม
- รดน้ำช่องที่ควรรดน้ำ
- ใส่ปุ๋ยช่องที่จำเป็นอย่างน้อย 1 ช่อง

## Stage 9: ภารกิจใหญ่ หลายเป้าหมาย หลายข้อจำกัด

**แนวคิดหลัก:** หลายเงื่อนไข + หลายเป้าหมาย + ทรัพยากรจำกัด  
**เป้าหมาย:** ป้องกันไม่ให้ใช้ pattern เดียวผ่านทุกด่าน  
**จำนวนด่านย่อย:** 3 ด่าน

---

### Stage 9-1: วันเก็บเกี่ยว

**เป้าหมายการเรียนรู้:** ใช้เงื่อนไขพืชสุกและการจัดทรัพยากรเก็บเกี่ยว  
**ภารกิจ:** เก็บเกี่ยวพืชสุกอย่างน้อย 4 ช่อง  
**ตาราง:** 4 แถว × 6 คอลัมน์  
**ทรัพยากรเริ่มต้น:** ผู้เล่นเลือกเองในความจุ 6 หน่วย  
**เงื่อนไขที่ใช้:** พืชสุก, ดินแห้ง  
**คำสั่งที่ใช้:** เก็บเกี่ยว, รดน้ำ, บินต่อ

กฎที่เหมาะสม:

```text
ถ้า [พืชสุก] → [เก็บเกี่ยว]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินต่อ]
```

**จุดออกแบบสำคัญ:**

- ต้องพกตะกร้าเพียงพอ
- ถ้าพกน้ำ/ปุ๋ยมากเกินไป จะไม่มีพื้นที่ตะกร้า
- ถ้าพกตะกร้าอย่างเดียว พืชบางช่องอาจเสีย HP เพราะดินแห้ง

**เงื่อนไขผ่านด่าน:**

- เก็บเกี่ยวอย่างน้อย 4 ช่อง
- โดรนกลับฐานได้
- ผลผลิตเสียหายไม่เกิน 1 ช่อง

---

### Stage 9-2: ไร่ป่วยก่อนเก็บเกี่ยว

**เป้าหมายการเรียนรู้:** ลำดับเงื่อนไขในสถานะซ้อน  
**ภารกิจ:** กำจัดแมลงก่อนเก็บเกี่ยวพืชสุก  
**ตาราง:** 4 แถว × 6 คอลัมน์  
**ทรัพยากรเริ่มต้น:** น้ำยา 2, ตะกร้า 3, น้ำ 1, แบตเตอรี่ 100  
**เงื่อนไขที่ใช้:** มีแมลง, พืชสุก, ดินแห้ง  
**คำสั่งที่ใช้:** กำจัดศัตรูพืช, เก็บเกี่ยว, รดน้ำ, บินต่อ

กฎที่เหมาะสม:

```text
ถ้า [มีแมลง] → [กำจัดศัตรูพืช]
มิฉะนั้นถ้า [พืชสุก] → [เก็บเกี่ยว]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินต่อ]
```

**จุดออกแบบสำคัญ:**

- บางช่องมีทั้ง “มีแมลง + พืชสุก”
- ถ้าเก็บเกี่ยวก่อนกำจัดแมลง จะได้ผลผลิตเสีย
- หากใช้ pattern จาก 9-1 จะไม่ผ่านเต็มดาว
- บังคับให้ผู้เรียนคิดลำดับใหม่

**เงื่อนไขผ่านด่าน:**

- กำจัดแมลงอย่างน้อย 2 ช่อง
- เก็บเกี่ยวผลผลิตดีอย่างน้อย 3 ช่อง
- ผลผลิตเสียไม่เกิน 1 ช่อง

---

### Stage 9-3: ภารกิจสุดท้าย ก่อนพายุเข้า

**เป้าหมายการเรียนรู้:** หลายเงื่อนไข + เหตุการณ์เปลี่ยนระหว่างทาง + แบตเตอรี่  
**ภารกิจ:** เก็บเกี่ยว กำจัดแมลง ดูแลดินแห้ง และกลับฐานก่อนแบตหมด  
**ตาราง:** 4 แถว × 7 คอลัมน์  
**ทรัพยากรเริ่มต้น:** ผู้เล่นจัดเอง ความจุ 6 หน่วย  
**เงื่อนไขที่ใช้:** แบตต่ำ, ฝนตก, มีแมลง, พืชสุก, ดินแห้ง  
**คำสั่งที่ใช้:** กลับฐาน, บินต่อ, กำจัดแมลง, เก็บเกี่ยว, รดน้ำ

กฎที่เหมาะสม:

```text
ถ้า [แบตต่ำ] → [กลับฐาน]
มิฉะนั้นถ้า [ฝนตก] → [บินต่อ]
มิฉะนั้นถ้า [มีแมลง] → [กำจัดศัตรูพืช]
มิฉะนั้นถ้า [พืชสุก] → [เก็บเกี่ยว]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินต่อ]
```

**เหตุการณ์พิเศษ:**

- หลังโดรนตรวจครบ 5 ช่อง ฝนเริ่มตกบางพื้นที่
- หากไม่ตรวจฝน จะเสี่ยงรดน้ำตอนฝนตก
- หากไม่ตรวจแบตต่ำ โดรนอาจตกกลางไร่
- หากพกของหนักเกิน แบตลดเร็วมาก

**เงื่อนไขผ่านด่าน:**

- เก็บเกี่ยวอย่างน้อย 3 ช่อง
- กำจัดแมลงอย่างน้อย 2 ช่อง
- ไม่มีน้ำท่วม
- โดรนกลับฐานได้ก่อนแบตหมด

**เหตุผลที่ไม่ใช้ pattern เดียวผ่านง่าย:**

- ถ้าใช้ pattern เก็บเกี่ยวก่อน จะพลาดแมลง
- ถ้าใช้ pattern แมลงก่อนอย่างเดียว อาจแบตหมด
- ถ้าไม่ตรวจฝน จะเกิดน้ำท่วม
- ถ้าไม่จัดทรัพยากรดี จะทำภารกิจไม่ครบ

## 14. Feedback และ Replay

### 14.1 Feedback ต้องมี 3 ส่วน

```text
เกิดอะไรขึ้น
เพราะอะไร
ควรแก้อย่างไร
```

ตัวอย่าง:

```text
ช่องที่ 3 ผลผลิตเสียหาย
เพราะมีแมลงอยู่ แต่โดรนเก็บเกี่ยวก่อนกำจัดแมลง
ลองย้ายเงื่อนไข “มีแมลง” ให้อยู่ก่อน “พืชสุก”
```

```text
โดรนแบตหมดกลางไร่
เพราะบรรทุกทรัพยากรหนักเกินและไม่มีเงื่อนไขกลับฐานเมื่อแบตต่ำ
ลองเพิ่มกฎ ถ้า [แบตต่ำ] → [กลับฐาน]
```

```text
เกิดน้ำขังในช่องที่ 5
เพราะฝนตกอยู่แล้ว แต่โดรนยังรดน้ำ
ลองตรวจเงื่อนไข “ฝนตก” ก่อน “ดินแห้ง”
```

### 14.2 Replay Log

หลังปล่อยโดรน ควรแสดง replay log แบบย่อ

```text
ช่อง 1: ดินแห้ง → รดน้ำ → สำเร็จ
ช่อง 2: มีแมลง → กำจัดศัตรูพืช → สำเร็จ
ช่อง 3: พืชสุก + มีแมลง → เก็บเกี่ยว → ผลผลิตเสีย
ช่อง 4: แบตต่ำ → ไม่มีคำสั่งกลับฐาน → โดรนตก
```

ควรมีไอคอนประกอบและสีบอกสถานะ

- เขียว = สำเร็จ
- เหลือง = สำเร็จแต่ไม่ดีที่สุด
- แดง = ผิดพลาด

## 19. Acceptance Criteria

### 19.1 ด้าน Gameplay

- [ ] เล่นด้วย Drag & Drop เป็นหลัก
- [ ] ลากทรัพยากรใส่โดรนได้
- [ ] ลากเงื่อนไขและคำสั่งสร้างกฎได้
- [ ] สลับลำดับกฎได้
- [ ] ปล่อยโดรนแล้วโดรนเคลื่อนไหวจริง
- [ ] โดรนทำงานตาม If / Else If / Else จริง
- [ ] ทรัพยากรลดตาม action
- [ ] แบตเตอรี่ได้รับผลจากน้ำหนักบรรทุก
- [ ] มี replay หรือ log หลังจบภารกิจ

### 19.2 ด้าน UI

- [ ] ปุ่มควบคุมแยกจากการ์ดชัดเจน
- [ ] พื้นที่เกมไม่ล้นกรอบเส้นประ
- [ ] รองรับคอมพิวเตอร์
- [ ] รองรับมือถือด้วย tap-to-place
- [ ] ข้อความอ่านง่าย
- [ ] สีแยกหมวดชัดเจน

### 19.3 ด้าน Stage Design

- [ ] Stage 7 มี 3 ด่านย่อย
- [ ] Stage 8 มี 3 ด่านย่อย
- [ ] Stage 9 มี 3 ด่านย่อย
- [ ] ความยากไต่ระดับ
- [ ] Stage 9 ใช้ pattern เดียวผ่านไม่ได้ง่าย
- [ ] แต่ละด่านมีโจทย์เฉพาะ

## 20. Prompt สำหรับสั่ง Codex พัฒนา

```text
ยกเครื่องบทเรียนที่ 3 ของเกมใหม่ทั้งหมด โดยออกแบบเป็นเกม Drone Farm Tactics: วางแผนโดรน พิชิตภารกิจฟาร์ม

แนวเกม:
Grid Strategy + Resource Management + Conditional Programming

ข้อกำหนดหลัก:
1. บทที่ 3 ประกอบด้วย stage_id 7, 8, 9
2. แต่ละ stage ต้องมีด่านย่อย 3 ด่าน รวมทั้งหมด 9 ด่านย่อย
3. เปลี่ยนรูปแบบการเล่นจากคลิกเลือกคำสั่งเป็น Drag & Drop เป็นหลัก
4. ผู้เล่นต้องลากทรัพยากรใส่โดรน เช่น น้ำ ปุ๋ย น้ำยากำจัดแมลง ตะกร้าเก็บเกี่ยว
5. โดรนมีข้อจำกัดด้านแบตเตอรี่ น้ำหนักบรรทุก และทรัพยากร
6. ผู้เล่นต้องลากบล็อกเงื่อนไขและคำสั่งเพื่อสร้าง If / Else If / Else
7. โดรนต้องเคลื่อนไหวในตารางฟาร์มจริง สแกนช่อง และทำงานตามกฎที่ผู้เล่นสร้าง
8. ตารางฟาร์มต้องมีฐานปล่อยโดรนและช่องไร่พืชที่มีสถานะ เช่น ดินแห้ง ขาดปุ๋ย มีแมลง พืชสุก ฝนตก วัชพืช
9. UI ต้องแยกชัดเจนระหว่างคลังเงื่อนไข คลังคำสั่ง ช่องทรัพยากรโดรน และปุ่มควบคุม
10. แก้ปัญหาพื้นที่เกมล้นกรอบ โดยปรับ layout และ Phaser scale ให้ responsive
11. Stage 7 เน้น If / Else
    - 7-1 รดน้ำให้ถูกแปลง
    - 7-2 อย่ารดน้ำมั่ว
    - 7-3 น้ำกับแบตพอดีเป๊ะ
12. Stage 8 เน้น Else If และลำดับเงื่อนไข
    - 8-1 ปุ๋ยก่อนน้ำหรืออะไรก่อนดี
    - 8-2 แมลงบุก ต้องจัดการก่อน
    - 8-3 ฝนกำลังมา อย่าทำน้ำท่วม
13. Stage 9 เน้นหลายเงื่อนไข หลายเป้าหมาย และ resource management
    - 9-1 วันเก็บเกี่ยว
    - 9-2 ไร่ป่วยก่อนเก็บเกี่ยว
    - 9-3 ภารกิจสุดท้าย ก่อนพายุเข้า
14. Stage 9 ต้องออกแบบไม่ให้ใช้ pattern เดียวผ่านง่าย โดยใช้ objective, resource, สถานะซ้อน และเหตุการณ์ระหว่างทางที่ต่างกัน
15. ต้องมี animation อย่างน้อย: โดรนบิน สแกน รดน้ำ ใส่ปุ๋ย กำจัดแมลง เก็บเกี่ยว พืชฟื้น พืชเสียหาย ฝนตก น้ำขัง แบตหมด
16. ต้องมี feedback และ replay log หลังปล่อยโดรน
17. คงระบบส่งคะแนนเดิมผ่าน window.sendResult(window.STAGE_ID, stars, duration, attempts)
18. หลีกเลี่ยงการเปลี่ยน database ถ้าไม่จำเป็น
```

## 23. สรุป

แนวทางใหม่ของบทเรียนที่ 3 คือการเปลี่ยนจากเกมคลิกเลือกคำสั่งหรือแบบฝึกจับคู่ ให้กลายเป็นเกม **Drone Farm Tactics** ที่มีตารางฟาร์ม ฐานปล่อยโดรน การลากวาง การจัดทรัพยากร และการสร้าง If / Else If / Else เพื่อควบคุมโดรน

จุดแข็งของแนวนี้คือ

- มีเกมเมคานิคจริง
- มีภาพเคลื่อนไหว
- มีการวางแผน
- มีทรัพยากรจำกัด
- มีความท้าทายเพิ่มขึ้นทีละระดับ
- มี 9 ด่านย่อยในบทที่ 3
- ไม่หลุดธีมเกษตร
- สอน Condition ได้ลึกกว่าเดิม
- แก้ปัญหา Stage 9 ที่ใช้ pattern เดียวผ่านง่าย

แกนสำคัญที่สุดคือ

> **ผู้เรียนต้องรู้สึกว่า “ฉันกำลังเขียนสมองให้โดรนเกษตร” ไม่ใช่ “ฉันกำลังทำแบบฝึกเรื่อง If-Else”**
