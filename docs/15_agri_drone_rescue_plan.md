# แผนการพัฒนาเกมบทเรียนที่ 3 ใหม่ทั้งหมด
# Agri-Drone Rescue: โดรนพิทักษ์ฟาร์ม

**บทเรียน:** บทที่ 3 เรื่องเงื่อนไข (Condition)  
**แนวเกม:** Action Automation Puzzle / Mission Rescue Game  
**ธีม:** เกษตรกรยุคใหม่ ใช้โดรนดูแลฟาร์ม  
**กลุ่มเป้าหมาย:** นักเรียนชั้นประถมศึกษาปีที่ 4  
**เป้าหมาย:** รื้อบทเรียนที่ 3 ใหม่ ให้เป็นเกมที่น่าเล่น มีภาพเคลื่อนไหว มีกลไกเกม และยังสอน If / Else If / Else ได้ชัดเจน

---

## 1. แนวคิดหลัก

บทเรียนที่ 3 ไม่ควรยึดติดกับเกมเครื่องรดน้ำแบบเดิมอีกต่อไป แต่ควรเปลี่ยนเป็นเกมที่ผู้เรียนรู้สึกว่า “คำสั่งของฉันทำให้ตัวละครในเกมทำงานจริง”

แนวคิดใหม่คือให้ผู้เรียนรับบทเป็นผู้ควบคุม **โดรนเกษตรอัจฉริยะ** ต้องตั้งกฎให้โดรนตัดสินใจเองว่าจะทำอะไรกับสถานการณ์ในฟาร์ม เช่น ดินแห้ง ฝนตก แมลงบุก ผลผลิตสุก หรือแบตเตอรี่ต่ำ

หัวใจของเกมคือ

> นักเรียนตั้งกฎ → โดรนบินตามกฎ → ฟาร์มเปลี่ยนแปลง → นักเรียนเห็นผล → นักเรียนแก้กฎให้ดีขึ้น

ผู้เรียนไม่ได้บังคับโดรนเองแบบเกมแอ็กชันเต็มรูปแบบ แต่เป็นผู้ “เขียนสมองให้โดรน” แล้วปล่อยให้โดรนทำงานตามเงื่อนไขที่สร้างไว้

---

## 2. ปัญหาของรูปแบบเดิม

รูปแบบเดิมยังไม่สนุกเท่าที่ควร เพราะมีลักษณะคล้ายแบบฝึกมากกว่าเกม ปัญหาหลักคือ

1. ภาพเคลื่อนไหวน้อย โลกของเกมยังไม่น่าติดตาม
2. ไม่มีตัวละครหลักที่ทำให้ผู้เรียนผูกพัน
3. การตอบถูก/ผิดยังไม่ทำให้ฟาร์มเปลี่ยนแปลงอย่างชัดเจน
4. ผู้เรียนยังไม่ได้รู้สึกว่ากำลังควบคุมระบบอัตโนมัติ
5. ไม่มีภารกิจที่สร้างความลุ้น เช่น ช่วยพืชก่อนเหี่ยว ไล่แมลงก่อนกัดพืช หรือเก็บเกี่ยวก่อนเสียหาย
6. เนื้อหาเรื่องเงื่อนไขยังถูกนำเสนอเป็นการจับคู่คำตอบ ไม่ใช่กลไกเกม

แนวใหม่จึงต้องทำให้เงื่อนไขกลายเป็น “ระบบควบคุมโดรน” ไม่ใช่ “คำตอบในแบบฝึก”

---

## 3. ภาพรวมเกม

เกมเป็นฉากฟาร์มมุมมองด้านบนหรือกึ่งไอโซเมตริก มีแปลงผักหลายแปลง ฐานโดรน บ่อน้ำ โรงเก็บผลผลิต และเหตุการณ์ต่าง ๆ ในฟาร์ม

ผู้เรียนตั้งกฎให้โดรน เช่น

```text
ถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินสำรวจต่อ]
```

ในด่านที่ยากขึ้น

```text
ถ้า [มีแมลง] → [ไล่แมลง]
มิฉะนั้นถ้า [ฝนตก] → [กลับฐาน]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้นถ้า [ผลผลิตสุก] → [เก็บเกี่ยว]
มิฉะนั้น → [บินสำรวจต่อ]
```

เมื่อกดปุ่ม **ปล่อยโดรน!** โดรนจะบินออกจากฐาน ตรวจแปลงทีละจุด และทำงานตามกฎที่ผู้เรียนวางไว้ หากกฎถูก ฟาร์มดีขึ้น หากกฎผิด ฟาร์มเสียหาย และมี feedback บอกเหตุผล

---

## 4. Core Game Loop

### 4.1 Briefing Phase: รับภารกิจ

เกมแสดงสถานการณ์สั้น ๆ เช่น

> “ฟาร์มกำลังมีปัญหา บางแปลงดินแห้ง บางแปลงมีแมลง และฝนกำลังมา ตั้งกฎให้โดรนช่วยฟาร์มให้รอด!”

### 4.2 Programming Phase: ตั้งสมองโดรน

ผู้เรียนเลือกการ์ดเงื่อนไขและการ์ดคำสั่งใส่ในแผงกฎ

```text
ถ้า [__________] → [__________]
มิฉะนั้นถ้า [__________] → [__________]
มิฉะนั้น → [__________]
```

### 4.3 Action Phase: ปล่อยโดรน

โดรนบินไปตรวจแต่ละแปลง มี animation เช่น บิน สแกน รดน้ำ ไล่แมลง เก็บเกี่ยว หรือกลับฐาน

### 4.4 Result Phase: สรุปผล

ระบบสรุปผลแบบภาพและข้อความสั้น ๆ เช่น

```text
แปลง A: ดินแห้ง → โดรนรดน้ำ → พืชฟื้น
แปลง B: ฝนตก → โดรนกลับฐาน → ปลอดภัย
แปลง C: มีแมลง → ไม่ได้ไล่แมลง → พืชเสีย HP
```

จากนั้นผู้เรียนแก้กฎและลองใหม่เพื่อให้ได้ดาวมากขึ้น

---

## 5. องค์ประกอบหลักของเกม

## 5.1 โดรนเกษตร

โดรนเป็นตัวละครหลัก ต้องมีชีวิตและตอบสนองต่อกฎของผู้เรียน

ความสามารถของโดรน:

1. บินจากฐานไปยังแปลงต่าง ๆ
2. สแกนสถานะแปลง
3. รดน้ำ
4. พ่นละอองสมุนไพรไล่แมลง
5. เก็บเกี่ยวผลผลิต
6. กลับฐานเมื่อฝนตกหรือแบตเตอรี่ต่ำ
7. เติมน้ำหรือชาร์จแบตเตอรี่
8. แสดงท่าทางดีใจหรือผิดพลาด

สถานะของโดรน:

- แบตเตอรี่
- น้ำในถังโดรน
- จำนวนผลผลิตที่เก็บได้
- ภารกิจปัจจุบัน
- จำนวนครั้งที่ทำงานผิดพลาด

## 5.2 แปลงผัก

แต่ละแปลงมีสถานะ เช่น

```javascript
{
  id: 'A',
  soil: 'dry',
  rain: false,
  pest: true,
  crop: 'ripe',
  hp: 70,
  harvested: false
}
```

สถานะที่ควรมี:

| สถานะ | ความหมาย | ภาพที่ควรแสดง |
|---|---|---|
| soil_dry | ดินแห้ง | ดินแตก พืชเหี่ยว |
| soil_wet | ดินชื้น | ดินเข้ม พืชปกติ |
| rain | ฝนตก | เมฆและฝน |
| pest | มีแมลง | หนอนหรือแมลงบนพืช |
| crop_ripe | ผลผลิตสุก | ผลผลิตมีประกาย |
| damaged | พืชเสียหาย | ใบตก สีหม่น |
| recovered | พืชฟื้น | สีสด มีประกาย |

## Stage 7: โดรนรดน้ำฉุกเฉิน

### เป้าหมายการเรียนรู้

สอน If-Else พื้นฐาน

```text
ถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินสำรวจต่อ]
```

### เรื่องราว

แดดแรงทำให้บางแปลงเริ่มแห้ง พืชเริ่มเหี่ยว ผู้เรียนต้องตั้งกฎให้โดรนรดน้ำเฉพาะแปลงที่ดินแห้ง และไม่รดน้ำแปลงที่ดินยังชื้นอยู่

### ฉาก

- ฐานโดรนอยู่มุมขวาล่าง
- มีแปลงผัก 3 แปลง
- แปลง A: ดินแห้ง พืชเหี่ยว
- แปลง B: ดินชื้น พืชปกติ
- แปลง C: ดินแห้ง พืชเริ่มเหี่ยว

### ลำดับการเล่น

1. ระบบแสดงสถานการณ์ดินแห้ง
2. ผู้เรียนวางการ์ด `ดินแห้ง` และ `รดน้ำ`
3. ผู้เรียนวาง Else เป็น `บินสำรวจต่อ`
4. กด “ปล่อยโดรน!”
5. โดรนบินออกจากฐาน
6. โดรนสแกนแปลง A แล้วรดน้ำ
7. พืชแปลง A ฟื้น
8. โดรนสแกนแปลง B แล้วบินผ่าน
9. โดรนสแกนแปลง C แล้วรดน้ำ
10. จบด่านและคำนวณดาว

### Animation สำคัญ

- โดรนบินออกจากฐาน
- เส้นสแกนส่องลงแปลง
- พื้นดินแห้งมีรอยแตก
- โดรนปล่อยละอองน้ำ
- พืชเหี่ยวเด้งกลับเป็นพืชสด
- ข้อความ “Perfect Water!” เด้งขึ้น
- ถ้ารดน้ำผิดแปลง เกิดน้ำขังเล็ก ๆ

### Feedback ตัวอย่าง

```text
ดินแห้งแต่โดรนไม่ได้รดน้ำ พืชจึงเหี่ยวลง
ลองใช้กฎ ถ้าดินแห้ง → รดน้ำ
```

```text
แปลงนี้ดินยังชื้น แต่โดรนรดน้ำเพิ่ม ทำให้เกิดน้ำขัง
กรณีที่ไม่ใช่ดินแห้ง ควรให้โดรนสำรวจต่อ
```

## Stage 8: พายุมาแล้ว เรียงเงื่อนไขให้ทัน

### เป้าหมายการเรียนรู้

สอน Else If และลำดับเงื่อนไข

```text
ถ้า [ฝนตก] → [กลับฐาน]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้น → [บินสำรวจต่อ]
```

### เรื่องราว

เมฆฝนกำลังเคลื่อนเข้ามาในฟาร์ม บางแปลงดินแห้งแต่ฝนตกแล้ว หากโดรนตรวจดินแห้งก่อนฝนตก มันจะรดน้ำซ้ำและทำให้แปลงน้ำท่วม

### ฉาก

- มีเมฆดำเคลื่อนจากด้านบนของจอ
- มีแปลงผัก 3 แปลง
- แปลง A: ฝนตกและดินแห้ง
- แปลง B: ไม่มีฝนและดินแห้ง
- แปลง C: ไม่มีฝนและดินชื้น

### ลำดับการเล่น

1. ระบบเตือนว่าพายุจะมา
2. ผู้เรียนตั้งกฎในสมองโดรน
3. กดปล่อยโดรน
4. โดรนบินไปแปลง A
5. ถ้ากฎเริ่มจากฝนตก โดรนจะกลับฐานหรือหลบฝน
6. ถ้ากฎเริ่มจากดินแห้ง โดรนจะรดน้ำทั้งที่ฝนตก ทำให้แปลง A น้ำท่วม
7. โดรนไปแปลง B ถ้าไม่มีฝนและดินแห้ง ต้องรดน้ำ
8. โดรนไปแปลง C แล้วบินสำรวจต่อ
9. ระบบสรุปผลและให้ดาว

### Animation สำคัญ

- เมฆดำลอยเข้ามา
- ฝนตกเป็นเส้นหรือ particle
- โดรนมีแอนิเมชันหลบฝนหรือกลับฐาน
- ถ้าวางกฎผิด โดรนปล่อยน้ำซ้ำกลางฝน
- น้ำท่วมแปลงค่อย ๆ สูงขึ้น
- พืชสั่นและเสีย HP
- ถ้าวางถูก ขึ้นโล่ “Storm Safe”
- Combo text เมื่อโดรนตัดสินใจถูกต่อเนื่อง

### Feedback ตัวอย่าง

```text
แปลง A น้ำท่วม
เพราะโดรนตรวจดินแห้งก่อนฝนตก จึงรดน้ำกลางฝน
ลองย้ายเงื่อนไข “ฝนตก” ไว้ก่อน “ดินแห้ง”
```

## Stage 9: วิกฤตฟาร์มใหญ่ แมลงบุก ฝนมา ผลผลิตสุก

### เป้าหมายการเรียนรู้

ใช้หลายเงื่อนไขร่วมกัน และจัดลำดับความสำคัญของภารกิจ

### เรื่องราว

ฟาร์มเข้าสู่วันสำคัญก่อนเก็บเกี่ยว แต่เกิดปัญหาหลายอย่างพร้อมกัน ทั้งแมลงบุก ดินแห้ง ฝนตก ผลผลิตสุก และแบตเตอรี่โดรนเริ่มต่ำ ผู้เรียนต้องตั้งกฎให้โดรนจัดลำดับภารกิจให้เหมาะสม

### กฎตัวอย่างสำหรับเวอร์ชันแรก

```text
ถ้า [มีแมลง] → [ไล่แมลง]
มิฉะนั้นถ้า [ฝนตก] → [กลับฐาน]
มิฉะนั้นถ้า [ดินแห้ง] → [รดน้ำ]
มิฉะนั้นถ้า [ผลผลิตสุก] → [เก็บเกี่ยว]
มิฉะนั้น → [บินสำรวจต่อ]
```

### โครงสร้าง Wave

#### Wave 1: เตรียมเก็บเกี่ยว

- มีแปลงดินแห้ง
- มีผลผลิตสุก
- ผู้เรียนต้องเลือกว่าจะรดน้ำหรือเก็บเกี่ยวให้ถูกสถานการณ์

#### Wave 2: แมลงบุก

- แมลงเริ่มเดินเข้าหาพืช
- ถ้าไม่จัดการก่อน พืชเสีย HP เร็ว
- ผู้เรียนต้องให้ความสำคัญกับเงื่อนไข “มีแมลง”

#### Wave 3: ฝนมา

- ฝนตกบางแปลง
- ถ้าโดรนรดน้ำกลางฝน จะเกิดน้ำท่วม
- ถ้าแบตต่ำ ควรกลับฐาน
- ต้องใช้ลำดับเงื่อนไขอย่างรอบคอบ

1. แมลงเดินเข้าหาพืช

4. ผลผลิตสุกมีประกาย

## 11. Algorithm สำคัญ

### 11.1 ประเมินกฎ

```javascript
function evaluateRules(plot, drone, rules) {
  for (const rule of rules) {
    if (rule.type === 'else') {
      return rule.action;
    }

    if (checkCondition(plot, drone, rule.condition)) {
      return rule.action;
    }
  }

  return 'skip';
}
```

### 11.2 ตรวจเงื่อนไข

```javascript
function checkCondition(plot, drone, condition) {
  switch (condition) {
    case 'soil_dry':
      return plot.soil === 'dry';
    case 'rain':
      return plot.rain === true;
    case 'pest':
      return plot.pest === true;
    case 'crop_ripe':
      return plot.crop === 'ripe';
    case 'battery_low':
      return drone.battery <= 25;
    case 'water_low':
      return drone.water <= 20;
    default:
      return false;
  }
}
```

### 11.3 ใช้คำสั่ง

```javascript
function applyAction(plot, drone, action) {
  if (action === 'water') {
    if (plot.rain) {
      return { result: 'flood', hpChange: -20, batteryChange: -5, waterChange: -10 };
    }

    if (plot.soil === 'dry') {
      return { result: 'recover', hpChange: +20, batteryChange: -5, waterChange: -10 };
    }

    return { result: 'overwater', hpChange: -10, batteryChange: -5, waterChange: -10 };
  }

  if (action === 'repel_pest') {
    if (plot.pest) {
      return { result: 'pest_cleared', hpChange: +15, batteryChange: -8, waterChange: 0 };
    }

    return { result: 'wrong_pest_action', hpChange: -5, batteryChange: -8, waterChange: 0 };
  }

  if (action === 'harvest') {
    if (plot.crop === 'ripe') {
      return { result: 'harvested', hpChange: +5, batteryChange: -8, waterChange: 0 };
    }

    return { result: 'unripe_harvest', hpChange: -10, batteryChange: -8, waterChange: 0 };
  }

  if (action === 'return_base') {
    return { result: 'return_base', hpChange: 0, batteryChange: +25, waterChange: 0 };
  }

  if (action === 'skip') {
    return { result: 'skip', hpChange: 0, batteryChange: -2, waterChange: 0 };
  }

  return { result: 'no_action', hpChange: -5, batteryChange: -2, waterChange: 0 };
}
```

## 15. ข้อจำกัดที่ต้องพิจารณาตามตรง

### 15.1 ต้องมี Asset

ถ้าไม่มีภาพหรือ animation เลย เกมจะยังดูเป็น prototype ดังนั้นควรเตรียมภาพอย่างน้อยชุดเล็ก ได้แก่ โดรน ฐานโดรน แปลงผัก พืชเหี่ยว พืชฟื้น เมฆฝน แมลง ผลผลิต และเอฟเฟกต์น้ำ

### 15.2 งานใหญ่กว่าเกมจับคู่

แนวนี้ต้องพัฒนา engine จริง มีโดรนเคลื่อนที่ มีสถานะ มี animation และมีระบบประเมินกฎ จึงใช้เวลามากกว่าเดิม แต่ผลลัพธ์จะดูเป็นเกมมากกว่าชัดเจน

### 15.3 ต้องคุมความยาก

ด่านแรกต้องง่ายมาก ด่านสองค่อยเพิ่มฝน ด่านสามค่อยเพิ่มแมลงและผลผลิต อย่าใส่ทุกอย่างตั้งแต่ต้น

### 15.4 อย่าให้เด็กบังคับโดรนเองทั้งหมด

ถ้าเด็กบังคับโดรนเอง เกมจะกลายเป็นเกมแอ็กชัน ไม่ใช่เกมสอนเงื่อนไข ต้องให้โดรนทำงานจากกฎของผู้เรียนเป็นหลัก

### 15.5 มือถือสำคัญมาก

เด็กบางคนอาจใช้แท็บเล็ตหรือมือถือ ต้องทำปุ่มและการ์ดให้ใหญ่ และใช้ tap-to-place แทน drag อย่างเดียว

## 18. สรุป

แนวทางใหม่ของบทเรียนที่ 3 คือ **Agri-Drone Rescue: โดรนพิทักษ์ฟาร์ม** ซึ่งใช้โดรนเป็นตัวละครหลัก และใช้ If / Else If / Else เป็น “สมอง” ของโดรน

จุดเด่นของแนวนี้คือ

1. ไม่หลุดธีมเกษตรกร
2. มีภาพเคลื่อนไหวชัดเจน
3. มีตัวละครหลักที่น่าจดจำ
4. มีภารกิจหลากหลาย
5. มีความสนุกมากกว่าเกมรดน้ำแบบเดิม
6. แสดงผลของเงื่อนไขได้ชัด
7. เหมาะกับการสอน If-Then-Else
8. เปิดพื้นที่ให้พัฒนาเป็นเกมที่เด็กอยากเล่นซ้ำ
